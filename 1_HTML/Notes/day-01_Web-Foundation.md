## What is the Internet?

The **internet** is a huge global network that connects millions of devices such as computers, mobiles, and servers across the world. These devices communicate with each other to send and receive data.

Data travels through physical mediums like **optical fiber cables**, undersea cables, and sometimes wireless networks.

### Purpose of the Internet

* To **share information** (websites, videos, files)
* To **communicate** (emails, chats, video calls)
* To access **online services** (banking, shopping, learning)

---

## Client and Server Concept

In web communication, two main roles exist:

### Client

* A client is the system that **requests data**
* Examples: Web browser, mobile app

### Server

* A server is the system that **responds with data**
* It stores websites, applications, and databases

Example:

* You open a website → your browser is the **client**
* The website machine is the **server**

---

## IP Address

* An **IP address** is a unique numerical identity given to every device connected to the internet
* It helps identify **where a request should go**

Without IP addresses, computers would not know where to send or receive data.

---

## Request Architecture (How Data Travels)

When you open a website, the request follows this flow:

1. **Client (Browser)** – You type a website URL
2. **Internet Service Provider (ISP)** – Example: Jio, Airtel
3. **DNS Server** – Finds the IP address of the website
4. **Destination Server** – Processes the request
5. **Response** – Server sends data back to the client

This entire process usually happens in milliseconds.

---

## DNS (Domain Name System)

DNS works like the **contact list of the internet**.

* Humans remember names like `google.com`
* Computers understand only IP addresses

DNS converts domain names into IP addresses so the correct server can be reached.

## What is Web Development?

Web development means **building and maintaining websites and web applications** that run on the internet.

It includes:

* Designing user interfaces
* Writing application logic
* Storing and retrieving data
* Handling user requests

---

## How a Web Page is Displayed

The process of loading a web page:

1. User types a **URL** in the browser
2. Browser sends a **request** to the server
3. Server responds with files:

   * **HTML** – structure
   * **CSS** – styling
   * **JavaScript** – behavior
4. Browser interprets these files
5. Final web page is displayed to the user

---

## Browser

A **browser** is software that allows users to access and interact with websites.

Main responsibilities:

* Send requests to servers
* Receive responses
* Display web pages
* Allow user interaction

Examples:

* Google Chrome
* Microsoft Edge
* Mozilla Firefox

---

## Full Stack Architecture

Full Stack Development involves working on all layers of a web application.

### Frontend

* Technologies: **HTML, CSS, JavaScript**
* Responsible for user interface & Handles user interactions (clicks, inputs)

### Backend

* Runs on the server
* Implements business logic
* Handles requests and responses
* Communicates with the database

### Database

* Stores application data permanently
* Examples: user details, orders, messages
