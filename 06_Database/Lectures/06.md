======================================================================
MYSQL AGGREGATE FUNCTIONS
======================================================================

DEFINITION:
Aggregate functions → perform calculations on multiple rows → return single value

MAIN FUNCTIONS:
COUNT()   |   SUM()   |   AVG()   |   MIN()   |   MAX()

======================================================================
1) COUNT()
======================================================================

DEFINITION:
Returns number of rows or number of non-NULL values

----------------------------------------------------------------------
TYPES
----------------------------------------------------------------------
COUNT(*)                     → all rows (NULL included)
COUNT(column)               → non-NULL values only
COUNT(DISTINCT column)      → unique non-NULL values

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
SELECT COUNT(*) FROM student;                 → total rows
SELECT COUNT(age) FROM student;              → non-NULL ages
SELECT COUNT(DISTINCT city) FROM student;    → unique cities

----------------------------------------------------------------------
WITH WHERE / GROUP BY
----------------------------------------------------------------------
SELECT COUNT(*) FROM student WHERE age > 20;

SELECT city, COUNT(*)
FROM student
GROUP BY city;

----------------------------------------------------------------------
MULTIPLE COLUMN DISTINCT
----------------------------------------------------------------------
SELECT COUNT(DISTINCT city, age) FROM student;

→ unique (city, age) combinations

----------------------------------------------------------------------
IMPORTANT BEHAVIOR
----------------------------------------------------------------------
• COUNT(*) counts rows
• COUNT(column) ignores NULL
• DISTINCT ignores NULL

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
• All NULL column      → COUNT(column) = 0
• Empty table          → COUNT(*) = 0
• WHERE no match       → 0
• COUNT(*) vs COUNT(1) → same in MySQL
• Expressions          → NULL ignored

----------------------------------------------------------------------
WITH JOIN ⚠
----------------------------------------------------------------------
SELECT COUNT(*)
FROM student s
JOIN orders o ON s.id = o.student_id;

→ counts joined rows (may increase count)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Always returns integer
2) Most used in analytics
3) Do not confuse COUNT(*) vs COUNT(column)

======================================================================
2) SUM()
======================================================================

DEFINITION:
Returns total of numeric column

SYNTAX:
SELECT SUM(column) FROM table;

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
SELECT SUM(age) FROM student;

----------------------------------------------------------------------
WITH WHERE / GROUP BY
----------------------------------------------------------------------
SELECT SUM(age) FROM student WHERE city = 'Delhi';

SELECT city, SUM(age)
FROM student
GROUP BY city;

 Execution order:cant use alias in group by because alias is created in select
  FROM → WHERE → GROUP BY → SELECT
----------------------------------------------------------------------
DISTINCT
----------------------------------------------------------------------
SELECT SUM(DISTINCT age) FROM student;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL ignored
• All NULL → result = NULL

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
• Empty table → NULL
• Non-numeric → error / wrong conversion ❌
• Mixed data  → strings → 0 ❌
• Large values → rare overflow

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Numeric columns only
2) Returns single value
3) Used in totals / reports

======================================================================
3) AVG()
======================================================================

DEFINITION:
Returns average value

FORMULA:
AVG = SUM(column) / COUNT(column)

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
SELECT AVG(age) FROM student;

----------------------------------------------------------------------
WITH WHERE / GROUP BY
----------------------------------------------------------------------
SELECT AVG(age) FROM student WHERE city = 'Delhi';

SELECT city, AVG(age)
FROM student
GROUP BY city;

----------------------------------------------------------------------
ROUNDING
----------------------------------------------------------------------
SELECT ROUND(AVG(age), 2) FROM student;  // 2 IS AFTER DECIMAL VALUES

----------------------------------------------------------------------
DISTINCT
----------------------------------------------------------------------
SELECT AVG(DISTINCT age) FROM student;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL ignored
• All NULL → NULL
• Always returns decimal

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
• Negative values → works normally
• Non-numeric     → error ❌

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Uses SUM + COUNT internally
2) Numeric only
3) Widely used in analytics

======================================================================
4) MIN()
======================================================================

DEFINITION:
Returns smallest value

SYNTAX:
SELECT MIN(column) FROM table;

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
SELECT MIN(age) FROM student;

----------------------------------------------------------------------
WITH WHERE / GROUP BY
----------------------------------------------------------------------
SELECT MIN(age) FROM student WHERE city = 'Delhi';

SELECT city, MIN(age)
FROM student
GROUP BY city;

----------------------------------------------------------------------
WORKS WITH
----------------------------------------------------------------------
• Numbers   → smallest
• Strings   → alphabetical first
• Dates     → earliest

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL ignored
• All NULL → NULL

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Single value output
2) Works on multiple data types

======================================================================
5) MAX()
======================================================================

DEFINITION:
Returns largest value

SYNTAX:
SELECT MAX(column) FROM table;

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
SELECT MAX(age) FROM student;

If you want the second highest salary, you need to skip the highest one and pick the next.

✅ Method 1 (Simple & Common)
SELECT MAX(salary) AS second_highest
FROM employee
WHERE salary < (SELECT MAX(salary) FROM employee);
----------------------------------------------------------------------
WITH WHERE / GROUP BY
----------------------------------------------------------------------
SELECT MAX(age) FROM student WHERE city = 'Delhi';

SELECT city, MAX(age)
FROM student
GROUP BY city;

----------------------------------------------------------------------
WORKS WITH
----------------------------------------------------------------------
• Numbers   → largest
• Strings   → alphabetical last
• Dates     → latest

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL ignored
• All NULL → NULL

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Single value output
2) Works on multiple data types

======================================================================
FINAL NOTES
======================================================================
1) All aggregates ignore NULL (except COUNT(*))
2) Operate on multiple rows → return single value
3) Often used with WHERE / GROUP BY
4) Core for analytics & reporting

MYSQL GROUP BY + HAVING
======================================================================

DEFINITION:
GROUP BY → groups rows with same values → apply aggregate per group  
HAVING   → filters grouped results (after GROUP BY)

GROUP BY is used to divide data into groups so aggregate functions can be
 applied to each group separately.

======================================================================
BASIC SYNTAX
======================================================================
SELECT column, AGG_FUNCTION(column)
FROM table
GROUP BY column;

======================================================================
BASIC EXAMPLE
======================================================================
SELECT city, COUNT(*)
FROM student
GROUP BY city;

→ count students per city

======================================================================
WHY GROUP BY
======================================================================
• Aggregation per category (city, dept, etc.)
• Data summarization (COUNT, SUM, AVG...)

======================================================================
MULTIPLE COLUMNS
======================================================================
SELECT city, age, COUNT(*)
FROM student
GROUP BY city, age;

→ grouping by combination (city + age)

======================================================================
COMMON AGGREGATIONS
======================================================================
SELECT city, AVG(age) FROM student GROUP BY city;
SELECT city, SUM(age) FROM student GROUP BY city;
SELECT city, MIN(age) FROM student GROUP BY city;
SELECT city, MAX(age) FROM student GROUP BY city;

======================================================================
GROUP BY + WHERE (ROW FILTER)
======================================================================
SELECT city, COUNT(*)
FROM student
WHERE age > 20
GROUP BY city;

→ WHERE filters rows BEFORE grouping

======================================================================
HAVING (GROUP FILTER)
======================================================================
SELECT city, COUNT(*)
FROM student
GROUP BY city
HAVING COUNT(*) > 2;

→ filters groups AFTER grouping

----------------------------------------------------------------------
HAVING WITH MULTIPLE CONDITIONS
----------------------------------------------------------------------
SELECT city, AVG(age) AS avg_age
FROM student
GROUP BY city
HAVING AVG(age) > 20 AND COUNT(*) >= 2;

----------------------------------------------------------------------
HAVING WITHOUT GROUP BY
----------------------------------------------------------------------
SELECT COUNT(*) 
FROM student
HAVING COUNT(*) > 5;

→ entire table treated as one group

======================================================================
WHERE vs HAVING
======================================================================
WHERE  → row-level filter (before GROUP BY)
HAVING → group-level filter (after GROUP BY)

----------------------------------------------------------------------
COMBINED FLOW
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
WHERE age > 20
GROUP BY city
HAVING COUNT(*) > 2;

======================================================================
GROUP BY + ORDER BY
======================================================================
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
ORDER BY total DESC;

→ sort grouped result

======================================================================
IMPORTANT RULE
======================================================================
Selected columns must be:
✔ in GROUP BY
OR
✔ used with aggregate

❌ WRONG:
SELECT city, age FROM student GROUP BY city;

✔ CORRECT:
SELECT city, COUNT(*) FROM student GROUP BY city;

======================================================================
GROUP BY vs DISTINCT (DISTINCT returns only unique values by removing duplicates.)
======================================================================
SELECT DISTINCT city FROM student;

≈

SELECT city FROM student GROUP BY city;

======================================================================
EXAMPLES (EMPLOYEES)
======================================================================
TABLE: employees(id, name, department, salary)

----------------------------------------------------------------------
UNIQUE DEPARTMENTS
----------------------------------------------------------------------
SELECT department FROM employees GROUP BY department;

----------------------------------------------------------------------
COUNT PER DEPARTMENT
----------------------------------------------------------------------
SELECT department, COUNT(*) AS total_emps
FROM employees
GROUP BY department;

----------------------------------------------------------------------
SUM PER DEPARTMENT
----------------------------------------------------------------------
SELECT department, SUM(salary) AS total_salary
FROM employees
GROUP BY department;

----------------------------------------------------------------------
FULL AGGREGATION
----------------------------------------------------------------------
SELECT 
  department,
  COUNT(*)    AS total_emps,
  SUM(salary) AS total_salary,
  AVG(salary) AS avg_salary,
  MIN(salary) AS min_salary,
  MAX(salary) AS max_salary
FROM employees
GROUP BY department;

======================================================================
ADVANCED CASES
======================================================================

----------------------------------------------------------------------
GROUP BY WITH EXPRESSIONS
----------------------------------------------------------------------
SELECT YEAR(order_date), COUNT(*)
FROM orders
GROUP BY YEAR(order_date);

----------------------------------------------------------------------
GROUP BY WITH JOIN
----------------------------------------------------------------------
SELECT s.city, COUNT(o.id)
FROM student s
JOIN orders o ON s.id = o.student_id
GROUP BY s.city;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL forms its own group
• Aggregates ignore NULL (except COUNT(*))

----------------------------------------------------------------------
NO ROWS
----------------------------------------------------------------------
→ returns EMPTY result

======================================================================
EXECUTION ORDER
======================================================================
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

======================================================================
COMMON MISTAKES
======================================================================
❌ Missing GROUP BY column
❌ Using WHERE instead of HAVING for aggregates
❌ Expecting row-level data after grouping
❌ Confusing DISTINCT with GROUP BY

======================================================================
PERFORMANCE
======================================================================
• Can be slow on large data
• Use indexes on grouped columns

======================================================================
FINAL NOTES
======================================================================
1) GROUP BY → core for analytics/reporting
2) HAVING → always used for aggregate filtering
3) WHERE + GROUP BY + HAVING often used together
4) Produces one row per group

MYSQL DATE, DATETIME & TIMESTAMP
======================================================================

======================================================================
1) DATE DATA TYPE
======================================================================

DEFINITION:
DATE → stores only date (no time)

FORMAT:
'YYYY-MM-DD'   ✔ correct  
'10-01-2024'   ❌ wrong  
'2024/01/10'   ❌ unreliable  

----------------------------------------------------------------------
CREATE / INSERT
----------------------------------------------------------------------
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  order_date DATE
);

INSERT INTO orders (order_date)
VALUES ('2024-01-10'), ('2024-02-15');

----------------------------------------------------------------------
COMPARISON
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date > '2024-02-01';

→ compared chronologically

----------------------------------------------------------------------
RANGE
----------------------------------------------------------------------
'1000-01-01' → '9999-12-31'

----------------------------------------------------------------------
COMMON FUNCTIONS
----------------------------------------------------------------------

CURDATE()  
SELECT CURDATE();                      → current date

NOW()  
SELECT NOW();                          → date + time

DATE()  
SELECT DATE('2024-03-20 10:30:00');    → extract date

YEAR / MONTH / DAY  
SELECT YEAR(order_date) FROM orders;

----------------------------------------------------------------------
DATEDIFF / TIMEDIFF
----------------------------------------------------------------------

DATEDIFF (days only)
SELECT DATEDIFF('2024-03-20','2024-03-10'); → 10
If first date is earlier → result is negative

TIMEDIFF (time only)
SELECT TIMEDIFF('10:30:00','08:15:00');     → 02:15:00

• DATEDIFF → returns number  
• TIMEDIFF → returns HH:MM:SS  
• TIME range → -838:59:59 to 838:59:59

----------------------------------------------------------------------
DATE MANIPULATION ...ADDDATE/SUBDATE..> ADDDATE('DATE', INTERVAL _ DAY);
----------------------------------------------------------------------

ADDDATE  
SELECT ADDDATE('2024-03-20', INTERVAL 5 DAY); → 2024-03-25

SUBDATE  
SELECT SUBDATE('2024-03-20', INTERVAL 5 DAY); → 2024-03-15

----------------------------------------------------------------------
FORMAT / CONVERSION
----------------------------------------------------------------------

STR_TO_DATE  
SELECT STR_TO_DATE('20-03-2024','%d-%m-%Y');

DATE_FORMAT  
SELECT DATE_FORMAT('2024-03-20','%d-%m-%Y');

SELECT DATE_FORMAT('2024-03-20 10:30:45','%d-%m-%Y %H:%i:%s');

----------------------------------------------------------------------
IMPORTANT RULES
----------------------------------------------------------------------
1) Always use quotes 'YYYY-MM-DD'
2) Stored & compared as DATE (not string)
3) Use functions for extraction/manipulation

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
• Invalid date      → '2024-02-30' ❌  
• No quotes         → treated as math ❌  
• NULL allowed      → ignored in comparisons  
• DATE vs DATETIME  → time affects comparison  
• No timezone effect (DATE only)

======================================================================
2) DATETIME
======================================================================

DEFINITION:
Stores date + time

FORMAT:
'YYYY-MM-DD HH:MM:SS'

EXAMPLE:
'2024-03-20 10:30:45'

----------------------------------------------------------------------
KEY POINTS
----------------------------------------------------------------------
• Stored as given (no timezone conversion)
• Larger range than TIMESTAMP
• No auto-update by default

======================================================================
3) TIMESTAMP
======================================================================

DEFINITION:
Stores date + time (timezone-aware)

----------------------------------------------------------------------
FORMAT
----------------------------------------------------------------------
'YYYY-MM-DD HH:MM:SS'

----------------------------------------------------------------------
CREATE TABLE
----------------------------------------------------------------------
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  created_at TIMESTAMP
);

----------------------------------------------------------------------
AUTO TIMESTAMP
----------------------------------------------------------------------
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

→ auto current date & time

----------------------------------------------------------------------
AUTO UPDATE
----------------------------------------------------------------------
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    ON UPDATE CURRENT_TIMESTAMP
);

→ updated_at changes on update

----------------------------------------------------------------------
EXAMPLE (REAL USE)
----------------------------------------------------------------------
CREATE TABLE user_logins(
  login_id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,
  login_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  logout_time TIMESTAMP
);

----------------------------------------------------------------------
COMMON FUNCTIONS
----------------------------------------------------------------------
SELECT NOW();
SELECT CURRENT_TIMESTAMP; 
// 👉 Both return current date and time — no difference in output

nOTE:
"NOW() is a function to fetch current timestamp,
while DATETIME is a data type to store date and time."
----------------------------------------------------------------------
COMPARISON
----------------------------------------------------------------------
SELECT * FROM orders
WHERE created_at > '2024-03-01 00:00:00';

----------------------------------------------------------------------
EXTRACT
----------------------------------------------------------------------
SELECT YEAR(created_at) FROM orders;
SELECT HOUR(created_at) FROM orders;

----------------------------------------------------------------------
RANGE
----------------------------------------------------------------------
'1970-01-01 00:00:01' → '2038-01-19 03:14:07'

----------------------------------------------------------------------
TIMESTAMP vs DATETIME
----------------------------------------------------------------------
TIMESTAMP:
• Timezone-aware (stored in UTC)
• Auto-update supported
• Smaller range (till 2038)

🌍 UTC Time Zone — Definition -UTC (Coordinated Universal Time) is the standard time reference used worldwide to keep clocks synchronized.
🔍 Simple Meaning:👉 It is the base time from which all other time zones are calculated

DATETIME:
• No timezone conversion
• No auto-update by default
• Larger range

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
• Timezone affects display
• 2038 limit (TIMESTAMP)
• Auto-update only if defined
• Invalid format → error ❌

----------------------------------------------------------------------
FINAL NOTES
----------------------------------------------------------------------
1) DATE → only date  
2) DATETIME → date + time (no timezone)  
3) TIMESTAMP → date + time (timezone-aware)  
4) Use TIMESTAMP for tracking (created_at, updated_at)  
5) Use DATETIME for long-term storage
