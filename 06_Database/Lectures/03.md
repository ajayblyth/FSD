MYSQL DEFAULT CONSTRAINT
====================================================================================================

Definition :
DEFAULT → Assigns value when no value is provided during INSERT

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
CREATE TABLE table_name (
    column datatype DEFAULT value
);

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    city VARCHAR(50) DEFAULT 'Delhi'
);

INSERT INTO student (id, name)
VALUES (1, 'Ajay');

Result :
id | name | city
-----------------------
1  | Ajay | Delhi   (default applied)

----------------------------------------------------------------------------------------------------
USAGE CASES
----------------------------------------------------------------------------------------------------
-- Override default
INSERT INTO student (id, name, city)
VALUES (2, 'Rahul', 'Mumbai');

-- Explicit default
INSERT INTO student (id, name, city)
VALUES (3, 'Amit', DEFAULT);

----------------------------------------------------------------------------------------------------
ALTER TABLE
----------------------------------------------------------------------------------------------------
-- Add / Change default
ALTER TABLE student
ALTER city SET DEFAULT 'Bangalore';

-- Remove default
ALTER TABLE student
ALTER city DROP DEFAULT;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Applied only when value not provided
• DEFAULT ≠ NOT NULL (NULL still allowed)
• Works with numbers, strings, dates
• Improves consistency

• Priority : Given value > NULL > DEFAULT

MYSQL AUTO_INCREMENT
====================================================================================================

Definition :
AUTO_INCREMENT → Auto-generates unique sequential numbers (mostly for ID)

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
CREATE TABLE table_name (
    id INT AUTO_INCREMENT PRIMARY KEY,
    column datatype
);

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

INSERT INTO student (name, age)
VALUES ('Ajay',22), ('Rahul',23), ('Amit',21);

Result :
id | name  | age
-----------------------
1  | Ajay  | 22
2  | Rahul | 23
3  | Amit  | 21

----------------------------------------------------------------------------------------------------
USAGE CASES
----------------------------------------------------------------------------------------------------
-- Auto next value
INSERT INTO student (name, age)
VALUES ('Neha', 20);   → id = 4

-- Manual override
INSERT INTO student (id, name, age)
VALUES (10, 'Ajay', 20);

→ Next values continue from 10+

----------------------------------------------------------------------------------------------------
SET START VALUE
----------------------------------------------------------------------------------------------------
ALTER TABLE student AUTO_INCREMENT = 100;

----------------------------------------------------------------------------------------------------
RULES
----------------------------------------------------------------------------------------------------
• Only ONE AUTO_INCREMENT column
• Must be PRIMARY KEY or UNIQUE
• Values increase automatically
• No duplicate values allowed
• Deleted values are NOT reused

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Used for ID columns
• Ensures uniqueness automatically
• Supports INT, BIGINT

====================================================================================================
MYSQL COMPARISON OPERATORS
====================================================================================================

Definition :
Used to compare values (mainly in WHERE clause)

----------------------------------------------------------------------------------------------------
OPERATORS
----------------------------------------------------------------------------------------------------
=   Equal          !=  , <> Not equal
>   Greater        <         Less
>=  Greater=       <=        Less=

----------------------------------------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------------------------------------
SELECT * FROM student WHERE age = 22;
SELECT * FROM student WHERE age != 22;
SELECT * FROM student WHERE age > 20;
SELECT * FROM student WHERE age < 25;
SELECT * FROM student WHERE age >= 21;
SELECT * FROM student WHERE age <= 23;

----------------------------------------------------------------------------------------------------
OTHER CONDITIONS
----------------------------------------------------------------------------------------------------
BETWEEN :
SELECT * FROM student WHERE age BETWEEN 18 AND 22;

IN :
SELECT * FROM student WHERE city IN ('Delhi','Mumbai');

LIKE :
SELECT * FROM student WHERE name LIKE 'A%';

NULL :
SELECT * FROM student WHERE age IS NULL;

----------------------------------------------------------------------------------------------------
IMPORTANT CASES
----------------------------------------------------------------------------------------------------

1) 3-VALUE LOGIC
----------------------------------------------------------------------------------------------------
TRUE / FALSE / NULL (UNKNOWN)

SELECT 5 = NULL;   → NULL
SELECT NULL > 10;  → NULL

----------------------------------------------------------------------------------------------------
2) NULL COMPARISON
----------------------------------------------------------------------------------------------------
WRONG  : age = NULL
RIGHT  : age IS NULL / IS NOT NULL

----------------------------------------------------------------------------------------------------
3) STRING COMPARISON
----------------------------------------------------------------------------------------------------
'Ajay' = 'Ajay ' → FALSE (space matters)

• CHAR may ignore trailing spaces (config dependent)

----------------------------------------------------------------------------------------------------
4) LEXICOGRAPHICAL ORDER
----------------------------------------------------------------------------------------------------
'apple' < 'banana' → TRUE
Depends on collation (case sensitivity)

----------------------------------------------------------------------------------------------------
5) NO MATCH CASE
----------------------------------------------------------------------------------------------------
No rows returned ≠ NULL
→ Just EMPTY RESULT

----------------------------------------------------------------------------------------------------
6) NULL IN CONDITIONS
----------------------------------------------------------------------------------------------------
SELECT * FROM student WHERE age < 30;
→ NULL rows ignored

----------------------------------------------------------------------------------------------------
7) NULL-SAFE equality OPERATOR
----------------------------------------------------------------------------------------------------
<=>  (important)

SELECT NULL <=> NULL → TRUE
SELECT 5 <=> NULL    → FALSE

NOTES
----------------------------------------------------------------------------------------------------
• NULL = unknown (not 0 / empty)
• Always use IS NULL
• Used inside WHERE
• Can combine using AND / OR

INTERVIEW LINE
---------------------------------------------------------------
"Use IS NULL for checking NULL, 
and <=> when comparing values where NULL handling is required."

MYSQL DATE + COMPARISON
====================================================================================================
----------------------------------------------------------------------------------------------------
TABLE
----------------------------------------------------------------------------------------------------
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_name VARCHAR(50),
    order_date DATE
);

----------------------------------------------------------------------------------------------------
INSERT
----------------------------------------------------------------------------------------------------
INSERT INTO orders (order_name, order_date)
VALUES 
('Order1','2024-01-10'),
('Order2','2024-02-15'),
('Order3','2024-03-20'),
('Order4','2024-03-25'),
('Order5',NULL);

----------------------------------------------------------------------------------------------------
USAGE
----------------------------------------------------------------------------------------------------

=  (Equal)
SELECT * FROM orders WHERE order_date = '2024-03-20';
→ Order3

>  (Greater)
SELECT * FROM orders WHERE order_date > '2024-02-01';
→ Order2,3,4

<  (Less)
SELECT * FROM orders WHERE order_date < '2024-03-01';
→ Order1,2

>= (Greater=)
SELECT * FROM orders WHERE order_date >= '2024-03-20';
→ Order3,4

<= (Less=)
SELECT * FROM orders WHERE order_date <= '2024-02-15';
→ Order1,2

BETWEEN
SELECT * FROM orders
WHERE order_date BETWEEN '2024-02-01' AND '2024-03-20';
→ Order2,3

!= (Not equal)
SELECT * FROM orders WHERE order_date != '2024-03-20';
→ Order1,2,4   (NULL ignored)

NULL
SELECT * FROM orders WHERE order_date IS NULL;
→ Order5

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Date format → 'YYYY-MM-DD'
• NULL values are ignored in comparisons
• Use IS NULL for NULL checks

MYSQL DATE / DATETIME — CASES & CORRECTIONS
====================================================================================================

----------------------------------------------------------------------------------------------------
COMMON CASES
----------------------------------------------------------------------------------------------------

1) WRONG DATE FORMAT
----------------------------------------------------------------------------------------------------
• Correct format → 'YYYY-MM-DD'

WRONG  :
SELECT * FROM orders WHERE order_date = '10-01-2024';
→ May return empty / incorrect

RIGHT  :
SELECT * FROM orders WHERE order_date = '2024-01-10';

----------------------------------------------------------------------------------------------------
2) QUOTES ARE IMPORTANT
----------------------------------------------------------------------------------------------------
WRONG  :
SELECT * FROM orders WHERE order_date = 2024-01-10;
→ Treated as arithmetic → (2024-1-10 = 2013)

RIGHT  :
SELECT * FROM orders WHERE order_date = '2024-01-10';

----------------------------------------------------------------------------------------------------
3) INVALID DATE
----------------------------------------------------------------------------------------------------
SELECT * FROM orders WHERE order_date = '2024-02-30';
→ Invalid because February never has 30 days, error / empty (depends on SQL mode)

====================================================================================================
DATE vs DATETIME (COMPARISON)
====================================================================================================

----------------------------------------------------------------------------------------------------
TABLE
----------------------------------------------------------------------------------------------------
CREATE TABLE events (
    id INT AUTO_INCREMENT PRIMARY KEY,
    event_date DATE,
    event_datetime DATETIME
);

----------------------------------------------------------------------------------------------------
DATA
----------------------------------------------------------------------------------------------------
INSERT INTO events (event_date, event_datetime)
VALUES
('2024-03-20','2024-03-20 10:30:00'),
('2024-03-21','2024-03-21 15:45:00'),
('2024-03-22','2024-03-22 00:00:00');

----------------------------------------------------------------------------------------------------
CASES
----------------------------------------------------------------------------------------------------

1) DATE = DATE
SELECT * FROM events WHERE event_date = '2024-03-20';
→ Row 1

2) DATETIME = DATETIME
SELECT * FROM events WHERE event_datetime = '2024-03-20 10:30:00';
→ Row 1

3) DATE vs DATETIME
SELECT * FROM events WHERE event_datetime = '2024-03-20';
→ EMPTY

Reason :
'2024-03-20' → treated as '2024-03-20 00:00:00' ≠ 10:30:00

----------------------------------------------------------------------------------------------------
SOLUTIONS
----------------------------------------------------------------------------------------------------

-- RANGE METHOD (best practice because Using range captures ALL times in that day)
SELECT * FROM events
WHERE event_datetime >= '2024-03-20'
AND event_datetime <  '2024-03-21';

-- USING DATE()(to ignore time)
SELECT * FROM events
WHERE DATE(event_datetime) = '2024-03-20';

----------------------------------------------------------------------------------------------------
MORE CASES
----------------------------------------------------------------------------------------------------
SELECT * FROM events WHERE event_datetime > '2024-03-20';
→ After midnight of 20th

----------------------------------------------------------------------------------------------------
KEY NOTES
----------------------------------------------------------------------------------------------------
• DATE     → only date
• DATETIME → date + time
• Time matters in comparison
• Use DATE() to ignore time

====================================================================================================
GENERAL DATE NOTES
====================================================================================================
• Format → 'YYYY-MM-DD'
• Compared chronologically
• BETWEEN includes both ends
• NULL ignored in comparison
• Use IS NULL for NULL checks

MYSQL LOGICAL OPERATORS
====================================================================================================

Definition :
Logical operators → Combine multiple conditions

----------------------------------------------------------------------------------------------------
OPERATORS
----------------------------------------------------------------------------------------------------
AND   → All conditions TRUE
OR    → Any condition TRUE
NOT   → Reverse condition
XOR   → Only one TRUE

----------------------------------------------------------------------------------------------------
AND
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE age > 20 AND city = 'Delhi';

→ Both conditions must be TRUE

----------------------------------------------------------------------------------------------------
OR
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE age < 20 OR city = 'Mumbai';

→ Any one TRUE

----------------------------------------------------------------------------------------------------
NOT
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE NOT city = 'Delhi';

→ Excludes Delhi

----------------------------------------------------------------------------------------------------
XOR
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE age > 20 XOR city = 'Delhi';

→ Only one condition TRUE
----------------------------------------------------------------------------------------------------
PRECEDENCE
----------------------------------------------------------------------------------------------------
1) NOT   → highest
2) AND
3) XOR
4) OR    → lowest

----------------------------------------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------------------------------------
TRUE AND NULL   → NULL
FALSE AND NULL  → FALSE

TRUE OR NULL    → TRUE
FALSE OR NULL   → NULL

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Use () for clarity
• NULL affects results
• XOR → less common but useful

====================================================================================================
AND + OR (WITHOUT PARENTHESES)
====================================================================================================

IMPORTANT RULE :
----------------------------------------------------------------------------------------------------
AND has higher priority than OR

Execution :
→ AND runs first → then OR

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE age > 20 OR city = 'Delhi' AND marks > 70;

ACTUAL :
WHERE age > 20
   OR (city = 'Delhi' AND marks > 70);

----------------------------------------------------------------------------------------------------
UNDERSTANDING
----------------------------------------------------------------------------------------------------
Result includes rows where:
1) age > 20
OR
2) city = Delhi AND marks > 70

----------------------------------------------------------------------------------------------------
COMMON MISTAKE
----------------------------------------------------------------------------------------------------
Expected :
(age < 18 OR age > 60) AND city = 'Mumbai'

Actual :
age < 18 OR (age > 60 AND city = 'Mumbai')

→ Different results ⚠️

----------------------------------------------------------------------------------------------------
BEST PRACTICE
----------------------------------------------------------------------------------------------------
Always use parentheses:

SELECT * FROM student
WHERE (age < 18 OR age > 60)
AND city = 'Mumbai';

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• AND > OR priority
• Without () → MySQL decides order
• Can cause logical errors
• Always use () in real projects
