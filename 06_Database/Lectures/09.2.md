=====================================================================
WINDOW FUNCTIONS
=====================================================================
DEFINITION
----------------------------------------------------------------------
• Perform calculations across a set of rows (window)
• Do NOT collapse rows (unlike GROUP BY)
• Each row keeps its identity

=====================================================================
AGGREGATE vs WINDOW
----------------------------------------------------------------------
• Aggregate → collapses rows into ONE result per group
• Window    → keeps all rows + adds calculated column

• Aggregate uses GROUP BY
• Window uses OVER()

• Aggregate loses row-level detail
• Window keeps row-level detail


=====================================================================
SYNTAX
----------------------------------------------------------------------
SELECT column1,
       column2,
       function_name(column) OVER (
           PARTITION BY column
           ORDER BY column
       ) AS alias_name
FROM table_name;


=====================================================================
IMPORTANT TERMS
----------------------------------------------------------------------
• OVER()        → defines window (Without OVER → aggregate collapses rows, With OVER → keeps rows, just adds calculation)
• PARTITION BY  → divides data into groups (keeps rows)
• ORDER BY      → defines order inside partition


=====================================================================
WINDOW FRAME (ADVANCED)
----------------------------------------------------------------------
ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
→ considers full partition (start to end)


=====================================================================
EXAMPLE: MAX SALARY PER DEPT
----------------------------------------------------------------------
SELECT *,
       MAX(salary) OVER (
           PARTITION BY department
           ORDER BY salary
           ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
       ) AS max_salary
FROM employees;

LOGIC:
• Partition by department
• Looks at ALL rows in that department
• Same max value shown in each row


=====================================================================
EXAMPLE 1: ROW_NUMBER()
----------------------------------------------------------------------
A window function that gives a unique number to each row
Starts from 1 and increases (no duplicates)

👉 “Find the ranking of employees within each department based on salary (highest salary = rank 1), without duplicates.”

SELECT name, department, salary,
       ROW_NUMBER() OVER (PARTITION BY department
                         ORDER BY salary DESC) AS rank
        FROM employees;

• Unique ranking (no duplicates)
• Highest salary → rank 1


---------------- IPL EXAMPLE ----------------

“For each match, assign a rank to players based on runs scored (highest runs = 1), resetting ranking for every match.”

SELECT *,
       ROW_NUMBER() OVER (
           PARTITION BY match_id
           ORDER BY runs DESC
       ) AS rn
FROM ipl_matches;

• Resets numbering per match
• Highest runs → rn = 1

match_id   player   runs   rn
--------------------------------
1          C        70     1
1          A        50     2
1          B        30     3

2          E        90     1   ← starts again
2          F        60     2
2          D        40     3

👉 Reset does NOT mean:
“value becomes 0 again”

👉 It means:
“ranking restarts for each partition”


---------------- TOP SCORER USING CTE ----------------

WITH top_scorer AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY match_id
               ORDER BY runs DESC
           ) AS rn
    FROM ipl_matches
)
SELECT * FROM top_scorer
WHERE rn = 1;


---------------- INTERVIEW GOLD ----------------

3. Top 1 employee per department

🎯 Question:
👉 “Find the employee with highest salary in each department.”

SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS rn
    FROM employees
) t
WHERE rn = 1;


🔥 4. Top 2 scorers in each match (IPL)

🎯 Question:
👉 “Find top 2 players with highest runs in each match.”

SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY match_id ORDER BY runs DESC) AS rn
    FROM ipl_matches
) t
WHERE rn <= 2;


🔥 5. Remove duplicates

🎯 Question:
👉 “Remove duplicate records, keeping only one row per group.”

SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY name ORDER BY id) AS rn
    FROM employees
) t
WHERE rn = 1;


🎯 Final understanding

👉 ROW_NUMBER() is used when:
You need unique ranking
You want top N per group
You want deduplication


🧠 Interview summary

ROW_NUMBER() is commonly used to assign unique ranks within partitions and solve problems like top-N per group, deduplication, and ranking scenarios.


NOTE:
• ROW_NUMBER() → different rank for same values
• Use RANK() if same rank needed

EXAMPLE 2: RANK() vs DENSE_RANK()
----------------------------------------------------------------------
SELECT name, salary,
       RANK() OVER (ORDER BY salary DESC) AS rnk,
       DENSE_RANK() OVER (ORDER BY salary DESC) AS dense_rnk
FROM employees;

RANK():        1, 2, 2, 4   (skips)
DENSE_RANK():  1, 2, 2, 3   (no skip)

EXAMPLE 3: RUNNING TOTAL
----------------------------------------------------------------------
SELECT name, salary,
       SUM(salary) OVER (ORDER BY salary) AS running_total
FROM employees;


=====================================================================
EXAMPLE 4: PARTITION + SUM
----------------------------------------------------------------------
SELECT department, name, salary,
       SUM(salary) OVER (PARTITION BY department) AS dept_total
FROM employees;


=====================================================================
EXAMPLE 5: LAG / LEAD (ROW COMPARISON)
----------------------------------------------------------------------
SELECT date, sales,
       LAG(sales, 1) OVER (ORDER BY date) AS prev_day_sales,
       sales - LAG(sales, 1) OVER (ORDER BY date) AS change
FROM sales;

SELECT date, sales,
       LEAD(sales, 1) OVER (ORDER BY date) AS next_day_sales,
       LEAD(sales, 1) OVER (ORDER BY date) - sales AS change
FROM sales;

• LAG()  → previous row
• LEAD() → next row
• Used for trend / day-to-day analysis


=====================================================================
EXAMPLE 6: NTILE() (BUCKETING)
----------------------------------------------------------------------
SELECT name, salary,
       NTILE(4) OVER (ORDER BY salary DESC) AS bucket
FROM employees;

• Divides data into equal groups


--- MORE CASES ---

SELECT name, salary,
       NTILE(2) OVER (ORDER BY salary DESC) AS half
FROM employees;

• 1 → top 50%
• 2 → bottom 50%


SELECT department, name, salary,
       NTILE(3) OVER (PARTITION BY department ORDER BY salary DESC) AS bucket
FROM employees;

• Bucket inside each department

=====================================================================
COMMON DOUBTS IN WINDOW FUNCTIONS
=====================================================================

1. WHY NOT USE GROUP BY?
---------------------------------------------------------
• GROUP BY → merges rows
• WINDOW → keeps rows
• Use WINDOW when you need row-level + aggregate together

2. DIFFERENCE BETWEEN PARTITION BY & GROUP BY?
---------------------------------------------------------
• GROUP BY → reduces rows
• PARTITION BY → divides data but keeps all rows

3. IS ORDER BY MANDATORY?
---------------------------------------------------------
• Not always
• Required for ranking, running total
• Optional for simple partition calculations

4. WHY OVER() IS REQUIRED?
---------------------------------------------------------
• Defines window (scope of calculation)
• Without OVER() → function becomes aggregate

5. CAN WE USE WHERE WITH WINDOW FUNCTION?
---------------------------------------------------------
• NO (directly)
• Use subquery or CTE

Example:
SELECT * FROM (
    SELECT name, salary,
           RANK() OVER (ORDER BY salary DESC) rnk
    FROM employees
) t
WHERE rnk <= 3;

6. PERFORMANCE?
---------------------------------------------------------
• Slightly heavy on large data
• But optimized in modern DBs

7. DIFFERENCE BETWEEN ROW_NUMBER, RANK, DENSE_RANK?
---------------------------------------------------------
• ROW_NUMBER → unique rank (1,2,3)
• RANK → skips duplicates (1,2,2,4)
• DENSE_RANK → no skip (1,2,2,3)



=====================================================================
ADDITIONAL / ADVANCED WINDOW CONCEPTS (NO REPETITION)
=====================================================================

=====================================================================
1. FIRST_VALUE() / LAST_VALUE()
----------------------------------------------------------------------
• Used to get first or last value in a window

Example:
SELECT name, salary,
       FIRST_VALUE(salary) OVER (ORDER BY salary DESC) AS highest_salary
FROM employees;

• FIRST_VALUE → first row in window
• LAST_VALUE  → last row in window (needs proper frame)


=====================================================================
2. ROWS vs RANGE (WINDOW FRAME DIFFERENCE)
----------------------------------------------------------------------
ROWS:
• Works on physical rows
• Counts exact number of rows

RANGE:
• Works on logical range of values
• Groups same values together (important with duplicates)

👉 Interview twist:
• RANGE may include multiple rows with same ORDER BY value
• ROWS always moves row by row


=====================================================================
3. DEFAULT WINDOW FRAME (IMPORTANT)
----------------------------------------------------------------------
When ORDER BY is used:

Default frame:
RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW

👉 Important:
• Affects running total calculations
• Many developers are unaware of this default behavior


=====================================================================
4. PERCENTAGE / RATIO CALCULATION
----------------------------------------------------------------------
• Used to find contribution of each row to total

Example:
SELECT name, salary,
       salary * 100.0 / SUM(salary) OVER() AS percentage
FROM employees;

• SUM(salary) OVER() → total of entire table
• Used in analytics and reporting


=====================================================================
5. WINDOW FUNCTION EXECUTION ORDER
----------------------------------------------------------------------
Execution order in SQL:

Step	Clause	Explanation
1	FROM	Tables are identified and joined (JOINs processed here).
2	WHERE	Filters rows before grouping. Only rows meeting conditions are passed forward.
3	GROUP BY	Groups rows based on specified columns.
4	HAVING	Filters groups after aggregation.
5	WINDOW FUNCTIONS	Calculates row-wise analytics (e.g., ROW_NUMBER(), RANK(), SUM() OVER …) after grouping but before SELECT final output.
6	SELECT	Columns are selected, expressions calculated.
7	ORDER BY	Final result rows are sorted.

👉 Important:
• Window functions are applied AFTER GROUP BY
• Cannot use window functions directly in WHERE


=====================================================================
6. FILTERING WINDOW FUNCTION RESULTS (PATTERN)
----------------------------------------------------------------------
• Must use subquery or CTE

Example:

WITH ranked AS (
    SELECT *,
           ROW_NUMBER() OVER (ORDER BY salary DESC) AS rn
    FROM employees
)
SELECT *
FROM ranked
WHERE rn <= 3;

👉 Standard pattern for:
• Top N queries
• Ranking filters


=====================================================================
7. PARTITION BY vs ORDER BY (INSIDE OVER)
----------------------------------------------------------------------
PARTITION BY:
• Divides data into groups
• Similar to GROUP BY but keeps rows

ORDER BY:
• Defines order inside partition
• Required for ranking and running totals

👉 Together:
• PARTITION → grouping
• ORDER → sequence within group
