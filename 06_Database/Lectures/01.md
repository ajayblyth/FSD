DATA IN DATABASE
====================================================================================================

Definition :
Data = Raw facts/information stored for processing.
Examples : Numbers, Text, Images, Transactions
Note     : Raw data has little meaning → becomes useful after processing

Important :
MySQL is generally case-insensitive, but can behave case-sensitive on some systems
→ Best practice : Always assume CASE-SENSITIVE

----------------------------------------------------------------------------------------------------
TYPES OF DATA
----------------------------------------------------------------------------------------------------

1) READABLE DATA (Human-readable)
----------------------------------------------------------------------------------------------------
Definition : Easily understood by humans

Examples   : Names ("Ajay"), Numbers (101), Text ("Hello"), Dates (17-03-2026)

Used in    : Reports, UI, Query Outputs

----------------------------------------------------------------------------------------------------
2) BINARY DATA (Machine-readable)
----------------------------------------------------------------------------------------------------
Definition : Stored as 0s and 1s (not directly readable)

Examples   : Images (JPG/PNG), Videos (MP4), Audio (MP3), Files (PDF/DOC)

Stored as  : BLOB (Binary Large Object)
Used in    : Multimedia & file storage

----------------------------------------------------------------------------------------------------
KEY POINT
----------------------------------------------------------------------------------------------------
Readable Data → Human-friendly
Binary Data   → Needs system/software to interpret

INTERVIEW LINE :
Data = Raw facts stored in structured form (readable or binary)

====================================================================================================
DATABASE
====================================================================================================

Definition :
Database = Structured collection of data stored digitally for easy access & management.
Electronically → stored in digital form inside a computer or storage device

Example :
ID    Name     Age
101   Ajay     22
102   Rahul    23

----------------------------------------------------------------------------------------------------
FEATURES
----------------------------------------------------------------------------------------------------
• Stores large data                  • Organized (tables: rows & columns)
• Easy CRUD operations              • Security + Backup & Recovery
• Reduced redundancy                • Multi-user access
• Ensures data consistency

----------------------------------------------------------------------------------------------------
COMPONENTS
----------------------------------------------------------------------------------------------------
• Table   → Data storage
• Row     → Record
• Column  → Field / Attribute

INTERVIEW LINE :
Database = Structured storage enabling easy access, security, and multi-user operations

====================================================================================================
DBMS (DATABASE MANAGEMENT SYSTEM)
====================================================================================================

Definition :
DBMS = Software used to create, manage, and control databases

Examples :
• MySQL   • Oracle   • SQL Server   • PostgreSQL

----------------------------------------------------------------------------------------------------
PURPOSE
----------------------------------------------------------------------------------------------------
• Store data     • Manage data     • Retrieve efficiently

----------------------------------------------------------------------------------------------------
FEATURES
----------------------------------------------------------------------------------------------------
1) Storage        → Large data handling
2) Organization   → Tables (rows/columns)
3) Operations     → Insert, Update, Delete, Retrieve
4) Security       → Users/Roles control
5) Backup         → Recovery support
6) Redundancy     → Minimization
7) Multi-user     → Concurrent access

----------------------------------------------------------------------------------------------------
COMPONENTS
----------------------------------------------------------------------------------------------------
• Database        • DBMS Software        • User        • SQL (Query Language)

----------------------------------------------------------------------------------------------------
ADVANTAGES / DISADVANTAGES
----------------------------------------------------------------------------------------------------
Advantages      : Consistency, Integrity, Security, Sharing, Reduced redundancy
Disadvantages   : Costly, Requires skill, Failure impacts entire system

INTERVIEW LINE :
DBMS = Software that provides storage, security, and efficient data management

====================================================================================================
SQL (STRUCTURED QUERY LANGUAGE)
====================================================================================================

Definition :
SQL = Language used to interact with DBMS (create, manage, retrieve data)

MySQL → an open-source Relational Database Management System (RDBMS)

• Used to store, manage, and retrieve data in structured form (tables)
• Works using SQL (Structured Query Language)
• Supports operations like INSERT, SELECT, UPDATE, DELETE
• Ensures data integrity, security, and multi-user access
• Widely used in web applications and backend systems

SQL vs MySQL
----------------------------------------------------------------------
SQL   → Language
MySQL → Software (RDBMS)

----------------------------------------------------------------------
SQL (Structured Query Language)
• Used to write queries (SELECT, INSERT, UPDATE, DELETE)
• Standard language for databases
• Works with many databases (MySQL, Oracle, PostgreSQL)

----------------------------------------------------------------------
MySQL
• Database management system (tool/software)
• Uses SQL to interact with data
• Stores and manages data in tables

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
"SQL is the language used to communicate with databases,
 while MySQL is a database system that uses SQL."

----------------------------------------------------------------------------------------------------
PURPOSE
----------------------------------------------------------------------------------------------------
• Create DB/Tables      • Insert/Update/Delete data
• Retrieve data         • Control access

----------------------------------------------------------------------------------------------------
TYPES OF SQL COMMANDS
----------------------------------------------------------------------------------------------------

DDL (Data Definition Language)   : CREATE, ALTER, DROP, TRUNCATE
→ Used to define/modify database structure

DML (Data Manipulation Language) : INSERT, UPDATE, DELETE
→ Used to modify data inside tables

DQL (Data Query Language)        : SELECT
→ Used to fetch/retrieve data

DCL (Data Control Language)      : GRANT, REVOKE
→ Used to control access/permissions

TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT
→ Used to manage transactions
----------------------------------------------------------------------------------------------------

----------------------------------------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------------------------------------
SELECT * FROM table_name;

INSERT INTO table_name VALUES (...);

UPDATE table_name
SET column = value
WHERE condition;

DELETE FROM table_name
WHERE condition;

----------------------------------------------------------------------------------------------------
KEY POINTS
----------------------------------------------------------------------------------------------------
• Case-insensitive (follow safe practice)
• Works on relational databases
• Table-based (rows & columns)

INTERVIEW LINE :
SQL = Language to create, manage, and retrieve data from databases

====================================================================================================
RDBMS (RELATIONAL DBMS)
====================================================================================================

Definition :
RDBMS = DBMS storing data in related tables using keys

----------------------------------------------------------------------------------------------------
KEY FEATURES
----------------------------------------------------------------------------------------------------
• Table-based storage        • Relationships via keys
• Data integrity             • Reduced redundancy
• SQL support

SCHEMA IN MYSQL
----------------
A schema is a logical container for database objects like tables, views, indexes, and procedures.
It helps organize and manage database objects within a database.
In MySQL, the terms "database" and "schema" are often used interchangeably.
----------------------------------------------------------------------------------------------------
KEY CONCEPTS
----------------------------------------------------------------------------------------------------
• Primary Key  → Unique identifier
• Foreign Key  → Links tables
• Relationship → Table connection

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
STUDENT
StudentID   Name     Age
101         Ajay     22
102         Rahul    23

COURSE
CourseID    CourseName
1           Java
2           SQL

ENROLLMENT
StudentID   CourseID

Explanation :
• StudentID → PK (STUDENT)
• CourseID  → PK (COURSE)
• ENROLLMENT → FK relationship (Student ↔ Course)

INTERVIEW LINE :
RDBMS = Data organized into related tables using keys

====================================================================================================
NoSQL (NOT ONLY SQL)
====================================================================================================

Definition :
NoSQL = Non-relational DB storing data in flexible formats (not table-based)

Used for : Big data, real-time apps, distributed systems

----------------------------------------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------------------------------------
• MongoDB   • Cassandra   • Redis   • Neo4j

----------------------------------------------------------------------------------------------------
KEY FEATURES
----------------------------------------------------------------------------------------------------
• Schema-less (no fixed structure)
• High scalability & performance
• Flexible models
• Handles unstructured/semi-structured data

----------------------------------------------------------------------------------------------------
DATA MODELS
----------------------------------------------------------------------------------------------------
• Document   → JSON-like
• Key-Value  → key:value pairs
• Column     → Column-based
• Graph      → Nodes & relationships

----------------------------------------------------------------------------------------------------
EXAMPLE (JSON)
----------------------------------------------------------------------------------------------------
{
  "id": 101,
  "name": "Ajay",
  "age": 22,
  "courses": ["Java", "SQL"]
}

INTERVIEW LINE :
NoSQL = Flexible, scalable, high-performance non-relational database
====================================================================================================
RDBMS vs NoSQL (COMPARISON)
====================================================================================================

Feature           | RDBMS                               | NoSQL
----------------------------------------------------------------------------------------------------
Full Form         | Relational DBMS                     | Non-relational (Document / Key-Value / Graph)
Structure         | Tables (rows & columns)             | Flexible (JSON, key-value, etc.)
Schema            | Fixed schema                        | Schema-less (dynamic)
Best For          | Structured data                     | Structured + Unstructured data
Scalability       | Vertical scaling                    | Horizontal scaling
Performance       | Moderate (joins may slow)           | High (optimized for speed)
Relationships     | Strong (keys & joins)               | Weak / handled differently
Query Language    | SQL                                 | No fixed standard (varies)
Examples          | MySQL, Oracle                       | MongoDB, Redis
Use Case          | Banking, ERP systems                | Big Data, real-time apps
Flexibility       | Low                                 | High

----------------------------------------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------------------------------------
RDBMS = Structured, table-based, fixed schema  
NoSQL = Flexible, schema-less, scalable systems

MySQL COMMAND PROMPT (CLI) - NOTES
====================================================================================================

1) START & LOGIN
----------------------------------------------------------------------------------------------------
mysql --version          → Check MySQL installation
mysql -u root -p         → Login (enter password - hidden)

Note : Always remember username & password

----------------------------------------------------------------------------------------------------
2) BASIC NAVIGATION
----------------------------------------------------------------------------------------------------
system cls               → Clear screen
exit                     → Exit MySQL
mysql -u root -p         → Login again

----------------------------------------------------------------------------------------------------
3) DATABASE COMMANDS
----------------------------------------------------------------------------------------------------
show databases;          → List all databases
create database test;  → Create database
drop database test;      → Delete database

use social_db;           → Select database
select database();       → Show current database

----------------------------------------------------------------------------------------------------
4) TABLE COMMANDS
----------------------------------------------------------------------------------------------------
create table people(
  id INT,
  name VARCHAR(30),
  age INT,
  location VARCHAR(30)
);

show tables;             → List tables in current DB

describe people;         → Table structure
-- OR
show columns from people;

----------------------------------------------------------------------------------------------------
5) TERMINAL TIPS
----------------------------------------------------------------------------------------------------
• ";" mandatory after every command
• \c → Cancel incomplete command
• ↑ (up arrow) → Previous commands

----------------------------------------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------------------------------------
MySQL CLI is used to execute SQL commands for database creation,
management, and data operations.
