STACK, CALL STACK, AND WHY JAVASCRIPT IS SINGLE-THREADED
========================================================================================================================

1) WHAT IS A STACK (GENERAL CONCEPT)
------------------------------------------------------------------------------------------------------------------------
Stack is a linear data structure that follows the principle:

                                LIFO → Last In First Out

Meaning:
The last element added to the stack is the first element removed.

Example (Real Life)

Stack of plates

        ┌───────┐   ← Last plate placed
        │ Plate │
        │ Plate │
        │ Plate │
        └───────┘
           ↑
        First plate placed


STACK OPERATIONS
------------------------------------------------------------------------------------------------------------------------
1) PUSH  → Add element to top of stack
2) POP   → Remove element from top of stack
3) PEEK  → View top element without removing


Example (Conceptual)

Push → 10
Push → 20
Push → 30

Stack

Top
30
20
10

Pop → removes 30

2) CALL STACK IN JAVASCRIPT
========================================================================================================================

DEFINITION
------------------------------------------------------------------------------------------------------------------------
Call Stack is a stack data structure used by the JavaScript engine to manage function execution.

It keeps track of:
• which function is currently running
• which function called another function
• where execution should return after a function finishes


PURPOSE
------------------------------------------------------------------------------------------------------------------------
1) Manage function execution order
2) Track nested function calls
3) Ensure functions execute one at a time
4) Remove functions after execution completes


HOW CALL STACK WORKS
------------------------------------------------------------------------------------------------------------------------

When a function is called  → PUSHED into the stack
When the function finishes → POPPED from the stack

Execution always happens from the TOP of the stack.


Example

function one(){
    console.log("Function One");
}

function two(){
    one();
    console.log("Function Two");
}

two();


STEP BY STEP EXECUTION
------------------------------------------------------------------------------------------------------------------------

Step 1 → Global execution starts

Call Stack
----------
global()


Step 2 → two() is called

Call Stack
----------
two()
global()


Step 3 → two() calls one()

Call Stack
----------
one()
two()
global()


Step 4 → one() finishes

Call Stack
----------
two()
global()


Step 5 → two() finishes

Call Stack
----------
global()


Step 6 → program ends

Call Stack
----------
(empty)


OUTPUT
------------------------------------------------------------------------------------------------------------------------
Function One
Function Two


IMPORTANT RULE
------------------------------------------------------------------------------------------------------------------------
JavaScript executes **only the function at the top of the stack**.

3) STACK OVERFLOW
========================================================================================================================

DEFINITION
------------------------------------------------------------------------------------------------------------------------
Stack Overflow occurs when too many function calls are pushed into the Call Stack and the stack memory limit is exceeded.


COMMON CAUSES
------------------------------------------------------------------------------------------------------------------------
1) Infinite recursion
2) Very deep nested function calls
3) Missing stopping condition in recursive functions


EXAMPLE (INFINITE RECURSION)
------------------------------------------------------------------------------------------------------------------------

function test(){
    test();
}

test();


FLOW
------------------------------------------------------------------------------------------------------------------------

Call Stack keeps growing

test()
test()
test()
test()
test()
test()
test()
...

Since there is **no stopping condition**, the stack keeps expanding
until the memory limit is reached.

JavaScript throws error:

RangeError: Maximum call stack size exceeded

4) RECURSION AND BASE CASE
========================================================================================================================

RECURSION DEFINITION
------------------------------------------------------------------------------------------------------------------------
Recursion is a programming technique where a function calls itself to solve a problem.

Every recursive function must have:

1) Base Case      → condition where recursion stops
2) Recursive Case → function calls itself again
3) Progress       → each call moves closer to base case


EXAMPLE WITH BASE CASE
------------------------------------------------------------------------------------------------------------------------

function countDown(n){

    if(n === 0){
        return;                 ← Base Case (stops recursion)
    }

    console.log(n);
    countDown(n - 1);           ← Recursive Call
}

countDown(5)


EXECUTION FLOW
------------------------------------------------------------------------------------------------------------------------

countDown(5)
countDown(4)
countDown(3)
countDown(2)
countDown(1)
countDown(0)  → stops


OUTPUT
------------------------------------------------------------------------------------------------------------------------
5
4
3
2
1


HOW TO AVOID STACK OVERFLOW
------------------------------------------------------------------------------------------------------------------------
1) Always define a BASE CASE
2) Ensure recursion moves toward the base case
3) Avoid infinite recursive calls
4) For very large iterations prefer loops


LOOP ALTERNATIVE
------------------------------------------------------------------------------------------------------------------------

for(let i = 5; i > 0; i--){
    console.log(i);
}



========================================================================================================================
5) JAVASCRIPT IS SINGLE-THREADED
========================================================================================================================

DEFINITION
------------------------------------------------------------------------------------------------------------------------
JavaScript is a single-threaded programming language, meaning it executes **one task at a time using a single Call Stack**.



WHAT IS A THREAD
------------------------------------------------------------------------------------------------------------------------
Thread = smallest unit of execution inside a program.

Single-threaded program
→ One task runs at a time

Multi-threaded program
→ Multiple tasks run simultaneously



WHY JAVASCRIPT WAS DESIGNED SINGLE-THREADED
------------------------------------------------------------------------------------------------------------------------
JavaScript was originally created for browsers to control webpage behavior and interact with the DOM.

If multiple threads modified the DOM simultaneously, problems could occur such as:

1) Race conditions
2) Data inconsistency
3) UI conflicts
4) Complex synchronization problems

Example problem

Thread A → changes button color
Thread B → deletes button

Both executing at same time → unpredictable results.

Therefore JavaScript uses a **single thread with controlled execution**.

6) HOW JAVASCRIPT HANDLES ASYNCHRONOUS TASKS (BEHIND THE SCENES)
========================================================================================================================

Even though JavaScript is single-threaded, it can handle asynchronous operations using the browser environment.

Main components:

1) Call Stack
2) Web APIs
3) Callback Queue
4) Event Loop


SIMPLIFIED FLOW
------------------------------------------------------------------------------------------------------------------------

JS Code
   ↓
Call Stack executes synchronous code
   ↓
Async operations (timer, fetch, events)
sent to Web APIs
   ↓
When completed → result goes to Callback Queue
   ↓
Event Loop moves it to Call Stack when stack becomes empty


EXAMPLE
------------------------------------------------------------------------------------------------------------------------

console.log("Start");

setTimeout(function(){
    console.log("Timer Done");
},2000);

console.log("End");


EXECUTION FLOW
------------------------------------------------------------------------------------------------------------------------

Step 1
Call Stack executes

console.log("Start")

Output
Start


Step 2
setTimeout registered in Web API
(not executed immediately)


Step 3
Next line executes

console.log("End")

Output
End


Step 4
After 2 seconds

Callback Queue
---------------
Timer callback


Step 5
Event Loop pushes callback to Call Stack

console.log("Timer Done")


FINAL OUTPUT
------------------------------------------------------------------------------------------------------------------------
Start
End
Timer Done
