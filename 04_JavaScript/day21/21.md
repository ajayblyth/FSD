==============================================================================
AJAX (Asynchronous JavaScript And XML)
==============================================================================

DEFINITION                              WHY USE AJAX
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
• AJAX = Asynchronous JavaScript and XML   • Faster user experience
• Used to send/receive data without reload• No full page reload
                                          • Partial data update
                                          • Improves performance

------------------------------------------------------------------------------
HOW IT WORKS
------------------------------------------------------------------------------
1. User action (click / input)   2. JavaScript sends request   3. Server processes request
4. Server sends response (JSON/XML)   5. Page updates dynamically

------------------------------------------------------------------------------
TECHNOLOGIES USED
------------------------------------------------------------------------------
• JavaScript        • XMLHttpRequest (old)        • Fetch API (modern)        • JSON (mostly used)

------------------------------------------------------------------------------
1. USING XMLHttpRequest (OLD WAY)
------------------------------------------------------------------------------
var xhr = new XMLHttpRequest();
xhr.open("GET", "https://api.example.com/data", true);
xhr.onload = function () {
    if (xhr.status === 200) {
        console.log(xhr.responseText);
    }
};
xhr.send();

------------------------------------------------------------------------------
2. USING FETCH API (MODERN WAY)
------------------------------------------------------------------------------
fetch("https://api.example.com/data")
.then(response => response.json())
.then(data => console.log(data))
.catch(err => console.log(err));

------------------------------------------------------------------------------
3. USING ASYNC/AWAIT (BEST PRACTICE)
------------------------------------------------------------------------------
async function getData() {
    try {
        let response = await fetch("https://api.example.com/data");
        let data = await response.json();
        console.log(data);
    } catch (err) {
        console.log(err);
    }
}

------------------------------------------------------------------------------
IMPORTANT POINTS                      AJAX vs SYNCHRONOUS
------------------------------------------------------------------------------  ---------------------------------------------
• AJAX is asynchronous (non-blocking) AJAX (Async)              SYNCHRONOUS
• No page reload required            ---------------------------------------------
• JSON replaced XML                  Non-blocking             Blocking
• Works in background                Faster UX                Slower
                                     No reload                Full reload
                                     Background processing    Wait for response

------------------------------------------------------------------------------
INTERVIEW LINE
------------------------------------------------------------------------------
"AJAX allows web pages to communicate with server asynchronously without reloading the entire page."
------------------------------------------------------------------------------

==============================================================================
HTTP (HyperText Transfer Protocol)
==============================================================================

DEFINITION                              KEY FEATURES
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
• Protocol used for communication        • Stateless (no memory of requests)
  between client and server              • Request-Response model
• Foundation of web                      • Works on TCP/IP
                                         • Uses URLs to access resources

------------------------------------------------------------------------------
HOW HTTP WORKS
------------------------------------------------------------------------------
Client (Browser) → HTTP Request → Server → HTTP Response → Client

------------------------------------------------------------------------------
HTTP REQUEST                          HTTP RESPONSE
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
• Sent by client                      • Sent by server
• Contains method, URL, headers       • Contains status code, headers, body
• May contain body                    • Contains requested data

------------------------------------------------------------------------------
COMMON HTTP METHODS
------------------------------------------------------------------------------
GET    → Fetch data  
POST   → Send/Create data  
PUT    → Update full data  
PATCH  → Update partial data  
DELETE → Remove data  

------------------------------------------------------------------------------
HTTP STATUS CODE CATEGORIES
------------------------------------------------------------------------------
1xx → Informational  
2xx → Success  
3xx → Redirection  
4xx → Client Error  
5xx → Server Error  

------------------------------------------------------------------------------
HEADERS
------------------------------------------------------------------------------
• Provide additional info (metadata)
• Examples:
  - Content-Type: application/json
  - Authorization: Bearer token

------------------------------------------------------------------------------
STATELESS NATURE
------------------------------------------------------------------------------
• Each request is independent
• Server does not remember previous requests
• Sessions handled via cookies / tokens

------------------------------------------------------------------------------
HTTP vs HTTPS
------------------------------------------------------------------------------
HTTP                              HTTPS
--------------------------------  --------------------------------
Not secure                         Secure (encrypted)
Data in plain text                 Data encrypted (SSL/TLS)
Port 80                            Port 443

------------------------------------------------------------------------------
IMPORTANT POINTS
------------------------------------------------------------------------------
• HTTP is stateless
• Uses request-response cycle
• Works on client-server model
• HTTPS is secure version

------------------------------------------------------------------------------
INTERVIEW LINE
------------------------------------------------------------------------------
"HTTP is a stateless protocol used for communication between client and server using a request-response model."
------------------------------------------------------------------------------

==============================================================================
HTTP REQUEST & STATUS CODES (COMPLETE NOTES)
==============================================================================

HTTP REQUEST                          STRUCTURE OF HTTP REQUEST
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
• A request sent from client → server 1. Method (GET, POST, etc.)
• Used to fetch or send data          2. URL (endpoint)
                                      3. Headers (metadata)
                                      4. Body (data – optional)

------------------------------------------------------------------------------
HTTP METHODS
------------------------------------------------------------------------------
GET    → Fetch data (no body, safe, idempotent)  
POST   → Send data, create resource  
PUT    → Update entire resource  
PATCH  → Partial update  
DELETE → Remove resource  

------------------------------------------------------------------------------
EXAMPLE (HTTP REQUEST)
------------------------------------------------------------------------------
GET /api/users HTTP/1.1        Host: example.com

------------------------------------------------------------------------------
HTTP STATUS CODES
------------------------------------------------------------------------------
• Server response to client request → Indicates success/failure

------------------------------------------------------------------------------
1xx → INFORMATIONAL            2xx → SUCCESS
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
• 100 Continue                 200 OK        → Request successful
                              201 Created   → Resource created
                              204 No Content→ No response body

------------------------------------------------------------------------------
3xx → REDIRECTION              4xx → CLIENT ERROR
------------------------------------------------------------------------------  ------------------------------------------------------------------------------
301 Moved Permanently          400 Bad Request   → Invalid request
302 Found                      401 Unauthorized  → Authentication required
                              403 Forbidden     → No permission
                              404 Not Found     → Resource not found

------------------------------------------------------------------------------
5xx → SERVER ERROR
------------------------------------------------------------------------------
500 Internal Server Error → Server failed
502 Bad Gateway
503 Service Unavailable

------------------------------------------------------------------------------
REAL EXAMPLE (FETCH)
------------------------------------------------------------------------------
fetch("/api/data")
.then(res => {
    if (res.status === 200) {
        return res.json();
    }
});

------------------------------------------------------------------------------
IMPORTANT POINTS
------------------------------------------------------------------------------
• 2xx → success        • 4xx → client mistake        • 5xx → server issue
• Status code always comes in response

------------------------------------------------------------------------------
INTERVIEW LINE
------------------------------------------------------------------------------
"HTTP request is sent by client to server, and status codes indicate the result of that request."

==============================================================================
ADDING DATA IN URL (KEY-VALUE PAIRS)
==============================================================================

DEFINITION
------------------------------------------------------------------------------
• Data can be sent in URL as key=value pairs
• Used mainly with GET requests

------------------------------------------------------------------------------
SYNTAX
------------------------------------------------------------------------------
?key1=value1&key2=value2

------------------------------------------------------------------------------
EXAMPLE
------------------------------------------------------------------------------
https://example.com/search?name=ajay&age=25

------------------------------------------------------------------------------
BREAKDOWN
------------------------------------------------------------------------------
• ?  → starts query parameters  
• &  → separates multiple key-value pairs  
• =  → assigns value to key  

------------------------------------------------------------------------------
REAL EXAMPLE (HTTP GET)
------------------------------------------------------------------------------
GET /search?name=ajay&age=25 HTTP/1.1

==================== FETCH API (COMPLETE NOTES) ====================

DEFINITION
----------------------------------------------------------------------
• Fetch API is a modern JavaScript interface to make HTTP requests
• Used to communicate with servers (APIs)
• Promise-based (asynchronous)

fetch is built in inside windows object
==================== WHY FETCH ====================
----------------------------------------------------------------------
• Replaces XMLHttpRequest (XHR)
• Cleaner and more readable syntax
• Works seamlessly with async/await
• Better request-response handling


==================== HOW FETCH WORKS ====================
----------------------------------------------------------------------
1. Client sends request using fetch()
2. Server processes request
3. Server returns response (JSON / text / etc.)
4. fetch() returns a Promise (Response object)
5. Data extracted using response.json(), text(), etc.


==================== BASIC SYNTAX ====================
----------------------------------------------------------------------
fetch(url, options)


==================== KEY CONCEPTS ====================
----------------------------------------------------------------------

1. PROMISE BASED
   • Returns Promise → pending → fulfilled → rejected

2. REQUEST
   • URL, method, headers, body

3. RESPONSE
   • status, headers, body

4. BODY METHODS
   • response.json()
   • response.text()
   • response.blob()


==================== DEFAULT BEHAVIOR ====================
----------------------------------------------------------------------
• Method = GET
• Only network errors reject Promise
• HTTP errors (404, 500) must be handled manually


==================== HTTP METHODS ====================
----------------------------------------------------------------------
GET     → fetch data
POST    → create data
PUT     → update full data
PATCH   → update partial data
DELETE  → remove data


==================== QUERY PARAMETERS ====================
----------------------------------------------------------------------
fetch("/api/users?city=delhi&age=30")

• Key-value pairs added using ? and &
• Used for filtering/searching
• Visible in URL (not secure)
• Length is limited

INTERVIEW LINE
"Query parameters are key-value pairs added to URL to send data in GET requests."

----------------------------------------------------------------------
async function getData() {
    try {
        const res = await fetch("https://api.example.com/data");

        if (!res.ok) {
            throw new Error("HTTP error");
        }

        const data = await res.json();
        console.log(data);

    } catch (err) {
        console.log(err);
    }
}

        "Authorization": "Bearer token_here"
    }

==================== ADVANTAGES ====================
----------------------------------------------------------------------

• Promise-based (async/await support)
• Flexible (headers, body, methods)
• Supports modern features (CORS, streams)


==================== LIMITATIONS ====================
----------------------------------------------------------------------
• No automatic timeout
• Manual JSON parsing (response.json())
• Manual error handling required


==================== FETCH vs XHR ====================
----------------------------------------------------------------------

----------------------------------------------------------------------

Modern API                     Older API

----------------------------------------------------------------------

----------------------------------------------------------------------

==================== SUMMARY ====================
----------------------------------------------------------------------
• fetch = modern way to call APIs in JavaScript
• Promise-based and async/await friendly
• Requires manual handling of response & errors



==================== AXIOS (COMPLETE NOTES - THEORY + PRACTICAL) ====================

DEFINITION
----------------------------------------------------------------------
• Axios is a Promise-based HTTP client for JavaScript
• Used to make API calls (GET, POST, PUT, PATCH, DELETE)
• Works in Browser + Node.js


==================== WHY AXIOS ====================
----------------------------------------------------------------------
• Automatic JSON parsing (no .json() needed)
• Better error handling than fetch
• Interceptors (very important in real apps)
• Handles headers, auth, base URL easily
• Cleaner syntax for complex APIs


==================== INSTALLATION ====================
----------------------------------------------------------------------
npm install axios

--in browserr


METHOD 1: CDN (EASIEST)
----------------------------------------------------------------------
• No installation needed
• Just add script tag in HTML

==================== BASIC USAGE ====================

==================== HTTP METHODS ====================
----------------------------------------------------------------------

==================== RESPONSE OBJECT ====================
----------------------------------------------------------------------
res.data       → actual data
res.status     → status code (200, 404)
res.headers    → response headers
res.config     → request config


==================== ERROR HANDLING (IMPORTANT) ====================
----------------------------------------------------------------------
axios.get("/users")
.catch(error => {

    if (error.response) {
        // Server responded (4xx, 5xx)
        console.log(error.response.status);
        console.log(error.response.data);

    } else if (error.request) {
        // Request sent but no response
        console.log("No response from server");

    } else {
        // Other errors
        console.log(error.message);
    }
})

==================== INTERCEPTORS (VERY IMPORTANT) ====================

REQUEST INTERCEPTOR
----------------------------------------------------------------------
• Modify request before sending (e.g., add token)

api.interceptors.request.use(config => {
    config.headers.Authorization = "Bearer token";
    return config;
});


RESPONSE INTERCEPTOR
----------------------------------------------------------------------
• Handle response globally (e.g., errors, logging)

api.interceptors.response.use(
    response => response,
    error => {
        if (error.response.status === 401) {
            console.log("Unauthorized");
        }
        return Promise.reject(error);
    }
);
