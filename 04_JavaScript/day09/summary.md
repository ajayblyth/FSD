// ================= QUICK REVISION SUMMARY =================


// ===== 1) FUNCTION EXPRESSIONS =====
// A function expression means storing a function inside a variable.
// It is usually anonymous (no name).
// It is not hoisted, so it cannot be called before definition.


// ===== 2) HIGHER ORDER FUNCTIONS =====
// A Higher Order Function either takes a function as argument
// or returns a function.
// Possible because functions are treated like values in JavaScript.


// ===== 3) RETURNING A FUNCTION (CLOSURE IDEA) =====
// A function can return another function.
// The returned function remembers variables from outer function.
// This concept is related to closures.


// ===== 4) METHODS IN OBJECTS =====
// A method is a function inside an object.
// It represents an action the object can perform.
// Called using dot notation (object.method()).


// ===== 5) ARRAYS ARE OBJECTS =====
// Arrays are special type of objects.
// That’s why typeof array gives "object".
// Arrays have built-in methods like push() and pop().


// ===== 6) this KEYWORD =====
// "this" refers to the object calling the method.
// Used to access properties inside the same object.
// Mostly used inside object methods.


// ===== 7) TRY & CATCH =====
// try block contains risky code.
// catch block runs if an error occurs.
// Prevents program from stopping due to errors.
