============================================================================================================================
ARRAY METHODS (JS)
============================================================================================================================

1) push(value) 
- Adds element at end. 
Syntax: arr.push(value); 
Example: let arr = [1,2]; arr.push(3); // [1,2,3]

2) pop() 
- Removes last element. 
Syntax: arr.pop(); 
Example: arr.pop(); // [1,2]

3) unshift(value) 
- Adds element at beginning. 
Syntax: arr.unshift(value); 
Example: arr.unshift(0); // [0,1,2]

4) shift() 
- Removes first element. 
Syntax: arr.shift(); 
Example: arr.shift(); // [1,2]

5) indexOf(value) 
- Returns first index of value, -1 if not found. 
Example: ["red","yellow","blue"].indexOf("yellow"); // 1

6) includes(value) 
- Checks if value exists, returns true/false. 
Example: ["red","yellow"].includes("green"); // false

7) concat(array) 
- Combines arrays, returns new array. 
Example: [1,2].concat([3,4]); // [1,2,3,4]

8) reverse() 
- Reverses array in place. 
Example: [1,2,3].reverse(); // [3,2,1]

============================================================================================================================
ARRAY METHOD : slice()
============================================================================================================================

Definition:
- Copies portion of array, does NOT modify original.

Syntax:
arr.slice(start, end) // end exclusive, negative index counts from end

Examples:
let colors = ["red","yellow","blue","orange","pink","white"];
colors.slice();       // ["red","yellow","blue","orange","pink","white"]
colors.slice(2);      // ["blue","orange","pink","white"]
colors.slice(2,3);    // ["blue"]
colors.slice(-2);     // ["pink","white"]

============================================================================================================================
ARRAY METHOD : splice()
============================================================================================================================

Definition:
- Removes / replaces / adds elements
- Modifies original array in-place
- Returns removed elements

Syntax:
arr.splice(start, deleteCount, item1, item2, ...)

Examples – COLORS ARRAY:
let colors = ["red","yellow","blue","orange","pink","white"];
colors.splice(4);               // ["pink","white"]
colors                              // ["red","yellow","blue","orange"]

colors.splice(0,1);             // ["red"]
colors                              // ["yellow","blue","orange"]

colors.splice(0,1,"black","grey"); // ["yellow"]
colors                              // ["black","grey","blue","orange"]

Examples – CARS ARRAY:
let cars = ["audi","bmw","xuv","maruti"];
cars.splice(3);                  // ["maruti"]
cars                              // ["audi","bmw","xuv"]

cars.splice(0,1);                // ["audi"]
cars                              // ["bmw","xuv"]

cars.push("maruti"); cars.push("ferrari"); 
cars                              // ["bmw","xuv","maruti","ferrari"]

cars.splice(1,2);                // ["xuv","maruti"]
cars                              // ["bmw","ferrari"]

cars.splice(0,0,"toyota","xuv","maruti"); // []
cars                              // ["toyota","xuv","maruti","bmw","ferrari"]

============================================================================================================================
ARRAY METHOD : sort() (JS)
============================================================================================================================

Definition:
- Sorts array elements in place (modifies original)
- Default: alphabetical (string-based)
- Numbers: use compare function

------------------------------------------------
STRING COMPARISON LOGIC
------------------------------------------------
- Compared character by character
- Based on Unicode values
- Uppercase comes before lowercase

Example:
let words = ["banana","Apple","apple"];
words.sort();                     // ["Apple","apple","banana"]
// Reason: 'A' (65) < 'a' (97) in Unicode

------------------------------------------------
NUMERIC COMPARISON USING COMPARE FUNCTION
------------------------------------------------
- compareFn receives two elements a, b
- Returns <0 → a before b
- Returns >0 → b before a
- Returns 0 → order unchanged

Syntax:
arr.sort();                  // default string sort
arr.sort((a,b) => a-b);      // ascending numbers
arr.sort((a,b) => b-a);      // descending numbers

Examples:
["banana","apple","cherry"].sort();   // ["apple","banana","cherry"]
[5,2,9,1].sort((a,b)=>a-b);        // [1,2,5,9] // If a - b is negative → a comes before b

[5,2,9,1].sort((a,b)=>b-a);       // [9,5,2,1]  // If b - a is negative → b comes before a

==================== ARRAY COPY IN JAVASCRIPT ====================

-------------------- ❌ WRONG WAY --------------------

let arr1 = [1,2,3];
let arr2 = arr1;     // NOT a copy (reference)

arr2.push(4);

console.log(arr1);   // [1,2,3,4]

👉 Both variables point to SAME memory


-------------------- ✅ SHALLOW COPY METHODS --------------------

1) Spread operator (MOST USED)
let arr2 = [...arr1];

2) Array.from()
let arr2 = Array.from(arr1);

👉 All create NEW array (top-level copy)


==================== SHALLOW vs DEEP COPY ====================

==================== SHALLOW COPY WITH NESTED ARRAY ====================

-------------------- 🔹 NESTED ARRAY --------------------

let arr1 = [
    [1,2],   // index 0
    [3,4]    // index 1
];

👉 This is a nested array (2D array = array inside array)


-------------------- 🔹 SHALLOW COPY --------------------

let arr2 = [...arr1];

👉 What happens:

✔ Outer array is copied
❌ Inner arrays are NOT copied (shared reference)

Memory view:

arr1 ---> [ A , B ]
arr2 ---> [ A , B ]   (same inner arrays)


-------------------- 🔹 MODIFY VALUE --------------------

arr2[0][0] = 99;

Step-by-step:

arr2[0]   → [1,2]   (first inner array)
[0]       → 1       (first element)

👉 Changing 1 → 99


-------------------- 🔹 RESULT --------------------

console.log(arr1);

Output:
[[99,2], [3,4]]   ❌ original changed


-------------------- 🔹 WHY THIS HAPPENS --------------------

- Shallow copy copies only top level
- Nested arrays/objects are shared
- Both arr1 and arr2 point to same inner arrays

-----------------------------------------------------
Here’s the 1-line fix for your exact case 👇

let arr1 = [[1,2], [3,4]];
let arr2 = structuredClone(arr1);   // ✅ deep copy
🔥 Now test
arr2[0][0] = 99;

console.log(arr1); // [[1,2], [3,4]] ✅ unchanged
🧠 Why this works
structuredClone() copies everything (including nested arrays)
so arr2 has completely separate memory
🎯 One-line

Use structuredClone() to deep copy nested arrays so changes don’t affect original.


-------------------- 🔹 SIMPLE UNDERSTANDING --------------------

- Outer array = copied (new container)
- Inner arrays = same (not copied)


-------------------- 🔹 ANALOGY --------------------

arr1 = big box with 2 small boxes  
arr2 = new big box  

👉 but small boxes inside are SAME  

Change inside → affects both


-------------------- 🔹 ONE-LINE SUMMARY --------------------

Shallow copy copies only outer array; nested arrays remain shared, so changes affect original.


-------------------- 🔹 SHALLOW SAFE CASE --------------------

let arr1 = [1,2,3];
let arr2 = [...arr1];

arr2.push(4);

console.log(arr1);   // [1,2,3] ✅

👉 Works fine because primitives (number, string)


-------------------- 🔹 DEEP COPY --------------------

- Copies EVERYTHING (including nested)
- Fully independent copy

Example:
let arr1 = [[1,2], [3,4]];
let arr2 = structuredClone(arr1);

arr2[0][0] = 99;

console.log(arr1);   // [[1,2],[3,4]] ✅


==================== DEEP COPY METHODS ====================

1) ✅ structuredClone()  (BEST MODERN)

let arr2 = structuredClone(arr1);

✔ Supports:
- objects, arrays
- nested data
- Date, Map, Set

❌ Not supported:
- functions


2) ✅ lodash (production safe)

npm install lodash

const _ = require("lodash");
let arr2 = _.cloneDeep(arr1);

✔ Works for almost all cases


3) ⚠️ JSON method (LIMITED)

let arr2 = JSON.parse(JSON.stringify(arr1));

❌ Problems:
- removes functions
- Date → becomes string
- removes undefined
- fails on circular references


4) ⚠️ Manual deep copy

function deepCopy(obj) {
    if (typeof obj !== "object" || obj === null) return obj;

    let copy = Array.isArray(obj) ? [] : {};

    for (let key in obj) {
        copy[key] = deepCopy(obj[key]);
    }

    return copy;
}

❌ Limitations:
- doesn’t handle Date, Map, Set properly


==================== WHEN TO USE WHAT ====================

Simple array (no nested):
[1,2,3] → use spread [...]

Nested array/object:
[[1,2],[3,4]] → use structuredClone()

Production complex data:
→ use lodash.cloneDeep()


==================== KEY DIFFERENCE ====================

Shallow Copy:
- copies top level only
- nested data shared
- fast

Deep Copy:
- copies full structure
- no shared reference
- safer but heavier


==================== ANALOGY ====================

Shallow → photocopy of cover page
Deep → full book copy


==================== FINAL SUMMARY ====================

- arr2 = arr1 → reference (not copy)
- [...arr1] → shallow copy (most used)
- shallow works for primitives
- deep copy needed for nested data
- best deep copy → structuredClone()

==================== DOUBT: SHALLOW vs DEEP ====================

Q) For non-nested arrays/objects, does shallow copy behave like deep copy?
   And only for nested it becomes shallow?


-------------------- ✅ ANSWER --------------------

Yes (almost correct):

- For non-nested data → shallow behaves like deep (safe)
- For nested data → shallow remains shallow (problem)


==================== WHY THIS HAPPENS ====================

🔹 Case 1: No nesting (primitives only)

let arr1 = [1,2,3];
let arr2 = [...arr1];

arr2[0] = 99;

console.log(arr1);   // [1,2,3] ✅

👉 Reason:
- elements are primitives
- copied by value
- no shared reference


------------------------------------------------------------

🔹 Case 2: Object but no nested object inside

let obj1 = { name: "Ajay" };
let obj2 = { ...obj1 };

obj2.name = "Rahul";

console.log(obj1);   // { name: "Ajay" } ✅

👉 Reason:
- "name" is primitive (string)
- safe copy


------------------------------------------------------------

🔹 Case 3: Nested data (problem)

let arr1 = [{ id: 1 }];
let arr2 = [...arr1];

arr2[0].id = 99;

console.log(arr1);   // [{ id: 99 }] ❌

👉 Reason:
- object inside array → reference copied
- both share same inner object


==================== REAL RULE ====================

Shallow copy:
- copies outer structure
- inner objects/arrays → reference shared

👉 If inner data is primitive → safe
👉 If inner data is object/array → problem


==================== SIMPLE MEMORY VIEW ====================

[1,2,3]      → values copied (safe) ✅
[{id:1}]     → reference copied (shared) ❌


==================== FINAL ONE-LINE ====================

Shallow copy behaves like deep copy only when there is no nested reference type inside.
