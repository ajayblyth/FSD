============================================================================================================================
ARRAY REFERENCES & NESTED ARRAYS (JS)
============================================================================================================================

ARRAY (BASIC)
------------------------------------------------
- Ordered collection of values (any type)
- Access using index (starts from 0)
- Helps in looping + built-in methods (push, pop, map, filter)


============================================================================================================================
PRIMITIVES vs REFERENCE TYPES (CORE IDEA)
============================================================================================================================

PRIMITIVES (Number, String, Boolean, null, undefined, BigInt, Symbol)
------------------------------------------------
- Stored by VALUE
- Copy creates NEW independent value

Example:
let a = 10;
let b = a;

b = 20;

console.log(a);   // 10 ✅ unchanged


REFERENCE TYPES (Arrays, Objects, Functions)
------------------------------------------------
- Stored by REFERENCE (memory address)
- Copy shares SAME memory

Example:
let arr1 = [1,2,3];
let arr2 = arr1;

arr2.push(4);

console.log(arr1);   // [1,2,3,4] ❌ changed


👉 Key Difference:
- Primitive → value copied
- Array/Object → reference copied


============================================================================================================================
ARRAY REFERENCES
============================================================================================================================

- Arrays are reference types (stored in heap)
- Variable stores memory reference, NOT actual data

Example:
let arr1 = [1,2,3];
let arr2 = arr1;

arr2.push(4);

arr1 → [1,2,3,4]
arr2 → [1,2,3,4]

👉 Both point to SAME memory


============================================================================================================================
COMPARISON USING ===
============================================================================================================================

👉 IMPORTANT: Behavior depends on type


-------------------- PRIMITIVES --------------------

- === compares VALUE + TYPE

Example:
10 === 10      // true
"10" === 10    // false


-------------------- ARRAYS / OBJECTS --------------------

- === compares REFERENCE (memory address)

Example:
let a = [1,2];
let b = [1,2];

console.log(a === b);   // false ❌ (different memory)

let c = a;
console.log(a === c);   // true ✅ (same reference)


👉 Final Rule:
- Primitives → compare value
- Objects/Arrays → compare reference


============================================================================================================================
SHALLOW COPY (IMPORTANT LINK)
============================================================================================================================

- Copies only top level
- Nested arrays/objects are shared

Example:
let arr1 = [[1,2], [3,4]];
let arr2 = [...arr1];

arr2[0][0] = 99;

console.log(arr1);   // [[99,2],[3,4]] ❌ changed

👉 Inner arrays share same reference


============================================================================================================================
CONSTANT ARRAYS
============================================================================================================================

const arr = [1,2,3];

arr.push(4);   // ✔ allowed
arr.pop();     // ✔ allowed

❌ arr = [5,6];   // not allowed

👉 const locks reference, NOT values


============================================================================================================================
SIMPLE ANALOGY
============================================================================================================================

Primitive:
- Copy a number on paper → separate

Array/Object:
- Copy address of a house → same place

Shallow copy:
- New box, same items inside


============================================================================================================================
FINAL SUMMARY
============================================================================================================================

- Primitives → stored by value, copied independently
- Arrays/Objects → stored by reference, shared memory
- ===:
    → primitives → value + type
    → objects → reference
- Shallow copy → only outer copy, inner shared
- const → reference fixed, values can change

============================================================================================================================
NESTED ARRAYS
------------------------------------------------
Definition:
- Array inside another array (2D structure)

Syntax:
let arr = [[1,2],[3,4],[5,6]];

Access:
arr[row][column]

Example:
let matrix = [[1,2,3],
              [4,5,6],
              [7,8,9]];

console.log(matrix[0]);     // [1,2,3]
console.log(matrix[1][2]);  // 6

Use Cases:
- 2D grids, tables, matrix problems
- Outer array → rows
- Inner array → columns
- Indexing starts from 0

Example Tic-Tac-Toe Board:
let arr = [["x", null, "o"],
           [null,"x",null],
           ["o",null,"x"]];

LOOPS IN JAVASCRIPT
------------------------------------------------
Purpose: Loops are used to execute a block of code repeatedly until a specified condition becomes false.

1) for loop (known iterations) – use when number of iterations is known
   for(initialization; condition; update) {
       // code
   }
   Example:
   for(let i = 0; i < 3; i++) {
       console.log(i);
   }
   // e.g., looping 10 times to print first 10 numbers

2) while loop (condition-based) – use when number of iterations is unknown
   let i = 0;
   while(i < 3) {
       console.log(i);
       i++;
   }
   // e.g., reading user input until they type "exit"

3) do...while (executes at least once) – use when code must run at least once
   let i = 0;
   do {
       console.log(i);
       i++;
   } while(i < 3);
   // e.g., showing a menu at least once before user chooses to quit

4) for...of (iterate values) – use when you want to directly access iterable values
for...of → used to iterate over values in arrays, strings, or other iterables.

   - ex.1) for(let val of [1, 2, 3]) 
   {
       console.log(val);
   }
   - ex.2) let str = "Hello";


for (let ch of str) {
    console.log(ch);

   // e.g., printing all items in an array of fruits

5) for...in (iterate keys / indices) – use when you want to iterate over object keys/indices
for...in → used to iterate over keys/properties in objects (or array indices).
   for(let key in {a:1, b:2}) {
       console.log(key);
   }

  ex.2) let arr = [10, 20, 30];
for (let index in arr) {
    console.log(index);
}
// Output:
// 0
// 1
// 2

ex.3) To get values using the index:
for (let index in arr) {
    console.log(arr[index]);
}
// Output:
// 10
// 20
// 30
   // e.g., checking all properties in a user object

Loop Controls:
- break → stops loop
- continue → skips current iteration
