Topic: Commit Timeline (Git)
==============================

Commit Timeline
---------------
- Commit timeline is the chronological history of commits in a Git repository.
- It shows how the project changes over time.
- Every commit represents a snapshot of the project at that moment.
- New commits are always added on top of the timeline.

HEAD in Commit Timeline
-----------------------
- HEAD is a pointer to the current commit you are working on.
- It usually points to the latest commit of the current branch.
- When you make a new commit, HEAD automatically moves forward.
- Switching branches moves HEAD to a different commit.

Viewing Commit Timeline
-----------------------
git log
-> Shows detailed commit history (author, date, message).

git log --oneline
-> Shows compact timeline (commit id + message).

git log --oneline --graph
-> Shows visual timeline with branches and merges.

Example Commit Timeline
-----------------------
* f3a1b2 (HEAD -> main) Add payment feature
* 9c8d7e Fix login bug
* 4b5a6c Add login page
* 1a2b3c Initial commit

Why Commit Timeline is Important
--------------------------------
- Helps understand project history.
- Allows going back to previous versions.
- Useful for debugging and fixing bugs.
- Shows who changed what and when.
- Essential for teamwork and collaboration.

What git restore does
---------------------
- Used to undo changes in files.
- Restores files to a previous state.
- Works in Working Directory and Staging Area.
- Does NOT delete commits.

Restore from Working Directory
------------------------------
git restore <file>
-> Discards local changes in working directory.
-> File goes back to last committed state.

Restore from Staging Area
-------------------------
git restore --staged <file>
-> Removes file from staging area.
-> File remains changed in working directory.

Restore both staged and unstaged
--------------------------------
git restore --staged <file>
git restore <file>

Restore all files
-----------------
git restore .
-> Discards all uncommitted changes.

Important Notes
---------------
- Cannot restore deleted commits.
- Use carefully, changes are lost.
- Alternative older command: git checkout <file>
