EJS (EMBEDDED JAVASCRIPT TEMPLATING)
------------------------------------------------------------

DEFINITION
------------------------------------------------------------
• EJS is a templating engine for Node.js (Express)
• Used to generate dynamic HTML pages
• Allows embedding JavaScript inside HTML

• Template → A replaceable part of HTML that can be
  dynamically changed based on data/content ✔

------------------------------------------------------------

WHY USE EJS?
------------------------------------------------------------
• To send dynamic data from backend to frontend ✔
• Avoid writing static HTML only
• Useful for rendering views (UI pages)

------------------------------------------------------------

SETUP
------------------------------------------------------------
npm install ejs

In app.js / index.js:
app.set('view engine', 'ejs');

------------------------------------------------------------

HOW IT WORKS
------------------------------------------------------------
1. Client sends request
2. Server processes data
3. EJS file renders HTML with data
4. Response sent as HTML page

------------------------------------------------------------

SYNTAX
------------------------------------------------------------
<%= %>  → Output value
<% %>   → Logic (if, loop)

------------------------------------------------------------

EXAMPLE
------------------------------------------------------------

// index.js
app.get('/user', (req, res) => {
    res.render('user', { name: "Ajay" });
});

// views/user.ejs
<h1>Hello <%= name %></h1>

------------------------------------------------------------

OUTPUT
------------------------------------------------------------
Hello Ajay

------------------------------------------------------------

FEATURES
------------------------------------------------------------
• Dynamic content rendering ✔
• Supports loops and conditions ✔
• Easy integration with Express ✔

res.render() (EXPRESS)
------------------------------------------------------------

DEFINITION
------------------------------------------------------------
• res.render() is used to render a template (like EJS)
  and send the final HTML to the browser ✔

------------------------------------------------------------

WHAT IT DOES
------------------------------------------------------------
1. Finds template file (from views folder)
2. Passes data to it
3. EJS processes template (HTML + JS)
4. Generates final HTML
5. Sends HTML as response ✔

------------------------------------------------------------

SYNTAX
------------------------------------------------------------
res.render('fileName', dataObject);

Example:
res.render('home', { name: "Ajay" });

------------------------------------------------------------

FLOW
------------------------------------------------------------
Backend → res.render()
EJS → process template
HTML → sent to browser ✔

------------------------------------------------------------

IMPORTANT
------------------------------------------------------------
• No need to write .ejs extension
• Works only if view engine is set

------------------------------------------------------------

SUMMARY
------------------------------------------------------------
res.render() → template + data → HTML response ✔
------------------------------------------------------------

SUMMARY
------------------------------------------------------------
EJS = HTML + JavaScript ✔
Used to create dynamic web pages ✔

------------------------------------------------------------

EJS TEMPLATE TAGS (ORGANIZED)
------------------------------------------------------------

TAG        → DESCRIPTION
------------------------------------------------------------
EJS TAGS (DETAILED ONE-LINERS)
------------------------------------------------------------

EJS Tags
<%=: Outputs the value into the template (HTML escaped)

<%: 'Scriptlet' tag, for control-flow, no output

<%_: 'Whitespace Slurping' Scriptlet tag, strips all whitespace before it

<%-: Outputs the unescaped value into the template

<%#: Comment tag, no execution, no output

<%%: Outputs a literal <%

%>: Plain ending tag

-%>: Trim-mode ('newline slurp') tag, trims following newline

_%>: 'Whitespace Slurping' ending tag, removes all whitespace after it

Quick Breakdown of the Essentials:
If you are just starting with EJS, these are the three you'll use 90% of the time:

<% ... %>: Use this for logic like if statements or for loops.

<%= ... %>: Use this to print a variable (like a username or title) safely.

<%- ... %>: Use this only when you want to render actual HTML code stored in a variable (like a partial or a blog post body).

SUMMARY
------------------------------------------------------------
<% %>   → logic
<%= %>  → print value
<%- %>  → print HTML ✔

SERVING STATIC FILES (PUBLIC FOLDER)
=====================================================================

DEFINITION
---------------------------------------------------------------------
• Public folder stores static (constant) files ✔
• These files do NOT change dynamically

---------------------------------------------------------------------

WHAT GOES INSIDE?
---------------------------------------------------------------------
• CSS files → style.css
• JS files  → script.js
• Images    → logo.png, bg.jpg
• Fonts     → custom fonts

---------------------------------------------------------------------

WHY PUBLIC FOLDER?
---------------------------------------------------------------------
• Serve static files directly to browser ✔
• No need to create routes for each file
• Keeps project clean and organized

=====================================================================

SETUP IN EXPRESS
---------------------------------------------------------------------
app.use(express.static('public'));  

// Better (absolute path):
app.use(express.static(path.join(__dirname, 'public')));

• Makes "public" folder available to frontend ✔

=====================================================================

HOW IT WORKS (FLOW)
---------------------------------------------------------------------
1. User visits route → /ig/cats
2. Server sends HTML (EJS → rendered HTML)
3. Browser reads HTML (<link>, <img>, etc.)
4. Browser requests static files:

   GET /style.css   → served from public/style.css ✔
   GET /image.jpg   → served from public/image.jpg ✔

=====================================================================

PUBLIC FOLDER MAPPING
---------------------------------------------------------------------
• Express treats "public" as root (/)

Mapping:
public/style.css   → /style.css
public/logo.png    → /logo.png
public/js/app.js   → /js/app.js

=====================================================================

EXAMPLE
---------------------------------------------------------------------
public/style.css

Use in HTML:
<link rel="stylesheet" href="/style.css">

=====================================================================

RELATIVE URL TRAP ⚠️
---------------------------------------------------------------------
❌ Wrong:
<link rel="stylesheet" href="style.css">

If current URL:
http://localhost:3000/ig/cats

Browser requests:
http://localhost:3000/ig/style.css ❌

---------------------------------------------------------------------

✔ Correct:
<link rel="stylesheet" href="/style.css">

→ Always starts from root ✔

=====================================================================

SUMMARY
---------------------------------------------------------------------
• Public folder = static files ✔
• express.static() serves them ✔
• Use "/" in paths to avoid errors ✔
