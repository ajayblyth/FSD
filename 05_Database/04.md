====================================================================================================
MYSQL BETWEEN CLAUSE
====================================================================================================

Definition :
BETWEEN → Filters values within a range (inclusive)

SYNTAX:
SELECT column FROM table
WHERE column BETWEEN value1 AND value2;

EQUIVALENT:
----------------------------------------------------------------------------------------------------
WHERE age BETWEEN 18 AND 22
= WHERE age >= 18 AND age <= 22

----------------------------------------------------------------------------------------------------
KEY POINT
----------------------------------------------------------------------------------------------------
• Inclusive → includes both limits (18 & 22)

----------------------------------------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------------------------------------

-- Numeric
SELECT * FROM student WHERE age BETWEEN 18 AND 22;

-- Date
SELECT * FROM orders
WHERE order_date BETWEEN '2024-02-01' AND '2024-03-01';

-- String (dictionary order)
SELECT * FROM student
WHERE name BETWEEN 'A' AND 'M';

-- NOT BETWEEN
SELECT * FROM student
WHERE age NOT BETWEEN 18 AND 22;
= age < 18 OR age > 22

----------------------------------------------------------------------------------------------------
IMPORTANT CASES
----------------------------------------------------------------------------------------------------
• Order matters :
  BETWEEN 22 AND 18 → EMPTY

• NULL values :
  ignored in results

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• BETWEEN = >= AND <=
• Works with numbers, dates, strings
• Use for clean range queries

====================================================================================================
MYSQL LIKE / NOT LIKE
====================================================================================================

Definition :
LIKE → Pattern matching (strings)

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
SELECT column FROM table
WHERE column LIKE pattern;

----------------------------------------------------------------------------------------------------
WILDCARDS
----------------------------------------------------------------------------------------------------
% → any characters (0+)
_ → exactly one character

----------------------------------------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------------------------------------

-- Starts with
SELECT * FROM student WHERE name LIKE 'A%';

-- Ends with
SELECT * FROM student WHERE name LIKE '%a';

-- Contains
SELECT * FROM student WHERE name LIKE '%aj%';

-- Middle pattern
SELECT * FROM student WHERE name LIKE 'r%n'; //rohan

-- Surname / suffix
SELECT * FROM student WHERE name LIKE '% sharma';

-- Single character
SELECT * FROM student WHERE name LIKE '_a%';

-- Exact length (4 chars)
SELECT * FROM student WHERE name LIKE '____';

-- Multiple conditions
SELECT * FROM student
WHERE name LIKE 'A%' AND city LIKE '%i';

-- NOT LIKE
SELECT * FROM student WHERE name NOT LIKE 'A%';


Note:IF NOTHING MATCHES- Query returns EMPTY RESULT SET
----------------------------------------------------------------------------------------------------
CASE SENSITIVITY
----------------------------------------------------------------------------------------------------
• Depends on collation (usually case-insensitive)

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Used only for strings
• Slower than '=' (may skip index)
• Used in search/filter features

====================================================================================================
MYSQL "IN" CLAUSE
====================================================================================================

Definition :
IN → Matches any value in a list (shortcut for OR)

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
SELECT column FROM table
WHERE column IN (v1, v2, v3);

----------------------------------------------------------------------------------------------------
EQUIVALENT
----------------------------------------------------------------------------------------------------
city IN ('Delhi','Mumbai')
= city='Delhi' OR city='Mumbai'

----------------------------------------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------------------------------------
-- String
SELECT * FROM student WHERE name IN ('Ajay','Rahul');

-- Numeric
SELECT * FROM student WHERE age IN (18,20,22,NULL);
• NULL inside IN list is ignored in comparison


-- NOT IN
SELECT * FROM student WHERE city NOT IN ('Delhi','Mumbai');

-- Subquery
SELECT * FROM student
WHERE id IN (SELECT student_id FROM orders);

----------------------------------------------------------------------------------------------------
IMPORTANT CASE (NULL)
----------------------------------------------------------------------------------------------------
SELECT * FROM student WHERE age NOT IN (18,20,NULL);
→ • Returns NO ROWS ❌
EMPTY (unexpected)

Reason :
NULL → comparison becomes UNKNOWN

✔ Avoid NULL in IN list

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• IN = multiple OR
• Improves readability
• Use carefully with NULL

====================================================================================================
MYSQL IS NULL
====================================================================================================

Definition :
IS NULL → Checks missing values

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
SELECT * FROM table WHERE column IS NULL;
SELECT * FROM table WHERE column IS NOT NULL;

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
SELECT * FROM student WHERE age IS NULL;

----------------------------------------------------------------------------------------------------
COMMON MISTAKE
----------------------------------------------------------------------------------------------------
WRONG  : age = NULL / age != NULL
RIGHT  : age IS NULL / IS NOT NULL

----------------------------------------------------------------------------------------------------
WHY
----------------------------------------------------------------------------------------------------
• NULL = unknown → cannot compare using =

----------------------------------------------------------------------------------------------------
COMBINED CONDITIONS
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE age IS NULL OR city = 'Delhi';

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• NULL ≠ 0 / empty string
• Ignored in comparisons
• Very important in real data

====================================================================================================
MYSQL ALIAS (AS)
====================================================================================================

Definition :
Alias → Temporary name for column/table

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
SELECT column AS alias FROM table;

----------------------------------------------------------------------------------------------------
COLUMN ALIAS
----------------------------------------------------------------------------------------------------
SELECT name AS student_name FROM student;

-- Without AS (valid)
SELECT name student_name FROM student;

-- Multiple
SELECT name AS n, age AS a FROM student;

-- Expression
SELECT name, age + 10 AS future_age FROM student;

----------------------------------------------------------------------------------------------------
TABLE ALIAS
----------------------------------------------------------------------------------------------------
SELECT s.name, s.age
FROM student AS s;

----------------------------------------------------------------------------------------------------
JOIN EXAMPLE
----------------------------------------------------------------------------------------------------
SELECT s.name, o.order_date
FROM student AS s
JOIN orders AS o
ON s.id = o.student_id;

----------------------------------------------------------------------------------------------------
RULES
----------------------------------------------------------------------------------------------------
• Exists only during query
• Cannot use in WHERE (same level)
• Can use in ORDER BY

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
SELECT name AS n
FROM student
ORDER BY n;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Improves readability
• Useful in joins & calculations
• Avoid spaces (or use quotes)
• Temporary (not stored)

====================================================================================================
MYSQL STRING FUNCTIONS
====================================================================================================

Definition :
String functions → Used to manipulate/process text data

Concept :
char  → character (A, B)
byte  → storage unit (memory)

====================================================================================================
CORE FUNCTIONS
====================================================================================================

1) LENGTH()
----------------------------------------------------------------------------------------------------
Returns length in BYTES

SELECT LENGTH('Ajay');         → 4
SELECT LENGTH('🙂');           → depends on bytes (multi-byte)

SELECT name, LENGTH(name) FROM users;

-- Result:
-- name   | LENGTH(name)
-- Ajay   | 4
-- Rahul  | 5
-- S      | 1

Use case : validation (password, phone length)

SELECT * FROM users
WHERE LENGTH(phone) = 10;


Can we pass multiple columns to LENGTH()?
→ NO ❌
→ LENGTH() accepts only ONE value at a time

---------------------------------------------------------------
WRONG
---------------------------------------------------------------

SELECT LENGTH(name, city) FROM users;   ❌ Not allowed
----------------------------------------------------------------------------------------------------
2) CHAR_LENGTH()
----------------------------------------------------------------------------------------------------
Returns number of characters

SELECT CHAR_LENGTH('Ajay');    → 4

----------------------------------------------------------------------------------------------------
3) UPPER() / LOWER()
----------------------------------------------------------------------------------------------------
SELECT UPPER('ajay');          → 'AJAY'
SELECT LOWER('AJAY');          → 'ajay'

-- Convert column values to uppercase
SELECT name, UPPER(name) 
FROM users;

-- Update only specific row
UPDATE users
SET name = UPPER(name)
WHERE id = 1;

--only first letter to upper
SELECT 
  CONCAT(UPPER(LEFT(name, 1)), LOWER(SUBSTRING(name, 2))) AS formatted_name
FROM your_table; //temporary

--for permanent
UPDATE your_table
SET name = CONCAT(UPPER(LEFT(name, 1)), LOWER(SUBSTRING(name, 2)));
----------------------------------------------------------------------------------------------------
4) CONCAT()
----------------------------------------------------------------------------------------------------
Joins strings

SELECT CONCAT('Ajay',' ','Sharma');     → 'Ajay Sharma'
SELECT CONCAT('Age: ', 25);             → 'Age: 25'

-- Combine two columns
SELECT CONCAT(first_name, ' ', last_name)
FROM users;

⚠ NULL case :
CONCAT('Ajay', NULL, 'Sharma') → NULL

----------------------------------------------------------------------------------------------------
5) CONCAT_WS()  (WITH SEPARATOR), handles null
----------------------------------------------------------------------------------------------------
SELECT CONCAT_WS('-', '2024','03','20');   → '2024-03-20'

CONCAT_WS('_','Ajay',NULL,'Sharma') → 'Ajay_Sharma'

• Automatically ignores NULL values
• Cleaner than IFNULL()
• Adds separator only when needed

----------------------------------------------------------------------------------------------------
6) SUBSTRING() / SUBSTR()
----------------------------------------------------------------------------------------------------
Extract part of string

SELECT SUBSTRING('AjaySharma',1,4); //implicit   → 'Ajay'
SELECT SUBSTRING('AjaySharma',5);   //implicit  → 'Sharma'

-- Extract part of column value

SELECT name, SUBSTRING(name, 1, 4)
FROM users;

Note: Index starts from 1 (not 0)

----------------------------------------------------------------------------------------------------
7) LEFT() / RIGHT()
----------------------------------------------------------------------------------------------------
Definition:
• LEFT(str, n)  → returns first n characters from the left
• RIGHT(str, n) → returns last n characters from the right

SELECT LEFT('AjaySharma',4);   → 'Ajay'
SELECT RIGHT('AjaySharma',6);  → 'Sharma'

Use cases :
• File extension →SELECT 
  RIGHT(file_name, 3) AS file_extension
FROM your_table;

• Masking → SELECT 
  CONCAT('******', RIGHT(phone_number, 4)) AS masked_number
FROM your_table;

==================== LEFT() vs SUBSTRING() ====================

Feature        | LEFT()                     | SUBSTRING()
---------------|----------------------------|------------------------------
Definition     | First N chars from left    | Extract part of string
Flexibility    | Only from start            | From any position
Syntax         | LEFT(str, n)               | SUBSTRING(str, start, len)

8) TRIM() 👉 Removes leading (start) and trailing (end) spaces.
----------------------------------------------------------------------------------------------------
SELECT TRIM('  Ajay  ');       → 'Ajay'

----------------------------------------------------------------------------------------------------
9) LTRIM() / RTRIM()
----------------------------------------------------------------------------------------------------
SELECT LTRIM('  Ajay');        → 'Ajay'
SELECT RTRIM('Ajay  ');        → 'Ajay'

----------------------------------------------------------------------------------------------------
10) REPLACE()
----------------------------------------------------------------------------------------------------
Replace substring
Syntax:
REPLACE(str, old_substring, new_substring)

• Replaces all occurrences of old_substring with new_substring

SELECT REPLACE('Hello World','World','Ajay');   → 'Hello Ajay'
SELECT REPLACE('app le apple','apple','mango');  → 'mango mango'

SELECT name, REPLACE(name, 'a', '@')
FROM users;

• Case-sensitive
• If not found → original string returned

====================================================================================================
SEARCH FUNCTIONS
====================================================================================================

11) INSTR()
----------------------------------------------------------------------------------------------------
Returns position of FIRST occurrence

SYNTAX :
INSTR(string, substring)

SELECT INSTR('Hello Ajay','Ajay');   → 7

----------------------------------------------------------------------------------------------------
KEY POINTS
----------------------------------------------------------------------------------------------------
• Index starts from 1
• Not found → 0
• First occurrence only
• Empty string → returns 1
• Case depends on collation

----------------------------------------------------------------------------------------------------
USAGE
----------------------------------------------------------------------------------------------------
SELECT * FROM student
WHERE INSTR(name,'aj') > 0;

----------------------------------------------------------------------------------------------------
12) LOCATE()
----------------------------------------------------------------------------------------------------
Similar to INSTR but more flexible

SYNTAX :
LOCATE(substring, string)
LOCATE(substring, string, start_pos)

SELECT LOCATE('Ajay','Hello Ajay');     → 7
SELECT LOCATE('Ajay','Ajay Ajay',2);    → 6

----------------------------------------------------------------------------------------------------
KEY DIFFERENCE
----------------------------------------------------------------------------------------------------
INSTR(string, substring)
LOCATE(substring, string [, start])

✔ LOCATE supports start position

----------------------------------------------------------------------------------------------------
COMMON USE CASE (EMAIL)
----------------------------------------------------------------------------------------------------
LOCATE('@', email) → position of '@'

Username extraction :
SELECT SUBSTRING(email,1,LOCATE('@',email)-1) FROM customers;

----------------------------------------------------------------------------------------------------
EDGE CASE
----------------------------------------------------------------------------------------------------
If '@' not present → LOCATE = 0 → result invalid

✔ Safe version :
CASE WHEN LOCATE('@',email)>0
THEN SUBSTRING(email,1,LOCATE('@',email)-1)
END

----------------------------------------------------------------------------------------------------
MEMORY TRICK
----------------------------------------------------------------------------------------------------
LOCATE → position
-1     → length before it

----------------------------------------------------------------------------------------------------

14) LPAD() / RPAD()
----------------------------------------------------------------------------------------------------

SELECT RPAD('Ajay',6,'*');   → 'Ajay**'

----------------------------------------------------------------------------------------------------
15) FORMAT()
----------------------------------------------------------------------------------------------------
Definition:
• FORMAT(number, decimal_places)
• Formats a number with commas (thousands separator) and rounds to given decimals
• Returns result as STRING

---------------------------------------------------------------
EXAMPLES
---------------------------------------------------------------

SELECT FORMAT(12345.678, 2);   → '12,345.68'

---------------------------------------------------------------
BREAKDOWN
---------------------------------------------------------------

• 12345.678 → original number
• ,         → comma added for thousands → 12,345
• 2         → round to 2 decimal places → .68
• Final     → '12,345.68'

----------------------------------------------------------------------------------------------------

17) CHAR()
