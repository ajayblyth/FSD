1) FIZZBUZZ PROBLEM
---------------------------------------------------------------------
Rule:
- Multiple of 3  → "Fizz"
- Multiple of 5  → "Buzz"
- Multiple of 3 & 5 → "FizzBuzz"

Code:
let number = 10;

if (number % 3 == 0 && number % 5 == 0) {
    console.log("FizzBuzz");
}
else if (number % 3 == 0) {
    console.log("Fizz");
}
else if (number % 5 == 0) {
    console.log("Buzz");
}

Explanation:
Checks divisibility using modulus (%).
If divisible by both 3 and 5 → FizzBuzz.
Otherwise checks individually.

2) STRINGS ARE IMMUTABLE
---------------------------------------------------------------------
Code:
let str = "hello";
str[0] = "H";
console.log(str);

Output:
hello

Explanation:
Strings in JavaScript are immutable.
Characters cannot be changed using index.
Index access works only for reading.



=====================================================================


3) MODIFYING STRING USING METHOD
---------------------------------------------------------------------
Code:
let str = "hello";
str = str.toUpperCase();
console.log(str);

Output:
HELLO

Explanation:
Strings cannot be modified directly,
but methods return a NEW string.
So we store the new value back in the variable.

4) STRING COPY BEHAVIOR
---------------------------------------------------------------------
Code:
let str = "hello";
let newStr = str;

newStr = "world";

console.log(str);
console.log(newStr);

Output:
hello
world

Explanation:
Primitive values (like strings) are copied by VALUE.
Changing newStr does not affect the original variable.



=====================================================================


5) ARRAY REFERENCE BEHAVIOR
---------------------------------------------------------------------
Code:
let a = [1,2,3];
let b = a;

b.push(4);

console.log(a);

Output:
[1,2,3,4]

Explanation:
Arrays are objects (reference types).
Variable b stores the same memory reference as a.
So modifying b also changes a.



=====================================================================


6) CONST ARRAY MODIFICATION
---------------------------------------------------------------------
Code:
const arr = [1,2,3];
arr.push(4);
console.log(arr);

Output:
[1,2,3,4]

Explanation:
const prevents reassignment of the variable,
but the array itself can still be modified.



=====================================================================


7) CONST ARRAY REASSIGNMENT ERROR
---------------------------------------------------------------------
Code:
const arr = [1,2,3];
arr = [4,5,6];

Output:
Error

Explanation:
const variables cannot be reassigned to a new value.
Only mutation of existing object/array is allowed.

8) ARRAY + NUMBER
---------------------------------------------------------------------
Code:
let arr = [1,2,3];
console.log(arr + 1);

Output:
1,2,31

Explanation:
Array converts to string "1,2,3".
Then string concatenation happens:
"1,2,3" + "1" → "1,2,31".



=====================================================================


9) ARRAY + ARRAY
---------------------------------------------------------------------
Code:
console.log([1,2] + [3,4]);

Output:
1,23,4

Explanation:
Both arrays convert to strings:
"1,2" + "3,4"
Result → "1,23,4"



=====================================================================


10) CHANGING ARRAY LENGTH
---------------------------------------------------------------------
Code:
let arr = [1,2,3];

arr.length = 1;

console.log(arr);

Output:
[1]

Explanation:
Reducing length truncates the array.
All elements after index 0 are removed.

11) REASSIGNING ARRAY VARIABLE
---------------------------------------------------------------------
Code:
let a = [1,2];
let b = a;

a = [5,6];

console.log(b);

Output:
[1,2]

Explanation:
Initially both variables refer to same array.
But when a is reassigned to a new array,
b still refers to the old array.
