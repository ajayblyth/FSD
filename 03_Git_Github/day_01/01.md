========================================================
Topic: Problems Git Solves
========================================================

+------------------------+-----------------------------------------------+
| Problem                | How Git Helps                                  |
+------------------------+-----------------------------------------------+
| manage code            | Tracks every change with timestamps           |
| collaboration          | Multiple devs work safely together             |
| system crashes         | Commits act as backup                          |
| online copy            | Code stored locally + GitHub                  |
| code sync              | git push / git pull keeps code in sync        |
| version tracking       | Every version is saved                        |
| go back in time        | Revert / checkout old commits                 |
| git history            | Record of every change                        |
| new member joining     | Repo link gives full project history          |
| branching & merging    | Safe feature experimentation                  |
| conflict management    | Detects and resolves overlaps                 |
| accountability         | Shows who changed what and when               |
| code integrity         | Prevents accidental loss                      |
| deployment readiness   | Separate dev and prod branches                |
+------------------------+-----------------------------------------------+

--------------------------------------------------------
Definition
--------------------------------------------------------
Git is a free and open source distributed version control system,
designed to handle everything from small to very large projects
with speed and efficiency.

========================================================
Types of Version Control Systems
========================================================

+---------------------------+---------------------------------------------+
| Type                      | Explanation                                 |
+---------------------------+---------------------------------------------+
| Local Version Control     | Multiple copies, one system, unsafe         |
| Central Version Control   | One central server, online history only     |
| Distributed (Git)         | Full history locally + online               |
+---------------------------+---------------------------------------------+

| Purpose                   | Command                                     |
+---------------------------+---------------------------------------------+

========================================================
Git Lifecycle
========================================================

+-------------------+--------------------------------------------+
| Stage             | Description                                |
+-------------------+--------------------------------------------+
| Working Directory | Editable local files                       |
| Staging Area      | Files selected for commit (git add)        |
| Local Repository  | Saved commits (git commit)                 |
| Remote Repository | Online repo (git push / pull)              |
+-------------------+--------------------------------------------+

Flow:
[WD] -> git add -> [Staging] -> git commit -> [Local Repo] -> git push -> [Remote Repo]

========================================================
Git vs GitHub
========================================================

+-----------+----------------------------------------------+
| Git       | GitHub                                       |
+-----------+----------------------------------------------+
| Tool      | Platform                                    |
| Local     | Cloud-based                                 |
| Tracks code | Hosts repositories                        |
| No UI     | Collaboration features                      |
+-----------+----------------------------------------------+

========================================================
Basic Git Commands
========================================================

+-------------------------------+-----------------------------------------+
| Command                       | Purpose                                 |
+-------------------------------+-----------------------------------------+
| git init                      | Start tracking folder                   |
| git status                    | Check file state                        |
| git add <file>                | Stage specific file                     |
| git add .                     | Stage all files                         |
| git commit -m "msg"           | Save snapshot                           |
| git log                       | View commit history                    |
| git branch                    | List branches                           |
| git remote add origin URL     | Connect local to remote                |
| git push -u origin master     | Push code to GitHub                    |
+-------------------------------+-----------------------------------------+
