==============================
Branches in Git
==============================

What is a Branch
----------------
- A branch is a separate line of development.
- Allows working on features without affecting main code.
- Branches point to commits.

Main (master) Branch
--------------------
- Production-level branch.
- Contains stable, tested code.
- Code ready for deployment.
- Direct changes should be minimal.
- Other branches are merged into main after testing.

Feature Branch
--------------
- Used to develop new features.
- Created from main branch.
- Keeps experimental work isolated.
- Safe place to write, test, and modify code.
- Merged into main after completion.

Common Feature Branch Flow
--------------------------
git branch feature-login
git checkout feature-login
-> work on feature
git commit
git checkout main
git merge feature-login

Advantages of Feature Branch
----------------------------
- No risk to production code.
- Easy collaboration.
- Clean commit history.
- Parallel development possible.
--------------------------------------------

==============================
Git Branch Commands
==============================

git branch
----------
- Shows all branches in the repository.
- Current branch is marked with * (asterisk).

git branch <branch_name>
------------------------
- Creates a new branch.
- Does NOT switch to it automatically.
Example:
git branch random_name

git checkout <branch_name>
--------------------------
- Switches to the specified branch.
Example:
git checkout random_name

(Modern shortcut)
git switch <branch_name>
- Cleaner alternative to checkout (new Git versions).

------------------------------------------------

git log --oneline
-----------------
- Shows commit history in compact form.
- Each line shows:
  <commit_hash> <commit_message>
- HEAD points to the current branch’s latest commit.

------------------------------------------------

Deleting Branches
-----------------

git branch -d <branch_name>
---------------------------
- Soft delete (safe delete).
- Deletes branch ONLY if it is already merged.
- Prevents accidental data loss.

git branch -D <branch_name>
---------------------------
- Force delete locally (even if not merged).

git push origin --delete <branch_name>
--------------------------------------
- Permanently deletes branch from remote (GitHub).
- Cannot be recovered easily.

------------------------------------------------

Branch Naming Convention
------------------------
- Not mandatory, but best practice.
- Improves readability and teamwork.

Common patterns:
feature-login
feature-user-auth
fix-bug_name
hotfix-payment
release-v1.0

------------------------------------------------

Main vs Feature Branch
---------------------

main branch
-----------
- Production-ready branch.
- Stable and deployable code.
- Merges happen AFTER testing.

feature branch
--------------
- Used to develop new features or fixes.
- Created from main.
- Merged back into main after completion.

------------------------------------------------

Merging Feature Branch into Main
--------------------------------

Step 1: Switch to main branch
git checkout main

Step 2: Merge feature branch
git merge feature-branch_name

- Combines feature code into main.
- May ask to resolve conflicts if changes overlap.

------------------------------------------------

Pushing Branch to Remote
------------------------

git push origin <branch_name>
-----------------------------
- Pushes the branch to GitHub.
- Creates the same branch in remote repository.
- Does NOT push changes into main automatically.
- Pull request or merge is required to update main.
