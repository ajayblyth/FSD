String functions:
MYSQL UPPER() / LOWER() (DETAILED)
======================================================================

DEFINITION:
UPPER() and LOWER() are used to convert string case.

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
UPPER(string)
LOWER(string)

----------------------------------------------------------------------
BASIC EXAMPLES
----------------------------------------------------------------------
SELECT UPPER('ajay');
→ 'AJAY'

SELECT LOWER('AJAY');
→ 'ajay'

----------------------------------------------------------------------
WORKING WITH TABLE DATA
----------------------------------------------------------------------
SELECT UPPER(name) FROM student;
→ Converts all names to uppercase

SELECT LOWER(email) FROM customers;
→ Converts all emails to lowercase

----------------------------------------------------------------------
USE IN WHERE CLAUSE (IMPORTANT)
----------------------------------------------------------------------
SELECT * FROM student
WHERE UPPER(name) = 'AJAY';

→ Matches 'ajay', 'Ajay', 'AJAY' (case-insensitive search)

----------------------------------------------------------------------
WHY USED
----------------------------------------------------------------------
• Normalize data for comparison
• Handle case-insensitive search
• Maintain consistent format

----------------------------------------------------------------------
REAL EXAMPLE
----------------------------------------------------------------------
SELECT * FROM users
WHERE LOWER(email) = 'ajay@gmail.com';

→ Matches:
   'AJAY@GMAIL.COM'
   'Ajay@gmail.com'

 --  INSERT INTO users (age, email)
VALUES (20,LOWER('AJAY@GMAIL.COM'));

→ Stored value: 'ajay@gmail.com'

----------------------------------------------------------------------
IMPORTANT POINTS
----------------------------------------------------------------------
1) Does NOT change actual data (only output)
2) Works only on strings
3) Useful in filtering + comparison
4) Can be combined with other functions

----------------------------------------------------------------------
PERFORMANCE NOTE
----------------------------------------------------------------------
SELECT * FROM student
WHERE UPPER(name) = 'AJAY';

→ May NOT use index (slower on large data)

Better:
Store data in consistent case OR use proper collation

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) UPPER → uppercase
2) LOWER → lowercase
3) Common in real-world queries
4) Helps avoid case mismatch issues

MYSQL TRIM() FUNCTION
======================================================================

DEFINITION:
Removes leading and trailing characters (default: spaces)

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
TRIM(string)
TRIM(character FROM string)

BASIC EXAMPLE
SELECT TRIM('  Ajay  ');
→ 'Ajay'

REMOVE SPECIFIC CHARACTER
SELECT TRIM('x' FROM 'xxxAjayxxx');
→ 'Ajay'


LEADING / TRAILING ONLY
----------------------------------------------------------------------
Leading:Removes characters ONLY from start (left side).
Trailing:Removes characters ONLY from end (right side)


TRIM(LEADING 'x' FROM 'xxxAjay')   → 'Ajay'
TRIM(TRAILING 'x' FROM 'Ajayxxx')  → 'Ajay'

----------------------------------------------------------------------
USE IN TABLE
----------------------------------------------------------------------
SELECT TRIM(name) FROM student;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Removes spaces by default
2) Does NOT affect middle spaces
3) Works on output (not stored data)
4) Useful for cleaning input data

MYSQL REVERSE() FUNCTION
======================================================================

DEFINITION:
REVERSE() reverses the given string.

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
REVERSE(string)

----------------------------------------------------------------------
BASIC EXAMPLE
----------------------------------------------------------------------
SELECT REVERSE('Ajay');
→ 'yajA'

----------------------------------------------------------------------
WITH TABLE DATA
----------------------------------------------------------------------
SELECT REVERSE(name) FROM student;

→ Reverses each name

----------------------------------------------------------------------
USE CASE
----------------------------------------------------------------------
• Reverse string
• Useful in pattern problems
• Can help extract last part of string

Example:
SELECT REVERSE('ajay@gmail.com');
→ 'moc.liamg@yaja'

----------------------------------------------------------------------
COMBINED USAGE
----------------------------------------------------------------------
SELECT REVERSE(SUBSTRING('AjaySharma', 1, 4));
→ 'yajA'

----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) Works only on strings
2) Does NOT change original data
3) Output only is reversed
4) Can be combined with other functions

MYSQL ARITHMETIC OPERATORS
======================================================================

DEFINITION:
Arithmetic operators are used to perform mathematical calculations.

----------------------------------------------------------------------
TYPES OF OPERATORS
----------------------------------------------------------------------

+   Addition
-   Subtraction
*   Multiplication
/   Division
%   Modulus (remainder)

----------------------------------------------------------------------
BASIC EXAMPLES
----------------------------------------------------------------------
SELECT 2 + 3;      → 5
SELECT 10 - 4;     → 6
SELECT 5 * 2;      → 10
SELECT 10 / 2;     → 5
SELECT 10 % 3;     → 1

----------------------------------------------------------------------
USING WITH TABLE
----------------------------------------------------------------------
SELECT name, age + 5 FROM student;
→ Adds 5 to each age

SELECT price * quantity FROM orders;
→ Total cost per row

----------------------------------------------------------------------
COLUMN ALIAS
----------------------------------------------------------------------
SELECT price * quantity AS total_amount
FROM orders;

SELECT name, age , age+5 as after_5_years FROM people;

----------------------------------------------------------------------
WITH WHERE
----------------------------------------------------------------------
SELECT * FROM student
WHERE age + 5 > 25;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
SELECT 5 + NULL;   → NULL
SELECT 10 * NULL;  → NULL

→ Any operation with NULL → NULL

----------------------------------------------------------------------
DIVISION NOTES
----------------------------------------------------------------------
SELECT 10 / 3;   → 3.3333
SELECT 10 DIV 3; → 3   (integer division)

----------------------------------------------------------------------
MODULUS (%)
----------------------------------------------------------------------
SELECT 10 % 3;   → 1

→ Gives remainder

----------------------------------------------------------------------
OPERATOR PRECEDENCE
----------------------------------------------------------------------
1) * , / , %
2) + , -

Example:
SELECT 2 + 3 * 4;
→ 14 (not 20)

----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) Used in SELECT, WHERE
2) Works on numeric values
3) Use () to control order
4) NULL affects result
5) Can combine with alias

DISCOUNT CALCULATION USING ARITHMETIC OPERATORS (MYSQL)
======================================================================

FORMULA:
Final Price = Original Price - Discount

OR

Final Price = Price - (Price * Discount / 100)

----------------------------------------------------------------------
EXAMPLE (BASIC)
----------------------------------------------------------------------
SELECT 1000 - (1000 * 10 / 100);
→ 900

----------------------------------------------------------------------
WITH TABLE
----------------------------------------------------------------------
Table: products (name, price, discount)

SELECT name, 
       price, 
       discount,
       price - (price * discount / 100) AS final_price
FROM products;

----------------------------------------------------------------------
OUTPUT LOGIC
----------------------------------------------------------------------
price = 1000, discount = 10

price * discount / 100 = 100
final_price = 1000 - 100 = 900

----------------------------------------------------------------------
SHOW DISCOUNT AMOUNT
----------------------------------------------------------------------
SELECT name,
       price,
       discount,
       (price * discount / 100) AS discount_amount
FROM products;

----------------------------------------------------------------------
WITH WHERE
----------------------------------------------------------------------
SELECT * FROM products
WHERE price - (price * discount / 100) < 500;

→ Products with final price < 500

----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) discount is in percentage
2) Always divide by 100
3) Use alias for readability
4) Works with numeric columns

ARITHMETIC OPERATORS — EDGE CASES (IMPORTANT)
======================================================================

1) DIVISION BY ZERO
----------------------------------------------------------------------
SELECT 10 / 0;
→ ERROR or NULL (depends on SQL mode) ❌

✔ Avoid:
WHERE denominator != 0

----------------------------------------------------------------------
2) NULL VALUES
----------------------------------------------------------------------
SELECT 10 + NULL;   → NULL
SELECT 5 * NULL;    → NULL

→ Any operation with NULL → NULL

----------------------------------------------------------------------
3) STRING + NUMBER
----------------------------------------------------------------------
SELECT '10' + 5;
→ 15  (string converted to number)

SELECT 'abc' + 5;
→ 5   ('abc' treated as 0)

----------------------------------------------------------------------
4) NEGATIVE VALUES
----------------------------------------------------------------------
SELECT -10 + 5;   → -5
SELECT 10 % -3;   → 1  (sign rules apply)

----------------------------------------------------------------------
5) FLOAT PRECISION
----------------------------------------------------------------------
SELECT 10 / 3;
→ 3.3333 (approx)

→ Floating point may not be exact

----------------------------------------------------------------------
6) INTEGER DIVISION
----------------------------------------------------------------------
SELECT 10 DIV 3;
→ 3  (removes decimal part)

----------------------------------------------------------------------
7) MODULUS WITH ZERO
----------------------------------------------------------------------
SELECT 10 % 0;
→ ERROR ❌

----------------------------------------------------------------------
8) LARGE NUMBERS (OVERFLOW)
----------------------------------------------------------------------
SELECT 9999999999999999999 * 999999999999;
→ May overflow / incorrect result

----------------------------------------------------------------------
9) ORDER OF OPERATIONS
----------------------------------------------------------------------
SELECT 2 + 3 * 4;
→ 14 (not 20)

Use brackets:
SELECT (2 + 3) * 4;
→ 20

----------------------------------------------------------------------
10) MIXED DATA TYPES
----------------------------------------------------------------------
SELECT 10 + '20abc';
→ 30  (string converted to number until invalid part)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Always validate data before calculation
2) Handle NULL and zero cases carefully
3) Use brackets for clarity
4) Be careful with type conversion

MYSQL MODIFYING TABLES (ALTER TABLE)
======================================================================

DEFINITION:
ALTER TABLE is used to modify existing table structure.

----------------------------------------------------------------------
1) ADD COLUMN
----------------------------------------------------------------------
ALTER TABLE student
ADD age INT;

ALTER TABLE student
ADD salary INT NOT NULL; // salary will take 0 , because NOT NULL is used, otherwise it will be NULL for unspecified.

----------------------------------------------------------------------
2) ADD MULTIPLE COLUMNS
----------------------------------------------------------------------
ALTER TABLE student
ADD (city VARCHAR(50), marks INT);

----------------------------------------------------------------------
3) MODIFY COLUMN (CHANGE DATATYPE / SIZE)
---------------------------------------------------------------------DEFINITION:
MODIFY is used to change datatype, size, or constraints 
of an existing column.

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
ALTER TABLE table_name
MODIFY column_name new_datatype;

----------------------------------------------------------------------
EXAMPLE (CHANGE SIZE)
----------------------------------------------------------------------
ALTER TABLE student
MODIFY name VARCHAR(100);

→ Increases size from smaller to 100

----------------------------------------------------------------------
CHANGE DATATYPE
----------------------------------------------------------------------
ALTER TABLE student
MODIFY age BIGINT;

→ INT → BIGINT

----------------------------------------------------------------------
ADD CONSTRAINT
----------------------------------------------------------------------
ALTER TABLE student
MODIFY name VARCHAR(50) NOT NULL;

→ Now name cannot be NULL

----------------------------------------------------------------------
REMOVE CONSTRAINT
----------------------------------------------------------------------
ALTER TABLE student
MODIFY name VARCHAR(50) NULL;

→ Allows NULL values

----------------------------------------------------------------------
IMPORTANT CASES
----------------------------------------------------------------------

1) DATA LOSS RISK
----------------------------------------------------------------------
• Reducing size may truncate data

VARCHAR(100) → VARCHAR(10)
→ Extra characters may be lost ❌

----------------------------------------------------------------------
2) INCOMPATIBLE DATATYPE
----------------------------------------------------------------------
• Changing datatype may fail if data is not compatible like characters cant turn into int.

VARCHAR → INT (if non-numeric data exists)

----------------------------------------------------------------------
3) DEFAULT VALUE RESET
----------------------------------------------------------------------
• MODIFY may remove default if not specified again

----------------------------------------------------------------------
WITH AFTER / FIRST
----------------------------------------------------------------------
ALTER TABLE student
MODIFY name VARCHAR(100) AFTER id;

→ Changes type + position

----------------------------------------------------------------------

case:Converting datatype when column has NULL values.
NULL remains NULL (no conversion happens)


NOTES
----------------------------------------------------------------------
1) Used to change structure
2) Always re-define full column (type + constraints)
3) Be careful with existing data
4) Backup recommended

----------------------------------------------------------------------
4) CHANGE COLUMN NAME
---------------------------------------------------------------------

DEFINITION:
CHANGE is used to rename a column (and optionally change datatype)

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
ALTER TABLE table_name
CHANGE old_column_name new_column_name datatype;

----------------------------------------------------------------------
EXAMPLE (RENAME COLUMN)
----------------------------------------------------------------------
ALTER TABLE student
CHANGE name full_name VARCHAR(50);

→ Column 'name' renamed to 'full_name'

----------------------------------------------------------------------
IMPORTANT RULE
----------------------------------------------------------------------
• Datatype MUST be specified (even if not changing)

----------------------------------------------------------------------
CHANGE NAME + DATATYPE
----------------------------------------------------------------------
ALTER TABLE student
CHANGE age student_age BIGINT;

→ Renames + changes datatype

----------------------------------------------------------------------
CHANGE WITH POSITION
----------------------------------------------------------------------
ALTER TABLE student
CHANGE name full_name VARCHAR(50) AFTER id;

→ Renamed + moved after 'id'

----------------------------------------------------------------------
COMMON MISTAKE
----------------------------------------------------------------------
ALTER TABLE student
CHANGE name full_name;   ❌ ERROR

→ Datatype missing

----------------------------------------------------------------------
MODERN ALTERNATIVE (MYSQL 8+)
----------------------------------------------------------------------
ALTER TABLE student
RENAME COLUMN name TO full_name;

→ Only renames (simpler)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) CHANGE = rename + redefine column
2) Must mention datatype
3) Use RENAME COLUMN if only renaming
4) Be careful → structure changes permanently

DROP PRIMARY KEY (MYSQL)
======================================================================

QUESTION:
Can we drop PRIMARY KEY?

✔ YES — but with conditions

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
ALTER TABLE table_name
DROP PRIMARY KEY;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
ALTER TABLE student
DROP PRIMARY KEY;

→ Removes primary key from table

----------------------------------------------------------------------
IMPORTANT RULES
----------------------------------------------------------------------

1) ONLY ONE PRIMARY KEY EXISTS
----------------------------------------------------------------------
• Table can have only one primary key
• DROP removes that constraint

----------------------------------------------------------------------
2) AUTO_INCREMENT DEPENDENCY
----------------------------------------------------------------------
If column has AUTO_INCREMENT:

ALTER TABLE student DROP PRIMARY KEY;
→ ERROR ❌

✔ Fix:
ALTER TABLE student
MODIFY id INT;   (remove AUTO_INCREMENT first)

Then:
ALTER TABLE student
DROP PRIMARY KEY;

----------------------------------------------------------------------
3) FOREIGN KEY DEPENDENCY
----------------------------------------------------------------------
If other table references this PK:

→ Cannot drop ❌

✔ Must drop FOREIGN KEY first

----------------------------------------------------------------------
4) DATA WILL REMAIN
----------------------------------------------------------------------
• Only constraint is removed
• Data stays unchanged

----------------------------------------------------------------------
5) NO PRIMARY KEY AFTER DROP
----------------------------------------------------------------------
• Table will have no unique identifier
• Not recommended in real systems

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Use carefully
2) Check dependencies before dropping
3) Often replaced with new primary key

ALTER TABLE ... ADD UNIQUE(name)  (EXPLANATION)
======================================================================

SYNTAX YOU WROTE:
----------------------------------------------------------------------
ALTER TABLE people
ADD UNIQUE(name);

----------------------------------------------------------------------
IS IT CORRECT?
----------------------------------------------------------------------
✔ YES — Valid syntax in MySQL

→ Adds UNIQUE constraint on 'name' column

----------------------------------------------------------------------
EQUIVALENT FORM (WITH NAME)
----------------------------------------------------------------------
ALTER TABLE people
ADD CONSTRAINT unique_name UNIQUE (name);

→ Same functionality (just gives constraint a name)

----------------------------------------------------------------------
DIFFERENCE
----------------------------------------------------------------------

ADD UNIQUE(name)
→ MySQL auto-generates constraint/index name

ADD CONSTRAINT unique_name UNIQUE(name)
→ You define the constraint name

----------------------------------------------------------------------
WHEN TO USE WHICH
----------------------------------------------------------------------

• Simple use → ADD UNIQUE(name) ✔
• Production / clarity → ADD CONSTRAINT ✔ (better)

----------------------------------------------------------------------
EDGE CASE
----------------------------------------------------------------------

If duplicates exist:

ALTER TABLE people ADD UNIQUE(name);
→ ERROR ❌ Duplicate entry

----------------------------------------------------------------------
HOW TO CHECK BEFORE ADDING
----------------------------------------------------------------------
SELECT name, COUNT(*)
FROM people
GROUP BY name
HAVING COUNT(*) > 1;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) UNIQUE creates index internally
2) Prevents duplicate values
3) Multiple UNIQUE constraints allowed
4) Naming constraint helps in future DROP

NOTES
----------------------------------------------------------------------
1) Constraints enforce data rules
2) Check existing data before applying
3) Changes are permanent
4) Use MODIFY for NOT NULL / DEFAULT

11) AFTER KEYWORD (WITH ALTER TABLE)
======================================================================

DEFINITION:
AFTER is used to specify position of a new column
while adding it to an existing table.

----------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------
ALTER TABLE table_name
ADD column_name datatype AFTER existing_column;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
ALTER TABLE student
ADD city VARCHAR(50) AFTER name;

→ Adds 'city' column after 'name'

----------------------------------------------------------------------
DEFAULT BEHAVIOR (WITHOUT AFTER)
----------------------------------------------------------------------
ALTER TABLE student
ADD age INT;

→ Column added at END of table

----------------------------------------------------------------------
ADD FIRST (RELATED)
----------------------------------------------------------------------
ALTER TABLE student
ADD id INT FIRST;

→ Adds column at beginning

----------------------------------------------------------------------
MULTIPLE COLUMNS WITH AFTER
----------------------------------------------------------------------
ALTER TABLE student
ADD marks INT AFTER age;

→ Added after 'age'

----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) Only affects column order (not data)
2) Works only with ADD COLUMN
3) Optional keyword
4) FIRST → beginning, AFTER → specific position

======================================================================
----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) ALTER TABLE modifies structure, not data
2) Be careful → changes are permanent
3) Large tables may take time
4) Backup recommended before changes

DELETE + AUTO_INCREMENT BEHAVIOR
======================================================================

QUESTION:
After DELETE, if we INSERT again → what id (index) is assigned?

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
Table:
id (AUTO_INCREMENT)

id | name
---------
1  | Ajay
2  | Rahul
3  | Amit

DELETE:
DELETE FROM student WHERE id = 2;

Now:
id | name
---------
1  | Ajay
3  | Amit

----------------------------------------------------------------------
INSERT NEW RECORD
----------------------------------------------------------------------
INSERT INTO student (name)
VALUES ('Neha');

→ New id = 4  ✔

----------------------------------------------------------------------
KEY POINT
----------------------------------------------------------------------
AUTO_INCREMENT does NOT reuse deleted values

→ It always continues forward

----------------------------------------------------------------------
WHY?
----------------------------------------------------------------------
• Ensures uniqueness
• Avoids conflicts
• Maintains consistency

----------------------------------------------------------------------
RESET AUTO_INCREMENT (OPTIONAL)
----------------------------------------------------------------------
ALTER TABLE student AUTO_INCREMENT = 2;

→ Next insert will try from 2 (only if free)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Deleted ids are NOT reused automatically
2) Gaps in id are normal
3) Do not rely on continuous numbering
4) AUTO_INCREMENT ensures unique ids

MYSQL ORDER BY
======================================================================

DEFINITION:
ORDER BY is used to sort result data.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column_name
FROM table_name
ORDER BY column_name;

----------------------------------------------------------------------
ASCENDING (DEFAULT)
----------------------------------------------------------------------
SELECT * FROM student
ORDER BY age;

→ Sorts age in ascending order (small → large)

----------------------------------------------------------------------
DESCENDING
----------------------------------------------------------------------
SELECT * FROM student
ORDER BY age DESC;

→ Sorts age in descending order (large → small)

----------------------------------------------------------------------
MULTIPLE COLUMNS
----------------------------------------------------------------------
SELECT * FROM student
ORDER BY city ASC, age DESC;

→ First sort by city, then age

----------------------------------------------------------------------
USING COLUMN POSITION
----------------------------------------------------------------------
SELECT name, age FROM student
ORDER BY 2;

→ Sort by 2nd column (age)

----------------------------------------------------------------------
WITH WHERE
----------------------------------------------------------------------
SELECT * FROM student
WHERE age > 20
ORDER BY age;

----------------------------------------------------------------------
WITH ALIAS
----------------------------------------------------------------------
SELECT name, age + 5 AS future_age
FROM student
ORDER BY future_age;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULLs come first in ASC
• NULLs come last in DESC

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Default sorting → ASC
2) Comes after WHERE
3) Can sort by multiple columns
4) Improves result readability

MYSQL LIMIT
======================================================================

DEFINITION:
LIMIT is used to restrict number of rows returned.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column_name
FROM table_name
LIMIT number;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
SELECT * FROM student
LIMIT 5;

→ Returns first 5 rows

----------------------------------------------------------------------
WITH OFFSET
----------------------------------------------------------------------
SELECT * FROM student
LIMIT offset, count;

SELECT * FROM student
LIMIT 2, 3;

→ Skip first 2 rows, return next 3 rows

----------------------------------------------------------------------
ALTERNATIVE SYNTAX
----------------------------------------------------------------------
SELECT * FROM student
LIMIT 3 OFFSET 2;

→ Same as LIMIT 2, 3

----------------------------------------------------------------------
WITH ORDER BY
----------------------------------------------------------------------
SELECT * FROM student
ORDER BY age DESC
LIMIT 3;

→ Top 3 highest ages

----------------------------------------------------------------------
USE CASES
----------------------------------------------------------------------
• Pagination
• Top N records
• Testing queries

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) LIMIT comes at end
2) Default offset = 0
3) Used with ORDER BY for meaningful results
4) Improves performance on large data
