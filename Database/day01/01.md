DATA IN DATABASE
======================================================================

Definition:
Data = Raw facts or information stored in a database for processing.
Examples:numbers,text,images, transactions.
Data alone has lottle meaning, when processed it becomes useful.

----------------------------------------------------------------------
TYPES OF DATA
----------------------------------------------------------------------

1) READABLE DATA (Human-readable)
----------------------------------------------------------------------
Definition:
Data that humans can easily read and understand.

Examples:
• Names       → "Ajay"
• Numbers     → 101, 22
• Text        → "Hello World"
• Dates       → 17-03-2026

Used in:
• Reports
• User interfaces
• Queries output

----------------------------------------------------------------------
2) BINARY DATA (Machine-readable)
----------------------------------------------------------------------
Definition:
Data stored in the form of 0s and 1s (not directly readable by humans).

Examples:
• Images      → JPG, PNG
• Videos      → MP4
• Audio       → MP3
• Files       → PDF, DOC

Stored as:
• BLOB (Binary Large Object)

Used in:
• Multimedia storage
• File storage in databases

----------------------------------------------------------------------
KEY POINT
----------------------------------------------------------------------
Readable Data → Easy for humans to understand
Binary Data   → Needs system/software to interpret

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
Data in database = Raw facts stored in structured form, can be
either human-readable (text, numbers) or binary (images, files).

DATABASE
======================================================================

Definition:
Database = A structured collection of data stored electronically
that can be easily accessed, managed, and updated.

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
Student Table:

ID     Name      Age
101    Ajay      22
102    Rahul     23

This organized collection is called a DATABASE.

----------------------------------------------------------------------
FEATURES
----------------------------------------------------------------------
• Stores large amount of data
• Organized in tables (rows & columns)
• Easy to access and retrieve,inserted, updated, deleted
• Provides security
• Provides backup and recovery
• Reduces data duplication (redundancy)
• Enables multiple users to access data simultaneously
- Ensure data consistency
INTERVIEW LINE
----------------------------------------------------------------------
Database provides structured storage, easy operations, security,
backup, reduced redundancy, and multi-user access.
======================================================================

TYPES OF DATA STORED
----------------------------------------------------------------------
• Readable Data → text, numbers, dates

----------------------------------------------------------------------
KEY COMPONENTS
----------------------------------------------------------------------

----------------------------------------------------------------------
INTERVIEW LINE

Database = Organized collection of data stored electronically
for easy access and management.

======================================================================

Definition:
DBMS = Software used to create, manage, and control databases.

----------------------------------------------------------------------
EXAMPLES:These languages below are used to communicate or interact with database.
----------------------------------------------------------------------
• MySQL
• Oracle
• SQL Server
• PostgreSQL

----------------------------------------------------------------------
PURPOSE
----------------------------------------------------------------------
• Store data
• Manage data
• Retrieve data efficiently

----------------------------------------------------------------------
FEATURES
----------------------------------------------------------------------
1) DATA STORAGE
   • Stores large amount of data

2) DATA ORGANIZATION
   • Data stored in tables (rows & columns)

3) DATA OPERATIONS
   • Insert, Update, Delete, Retrieve

4) SECURITY
   • Controls access using users/roles

5) BACKUP & RECOVERY
   • Protects data from loss

6) REDUNDANCY CONTROL
   • Minimizes duplicate data

7) MULTI-USER ACCESS
   • Multiple users can work simultaneously

----------------------------------------------------------------------
COMPONENTS
----------------------------------------------------------------------
• Database        → Collection of data
• DBMS Software   → Manages database
• User            → Interacts with DB

ADVANTAGES

• Better security
• Reduced redundancy

DISADVANTAGES

SQL (STRUCTURED QUERY LANGUAGE) - NOTES
======================================================================

Definition:
SQL = Language used to communicate with DBMS to manage data, developed by microsoft.

----------------------------------------------------------------------
PURPOSE
----------------------------------------------------------------------
• Create database & tables
• Insert, update, delete data
• Retrieve data (queries)
• Control access

----------------------------------------------------------------------
TYPES OF SQL COMMANDS
----------------------------------------------------------------------

1) DDL (Data Definition Language)
----------------------------------------------------------------------
Used to define database structure

• CREATE  → create database/table
• ALTER   → modify structure
• DROP    → delete database/table
• TRUNCATE→ remove all data (structure remains)

----------------------------------------------------------------------
2) DML (Data Manipulation Language)
----------------------------------------------------------------------
Used to modify data

• INSERT → add data
• UPDATE → modify data
• DELETE → remove data

----------------------------------------------------------------------
3) DQL (Data Query Language)
----------------------------------------------------------------------
Used to fetch data

• SELECT → retrieve data

----------------------------------------------------------------------
4) DCL (Data Control Language)
----------------------------------------------------------------------
Used for permissions

• GRANT  → give access
• REVOKE → remove access

----------------------------------------------------------------------
5) TCL (Transaction Control Language)
----------------------------------------------------------------------
Used for transaction management

• COMMIT   → save changes
• ROLLBACK → undo changes
• SAVEPOINT→ temporary save

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------

SELECT * FROM table_name;

INSERT INTO table_name VALUES (...);

UPDATE table_name
SET column = value
WHERE condition;

DELETE FROM table_name
WHERE condition;

----------------------------------------------------------------------
KEY POINTS
----------------------------------------------------------------------
• SQL is case-insensitive
• Works with relational databases
• Uses tables (rows & columns)

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
SQL = Language used to create, manage, and retrieve data
from relational databases.

RDBMS (RELATIONAL DATABASE MANAGEMENT SYSTEM) - SHORT NOTES
======================================================================

Definition:
RDBMS = DBMS that stores data in tables with relationships.

----------------------------------------------------------------------
KEY FEATURES
----------------------------------------------------------------------
• Data stored in tables (rows & columns)
• Tables are related using keys
• Ensures data integrity
• Reduces redundancy
• Supports SQL

----------------------------------------------------------------------
KEY CONCEPTS
----------------------------------------------------------------------
• Primary Key   → uniquely identifies a record
• Foreign Key   → links tables
• Relationship  → connection between tables

----------------------------------------------------------------------
EXAMPLE TABLES
----------------------------------------------------------------------

STUDENT TABLE
----------------------------------------------------------------------
StudentID   Name     Age
101         Ajay     22
102         Rahul    23

COURSE TABLE
----------------------------------------------------------------------
CourseID    CourseName
1           Java
2           SQL

ENROLLMENT TABLE (Relation)
----------------------------------------------------------------------
StudentID   CourseID
101         1
102         2

Explanation:
• StudentID → Primary Key in STUDENT table
• CourseID  → Primary Key in COURSE table
• In ENROLLMENT table:
  - StudentID & CourseID act as Foreign Keys
  - This creates relationship between tables

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
RDBMS = DBMS that organizes data into related tables
using keys to maintain relationships.

NoSQL (NOT ONLY SQL) - SHORT NOTES
======================================================================

Definition:
NoSQL = A type of database management system that does NOT use the
traditional table-based (relational) structure. Instead, it stores
data in flexible formats like JSON documents, key-value pairs,
columns, or graphs.

It is designed to handle large volumes of data, high-speed operations,
and distributed systems, making it suitable for modern applications
like web apps, real-time systems, and big data.

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------
• MongoDB
• Cassandra
• Redis
• Neo4j

----------------------------------------------------------------------
KEY FEATURES
----------------------------------------------------------------------
• No fixed table structure (schema-less)
• Stores large-scale data (Big Data)
• High performance & scalability
• Flexible data models
• Suitable for unstructured/semi-structured data

----------------------------------------------------------------------
KEY CONCEPTS
----------------------------------------------------------------------
• Document-based   → JSON-like data
• Key-Value        → data stored as key:value
• Column-based     → column-wise storage
• Graph-based      → nodes & relationships

----------------------------------------------------------------------
EXAMPLE (DOCUMENT - JSON FORMAT)
----------------------------------------------------------------------

{
  "id": 101,
  "name": "Ajay",
  "age": 22,
  "courses": ["Java", "SQL"]
}

Explanation:
• Data stored as JSON document
• No fixed columns required
• Flexible structure (can add/remove fields)

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
NoSQL = Non-relational database designed for flexible,
scalable, and high-performance data storage.

RDBMS vs NoSQL (COMPARISON TABLE)
======================================================================

Feature              RDBMS                          NoSQL
----------------------------------------------------------------------
Full Form            Relational DBMS                Document, key-value, pair, graphs
----------------------------------------------------------------------
Structure            Tables (rows & columns)        Flexible (JSON, key-value, etc.)
----------------------------------------------------------------------
Schema               Fixed schema                   Schema-less (dynamic)
----------------------------------------------------------------------
Best For            Structured data                Structured + Unstructured
----------------------------------------------------------------------
Scalability          Vertical scaling               Horizontal scaling
----------------------------------------------------------------------
Performance          Moderate (joins affect speed)  High (optimized for speed)
----------------------------------------------------------------------
Relationships        Strong (using keys & joins)    Weak / handled differently
----------------------------------------------------------------------
Query Language       SQL                            No fixed standard (varies)
----------------------------------------------------------------------
Examples             MySQL, Oracle                  MongoDB, Redis
----------------------------------------------------------------------
Use Case             Banking, ERP systems           Big Data, real-time apps
----------------------------------------------------------------------
Flexibility          Low                            High

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
RDBMS = Structured, table-based with fixed schema  
NoSQL = Flexible, schema-less for scalable systems

MySQL COMMAND PROMPT - NOTES
======================================================================

----------------------------------------------------------------------
1) START & LOGIN
----------------------------------------------------------------------
mysql --version
→ Check MySQL installation

mysql -u root -p
→ Login using username & password

Note:
• Enter password carefully (not visible while typing)
• Note down username & password

----------------------------------------------------------------------
2) BASIC NAVIGATION
----------------------------------------------------------------------
mysql> system cls
→ Clear screen

exit
→ Exit MySQL

mysql -u root -p
→ Login again

----------------------------------------------------------------------
3) DATABASE COMMANDS
----------------------------------------------------------------------
show databases;
→ List all databases

create database abc_de;
→ Create new database

drop database test;
→ Delete database

use social_db;
→ Select database (all operations will apply here)

select database();
→ Check currently selected database

----------------------------------------------------------------------
4) TABLE COMMANDS
----------------------------------------------------------------------
create table people(
  ID INT,
  name VARCHAR(30),
  age INT,
  location VARCHAR(30)
);

SHOW TABLES;
→ Show all tables in current database

DESCRIBE people;
or
SHOW COLUMNS FROM people;
→ Show table structure

----------------------------------------------------------------------
5) TERMINAL TIPS
----------------------------------------------------------------------
• ";" is mandatory after every SQL command
• If command is incomplete → use \c to cancel
• Use ↑ (up arrow) to see previous commands

----------------------------------------------------------------------
INTERVIEW LINE
----------------------------------------------------------------------
MySQL command prompt is used to execute SQL commands for
database creation, manipulation, and management.
