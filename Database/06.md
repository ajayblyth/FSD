======================================================================
MYSQL AGGREGATE FUNCTIONS (COMPLETE NOTES)
======================================================================

DEFINITION:
Aggregate functions perform calculations on multiple rows
and return a single result.

----------------------------------------------------------------------
MAIN AGGREGATE FUNCTIONS
----------------------------------------------------------------------

1) COUNT()
2) SUM()
3) AVG()
4) MIN()
5) MAX()

----------------------------------------------------------------------
1) COUNT()
----------------------------------------------------------------------
======================================================================
MYSQL COUNT() FUNCTION (COMPLETE + EDGE CASES)
======================================================================

DEFINITION:
COUNT() returns number of rows or number of non-NULL values.

----------------------------------------------------------------------
TYPES OF COUNT
----------------------------------------------------------------------

1) COUNT(*)
2) COUNT(column)
3) COUNT(DISTINCT column)

----------------------------------------------------------------------
1) COUNT(*)
----------------------------------------------------------------------
SELECT COUNT(*) FROM student;
→ Counts ALL rows (NULL or NOT NULL)

Example:
Rows = 5 (even if some columns have NULL)
→ Output: 5

----------------------------------------------------------------------
2) COUNT(column)
----------------------------------------------------------------------
SELECT COUNT(age) FROM student;
→ Counts only NON-NULL values in age

Example:
age = 20, 22, NULL, 25
→ Output: 3

----------------------------------------------------------------------
3) COUNT(DISTINCT column)
----------------------------------------------------------------------
SELECT COUNT(DISTINCT city) FROM student;
→ Counts UNIQUE non-NULL values

Example:
city = Delhi, Mumbai, Delhi, NULL
→ Output: 2

----------------------------------------------------------------------
USING WITH WHERE
----------------------------------------------------------------------
SELECT COUNT(*) FROM student
WHERE age > 20;

→ Counts rows satisfying condition

----------------------------------------------------------------------
USING WITH GROUP BY
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
GROUP BY city;

→ Count per city

----------------------------------------------------------------------
COUNT WITH MULTIPLE COLUMNS (MySQL)
----------------------------------------------------------------------
SELECT COUNT(DISTINCT city, age) FROM student;

→ Counts unique combinations of (city, age)

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------

1) ALL VALUES NULL
----------------------------------------------------------------------
SELECT COUNT(age) FROM student;
→ 0

SELECT COUNT(*) FROM student;
→ Total rows (e.g., 5)

----------------------------------------------------------------------
2) NO ROWS IN TABLE
----------------------------------------------------------------------
SELECT COUNT(*) FROM student;
→ 0

SELECT COUNT(age) FROM student;
→ 0

----------------------------------------------------------------------
3) WITH WHERE RETURNING NO ROWS
----------------------------------------------------------------------
SELECT COUNT(*) FROM student WHERE city = 'XYZ';
→ 0

----------------------------------------------------------------------
4) COUNT(*) vs COUNT(1)
----------------------------------------------------------------------
SELECT COUNT(*) FROM student;
SELECT COUNT(1) FROM student;

→ Same result (no difference in MySQL)

----------------------------------------------------------------------
5) NULL VALUES IN DISTINCT
----------------------------------------------------------------------
SELECT COUNT(DISTINCT city) FROM student;

→ NULL is ignored (not counted)

----------------------------------------------------------------------
6) COUNT WITH EXPRESSIONS
----------------------------------------------------------------------
SELECT COUNT(age + 10) FROM student;

→ NULL values still ignored

----------------------------------------------------------------------
7) COUNT WITH JOIN (IMPORTANT)
----------------------------------------------------------------------
SELECT COUNT(*)
FROM student s
JOIN orders o ON s.id = o.student_id;

→ Counts joined rows (can increase count)

----------------------------------------------------------------------
8) COUNT OVERFLOW (RARE)
----------------------------------------------------------------------
Very large tables → BIGINT used internally
(no overflow in normal use)

----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------

❌ Expecting COUNT(column) to include NULL
→ It does NOT

❌ Confusing COUNT(*) with COUNT(column)
→ COUNT(*) counts rows
→ COUNT(column) counts values

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) COUNT(*) = total rows
2) COUNT(column) = non-NULL values
3) COUNT(DISTINCT) = unique non-NULL values
4) Always returns integer
5) Widely used in analytics

----------------------------------------------------------------------
2) SUM()
----------------------------------------------------------------------
DEFINITION:
Returns the total (sum) of a numeric column.

SYNTAX:
SELECT SUM(column_name) FROM table_name;

EXAMPLE:
SELECT SUM(age) FROM student;
→ Total of all ages (NULL ignored)

----------------------------------------------------------------------
HOW SUM IS CALCULATED
----------------------------------------------------------------------
SUM(age) = age1 + age2 + age3 + ...

Example:
Ages: 20, 25, NULL, 30
SUM = 20 + 25 + 30 = 75

→ NULL is ignored

----------------------------------------------------------------------
USAGE WITH WHERE
----------------------------------------------------------------------
SELECT SUM(age) FROM student
WHERE city = 'Delhi';

→ Total age of Delhi students

----------------------------------------------------------------------
USAGE WITH GROUP BY
----------------------------------------------------------------------
SELECT city, SUM(age)
FROM student
GROUP BY city;

→ Total age per city

----------------------------------------------------------------------
SUM WITH DISTINCT
----------------------------------------------------------------------
SELECT SUM(DISTINCT age) FROM student;

→ Adds only unique values

Example:
Ages: 20, 20, 30
SUM = 20 + 30 = 50

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL values are ignored
• If all values are NULL → result is NULL

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
1) Empty table → NULL
2) All NULL values → NULL
3) Non-numeric column → 0 or error (implicit conversion)
4) Mixed data → strings may convert to 0 ❌
5) Very large values → possible overflow (rare)

----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------
❌ Expecting NULL to be counted
→ NULL is ignored

❌ Using SUM on string column
→ May give wrong result

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Works only on numeric columns
2) Returns single value
3) Ignores NULL values
4) Can be used with GROUP BY
5) Used in totals, reports

======================================================================

----------------------------------------------------------------------
3) AVG()
----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT AVG(column_name)
FROM table_name;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
SELECT AVG(age) FROM student;
→ Returns average age (NULL values ignored)

----------------------------------------------------------------------
HOW AVG IS CALCULATED
----------------------------------------------------------------------
AVG(age) = SUM(age) / COUNT(age)

Note:
• COUNT(age) counts only NON-NULL values

Example:
Ages: 20, 22, 24, NULL

SUM = 66
COUNT = 3
AVG = 66 / 3 = 22

----------------------------------------------------------------------
USING WITH WHERE
----------------------------------------------------------------------
SELECT AVG(age)
FROM student
WHERE city = 'Delhi';

→ Average age of Delhi students

SELECT AVG (salary) FROM employees
WHERE department = "IT" and salary > 3000;
----------------------------------------------------------------------
USING WITH GROUP BY
----------------------------------------------------------------------
SELECT city, AVG(age)
FROM student
GROUP BY city;

→ Average age per city

----------------------------------------------------------------------
ROUNDING RESULT
----------------------------------------------------------------------
SELECT ROUND(AVG(age), 2)
FROM student;

→ Average rounded to 2 decimal places

----------------------------------------------------------------------
AVG WITH DISTINCT
----------------------------------------------------------------------
SELECT AVG(DISTINCT age)
FROM student;

→ Average of unique age values only

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL values are ignored
• If all values are NULL → result is NULL
→ AVG always returns decimal

-- ROUNDING ISSUES
SELECT AVG(age) FROM student;
→ 21.666666

SELECT ROUND(AVG(age), 2);
→ 21.67

--NEGATIVE VALUES
Values: -10, 20

AVG = 5

→ Works normally
----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------

❌ AVG on non-numeric column
SELECT AVG(name) FROM student;
→ Error / invalid

❌ Expecting NULL to be counted
→ NULL is ignored in calculation

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Works only on numeric columns
2) Returns decimal value
3) Ignores NULL values
4) Internally uses SUM / COUNT
5) Widely used in analytics


----------------------------------------------------------------------
4) MIN()
----------------------------------------------------------------------
DEFINITION:
Returns the smallest (minimum) value from a column.

SYNTAX:
SELECT MIN(column_name) FROM table_name;

EXAMPLE:
SELECT MIN(age) FROM student;
→ Minimum age

----------------------------------------------------------------------
USAGE WITH WHERE
----------------------------------------------------------------------
SELECT MIN(age) FROM student
WHERE city = 'Delhi';

→ Minimum age among Delhi students

----------------------------------------------------------------------
USAGE WITH GROUP BY
----------------------------------------------------------------------
SELECT city, MIN(age)
FROM student
GROUP BY city;

→ Minimum age per city

----------------------------------------------------------------------
WORKS WITH STRINGS & DATES
----------------------------------------------------------------------
SELECT MIN(name) FROM student;
→ First name alphabetically

SELECT MIN(order_date) FROM orders;
→ Earliest date

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL values are ignored
• If all values are NULL → result is NULL

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
1) Empty table → NULL
2) All NULL values → NULL
3) String → dictionary order
4) Date → earliest date
5) Mixed types → may cause implicit conversion issues

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Returns single value
2) Works on numbers, strings, dates
3) Ignores NULL values
4) Useful for finding lowest values

----------------------------------------------------------------------
DEFINITION:
Returns the largest (maximum) value from a column.

SYNTAX:
SELECT MAX(column_name) FROM table_name;

EXAMPLE:
SELECT MAX(age) FROM student;
→ Maximum age

----------------------------------------------------------------------
USAGE WITH WHERE
----------------------------------------------------------------------
SELECT MAX(age) FROM student
WHERE city = 'Delhi';

→ Maximum age among Delhi students

----------------------------------------------------------------------
USAGE WITH GROUP BY
----------------------------------------------------------------------
SELECT city, MAX(age)
FROM student

→ Maximum age per city

----------------------------------------------------------------------
WORKS WITH STRINGS & DATES
----------------------------------------------------------------------
SELECT MAX(name) FROM student;
→ Last name alphabetically

→ Latest date

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL values are ignored
• If all values are NULL → result is NULL

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------
1) Empty table → NULL
2) All NULL values → NULL
3) String → dictionary order
4) Date → latest date
5) Mixed types → may cause unexpected results

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Returns single value
2) Works on numbers, strings, dates
3) Ignores NULL values
4) Useful for finding highest values

======================================================================
----------------------------------------------------------------------
IMPORTANT BEHAVIOR
----------------------------------------------------------------------

• NULL values are ignored (except COUNT(*))
• Works on multiple rows → returns single value

----------------------------------------------------------------------
USING WITH WHERE
----------------------------------------------------------------------
SELECT AVG(age)
FROM student
WHERE city = 'Delhi';

→ Average age of Delhi students

----------------------------------------------------------------------
GROUP BY (VERY IMPORTANT)
----------------------------------------------------------------------
Used to apply aggregate per group

SELECT city, COUNT(*)
FROM student
GROUP BY city;

→ Count students per city

----------------------------------------------------------------------
GROUP BY WITH MULTIPLE COLUMNS
----------------------------------------------------------------------
SELECT city, age, COUNT(*)
FROM student
GROUP BY city, age;

----------------------------------------------------------------------
HAVING (FILTER GROUPS)
----------------------------------------------------------------------
Used after GROUP BY

SELECT city, COUNT(*)
FROM student
GROUP BY city
HAVING COUNT(*) > 2;

→ Cities having more than 2 students

----------------------------------------------------------------------
WHERE vs HAVING
----------------------------------------------------------------------
WHERE  → filters rows before grouping
HAVING → filters after grouping

----------------------------------------------------------------------
AGGREGATE + ORDER BY
----------------------------------------------------------------------
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
ORDER BY total DESC;

----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------

❌ SELECT city, COUNT(*) FROM student;
→ Error (city must be in GROUP BY)

✔ Correct:
SELECT city, COUNT(*) FROM student GROUP BY city;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Aggregate = many rows → one result
2) NULL ignored (except COUNT(*))
3) GROUP BY is very important
4) HAVING used with aggregate conditions
5) Used in reports, analytics

======================================================================

 GROUP BY 
======================================================================

DEFINITION:
GROUP BY is used to group rows having same values in one or more
columns and apply aggregate functions on each group.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column_name, AGG_FUNCTION(column)
FROM table_name
GROUP BY column_name;

----------------------------------------------------------------------
WHY GROUP BY IS USED
----------------------------------------------------------------------
• To perform aggregation per category
• To summarize data (count, avg, sum, etc.)

----------------------------------------------------------------------
BASIC EXAMPLE
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
GROUP BY city;

→ Output:
Delhi    3
Mumbai   2

→ Rows grouped by city


======================================================================
GROUP BY EXAMPLES 
======================================================================

ASSUME TABLE: employees
----------------------------------------------------------------------
id | name  | department | salary
------------------------------------------
1  | A     | IT         | 50000
2  | B     | HR         | 40000
3  | C     | IT         | 60000
4  | D     | HR         | 45000
5  | E     | SALES      | 30000

----------------------------------------------------------------------
1) SELECT department FROM employees GROUP BY department;
----------------------------------------------------------------------

SELECT department FROM employees
GROUP BY department;

→ OUTPUT:
IT
HR
SALES

→ Returns unique departments (similar to DISTINCT)

----------------------------------------------------------------------
2) COUNT PER DEPARTMENT
----------------------------------------------------------------------

SELECT department, COUNT(*) AS total_emps
FROM employees
GROUP BY department;

→ OUTPUT:
IT      2
HR      2
SALES   1

→ Counts employees in each department

----------------------------------------------------------------------
3) SUM OF SALARY PER DEPARTMENT
----------------------------------------------------------------------

SELECT department, SUM(salary) AS total_salary
FROM employees
GROUP BY department;

→ OUTPUT:
IT      110000
HR      85000
SALES   30000

→ Total salary per department

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) GROUP BY creates one row per group
2) COUNT(*) counts rows in each group
3) SUM() adds values per group
4) Output order may vary unless ORDER BY used

======================================================================
----------------------------------------------------------------------
MULTIPLE COLUMNS
----------------------------------------------------------------------
SELECT city, age, COUNT(*)
FROM student
GROUP BY city, age;

→ Groups based on combination (city + age)

----------------------------------------------------------------------
WITH AVG / SUM / MIN / MAX
----------------------------------------------------------------------
SELECT city, AVG(age)
FROM student
GROUP BY city;

→ Average age per city

----------------------------------------------------------------------
GROUP BY WITH WHERE
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
WHERE age > 20
GROUP BY city;

→ WHERE filters rows BEFORE grouping

----------------------------------------------------------------------
GROUP BY WITH HAVING
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
GROUP BY city
HAVING COUNT(*) > 2;

→ HAVING filters AFTER grouping

----------------------------------------------------------------------
WHERE vs HAVING
----------------------------------------------------------------------
WHERE  → row-level filter (before grouping)
HAVING → group-level filter (after grouping)

----------------------------------------------------------------------
ORDER BY WITH GROUP BY
----------------------------------------------------------------------
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
ORDER BY total DESC;

→ Sort grouped result

----------------------------------------------------------------------
IMPORTANT RULE
----------------------------------------------------------------------
All selected columns must be:
✔ In GROUP BY
OR
✔ Used with aggregate function

❌ WRONG:
SELECT city, age FROM student GROUP BY city;

→ Error (age not grouped or aggregated)

----------------------------------------------------------------------
GROUP BY WITH DISTINCT (RELATION)
----------------------------------------------------------------------
SELECT DISTINCT city FROM student;

≈ similar to:

SELECT city FROM student GROUP BY city;

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------

1) NULL VALUES
----------------------------------------------------------------------
SELECT city, COUNT(*)
FROM student
GROUP BY city;

→ NULL treated as separate group

----------------------------------------------------------------------
2) NO ROWS
----------------------------------------------------------------------
→ Returns EMPTY result

----------------------------------------------------------------------
3) GROUP BY WITHOUT AGGREGATE
----------------------------------------------------------------------
SELECT city FROM student GROUP BY city;

→ Returns unique cities

----------------------------------------------------------------------
4) GROUP BY WITH EXPRESSIONS
----------------------------------------------------------------------
SELECT YEAR(order_date), COUNT(*)
FROM orders
GROUP BY YEAR(order_date);

→ Grouping by derived value

----------------------------------------------------------------------
5) GROUP BY WITH JOIN
----------------------------------------------------------------------
SELECT s.city, COUNT(o.id)
FROM student s
JOIN orders o ON s.id = o.student_id
GROUP BY s.city;

→ Aggregation after join

----------------------------------------------------------------------
6) LARGE DATA PERFORMANCE
----------------------------------------------------------------------
• GROUP BY can be slow on large data
• Use indexes for better performance

----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------

❌ Missing GROUP BY column
❌ Using WHERE instead of HAVING
❌ Confusing DISTINCT with GROUP BY
❌ Expecting row-level data in grouped result

----------------------------------------------------------------------
EXECUTION ORDER
----------------------------------------------------------------------
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Groups rows into categories
2) Used with aggregate functions
3) HAVING is used with GROUP BY
4) NULL forms its own group
5) Essential for reports and analytics

======================================================================
example:including multiple clauses
SELECT 
    department,
    COUNT(*)      AS total_emps,
    SUM(salary)   AS total_salary,
    AVG(salary)   AS avg_salary,
    MIN(salary)   AS min_salary,
    MAX(salary)   AS max_salary
FROM employees
GROUP BY department;

======================================================================
HAVING CLAUSE 
======================================================================

DEFINITION:
HAVING is used to filter grouped results (after GROUP BY).
It works with aggregate functions.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column_name, AGG_FUNCTION(column)
FROM table_name
GROUP BY column_name
HAVING condition;

----------------------------------------------------------------------
WHY HAVING IS USED
----------------------------------------------------------------------
• WHERE cannot filter aggregate results
• HAVING filters groups after aggregation

----------------------------------------------------------------------
BASIC EXAMPLE
----------------------------------------------------------------------
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
HAVING COUNT(*) > 2;

→ Returns cities having more than 2 students

SELECT product, region, SUM(quantity) as total
FROM transactions
GROUP BY product, region
HAVING SUM (quantity) >5;
----------------------------------------------------------------------
WITH MULTIPLE CONDITIONS
----------------------------------------------------------------------
SELECT city, AVG(age) AS avg_age
FROM student
GROUP BY city
HAVING AVG(age) > 20 AND COUNT(*) >= 2;

→ Filters groups based on multiple conditions

----------------------------------------------------------------------
HAVING WITHOUT GROUP BY
----------------------------------------------------------------------
SELECT COUNT(*) 
FROM student
HAVING COUNT(*) > 5;

→ Works, treats entire table as one group

----------------------------------------------------------------------
HAVING vs WHERE
----------------------------------------------------------------------
WHERE  → filters rows BEFORE grouping
HAVING → filters groups AFTER grouping

Example:
SELECT city, COUNT(*)
FROM student
WHERE age > 20
GROUP BY city
HAVING COUNT(*) > 2;

→ WHERE filters rows first
→ GROUP BY groups remaining rows
→ HAVING filters final groups

----------------------------------------------------------------------
USING ALIAS IN HAVING
----------------------------------------------------------------------
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
HAVING total > 2;

→ Alias can be used in HAVING

----------------------------------------------------------------------
WITH ORDER BY
----------------------------------------------------------------------
SELECT city, COUNT(*) AS total
FROM student
GROUP BY city
HAVING total > 2
ORDER BY total DESC;

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------

1) NO GROUPS MATCH
----------------------------------------------------------------------
→ Returns EMPTY result

----------------------------------------------------------------------
2) NULL VALUES
----------------------------------------------------------------------
• NULL values ignored in aggregates
• Groups still formed normally

----------------------------------------------------------------------
3) WITHOUT AGGREGATE (RARE)
----------------------------------------------------------------------
SELECT city FROM student
GROUP BY city
HAVING city = 'Delhi';

→ Works but usually use WHERE instead

----------------------------------------------------------------------
4) USING MULTIPLE AGGREGATES
----------------------------------------------------------------------
SELECT city, SUM(age), AVG(age)
FROM student
GROUP BY city
HAVING SUM(age) > 100;

----------------------------------------------------------------------
COMMON MISTAKES
----------------------------------------------------------------------

❌ Using WHERE with aggregate
SELECT city FROM student WHERE COUNT(*) > 2;
→ ERROR

✔ Correct:
Use HAVING

❌ Confusing WHERE and HAVING

----------------------------------------------------------------------
EXECUTION ORDER
----------------------------------------------------------------------
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) HAVING works after GROUP BY
2) Used with aggregate conditions
3) Can use alias
4) Important for filtering grouped data
5) Common in reports and analytics


======================================================================
MYSQL DATE DATA TYPE curdate, date_format, adddate, subdate, datediff, datetime, now, 
======================================================================

DEFINITION:
DATE is used to store only date values (no time).

----------------------------------------------------------------------
FORMAT
----------------------------------------------------------------------
'YYYY-MM-DD'

Example:
'2024-03-20'

----------------------------------------------------------------------
CREATE TABLE WITH DATE
----------------------------------------------------------------------
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_date DATE
);

----------------------------------------------------------------------
INSERT DATE VALUES
----------------------------------------------------------------------
INSERT INTO orders (order_date)
VALUES ('2024-01-10'),
       ('2024-02-15'),
       ('2024-03-20');

----------------------------------------------------------------------
INVALID / WRONG FORMAT
----------------------------------------------------------------------
'10-01-2024' → Wrong format ❌
'2024/01/10' → May fail / convert ❌

✔ Always use: 'YYYY-MM-DD'

----------------------------------------------------------------------
DATE RANGE
----------------------------------------------------------------------
Supported range:
'1000-01-01' to '9999-12-31'

----------------------------------------------------------------------
DATE COMPARISON
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date > '2024-02-01';

→ Dates compared chronologically

----------------------------------------------------------------------
COMMON DATE FUNCTIONS
----------------------------------------------------------------------

1) CURDATE()
----------------------------------------------------------------------
SELECT CURDATE();
→ Current date

----------------------------------------------------------------------
2) NOW()
----------------------------------------------------------------------
SELECT NOW();
→ Current date + time

----------------------------------------------------------------------
3) DATE()
----------------------------------------------------------------------
SELECT DATE('2024-03-20 10:30:00');
→ Extracts date part

----------------------------------------------------------------------
4) YEAR(), MONTH(), DAY()
----------------------------------------------------------------------
SELECT YEAR(order_date) FROM orders;
SELECT MONTH(order_date) FROM orders;
SELECT DAY(order_date) FROM orders;

----------------------------------------------------------------------
----------------------------------------------------------------------
5) DATEDIFF()
----------------------------------------------------------------------
SELECT DATEDIFF('2024-03-20', '2024-03-10');
→ 10 days difference

• Returns difference in DAYS only
• Ignores time part (if present)

----------------------------------------------------------------------
5.1) TIMEDIFF()
----------------------------------------------------------------------
SELECT TIMEDIFF('10:30:00', '08:15:00');
→ 02:15:00

• Returns difference in TIME (HH:MM:SS)

----------------------------------------------------------------------
WITH DATETIME
----------------------------------------------------------------------
SELECT TIMEDIFF('2024-03-20 12:00:00', '2024-03-20 10:30:00');
→ 01:30:00

• Works when both values have same date

----------------------------------------------------------------------
IMPORTANT POINTS
----------------------------------------------------------------------
1) DATEDIFF → returns number (days)
2) TIMEDIFF → returns time format
3) TIMEDIFF considers only time part
4) If dates differ → still calculates full time difference

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
• Use DATEDIFF for date comparison
• Use TIMEDIFF for time difference
• Both are very common in real-world queries
TIME range:
-838:59:59 to 838:59:59

→ Large differences may overflow ❌

======================================================================
----------------------------------------------------------------------
6) ADDDATE()
----------------------------------------------------------------------
SELECT ADDDATE('2024-03-20', INTERVAL 5 DAY);
→ '2024-03-25'

SELECT adddate("2026-03-22 09:15:15", 8);
• SELECT addtime("2026-03-22 09:15:15", "01:20:00");
• SELECT addtime("2026-03-22 23:15:15", "01:20:00");

Summary of the logic:
Line 8: Uses adddate to add 8 days to the specified date. The result will be 2026-03-30 09:15:15.

Line 9: Uses addtime to add 1 hour and 20 minutes to the timestamp. The result will be 2026-03-22 10:35:15.

Line 10: Uses addtime on a late-night timestamp. Adding 1 hour and 20 minutes will cross into the next day, resulting in 2026-03-23 00:35:15.
----------------------------------------------------------------------
7) SUBDATE()
----------------------------------------------------------------------
SELECT SUBDATE('2024-03-20', INTERVAL 5 DAY);
→ '2024-03-15'

----------------------------------------------------------------------
8) STR_TO_DATE()
----------------------------------------------------------------------
SELECT STR_TO_DATE('20-03-2024', '%d-%m-%Y');
→ Converts string to DATE

----------------------------------------------------------------------
9) DATE_FORMAT()
----------------------------------------------------------------------
SELECT DATE_FORMAT('2024-03-20', '%d-%m-%Y');
→ '20-03-2024'

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• NULL allowed unless NOT NULL constraint
• NULL ignored in comparisons

----------------------------------------------------------------------
IMPORTANT RULES
----------------------------------------------------------------------
1) Always use single quotes
2) Format must be YYYY-MM-DD
3) Compared as dates, not strings
4) Use functions for extraction/manipulation

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------

1) INVALID DATE
----------------------------------------------------------------------
'2024-02-30' → Error or invalid

----------------------------------------------------------------------
2) NO QUOTES
----------------------------------------------------------------------
2024-03-20 → treated as arithmetic ❌

----------------------------------------------------------------------
3) DATE vs DATETIME
----------------------------------------------------------------------
DATE = '2024-03-20'
DATETIME = '2024-03-20 10:30:00'

→ Time matters in comparison

SELECT DATE_FORMAT('2024-03-20 10:30:45', '%d-%m-%Y %H:%i:%s');

→ OUTPUT:
20-03-2024 10:30:45
----------------------------------------------------------------------
4) TIMEZONE
----------------------------------------------------------------------
DATE has no timezone effect (unlike TIMESTAMP)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Stores only date (no time)
2) Format is fixed
3) Widely used in filtering, reports
4) Combine with functions for flexibility

MYSQL TIMESTAMP
======================================================================

DEFINITION:
TIMESTAMP stores date and time values.
It is often used to track record creation and updates.

----------------------------------------------------------------------
FORMAT
----------------------------------------------------------------------
'YYYY-MM-DD HH:MM:SS'

Example:
'2024-03-20 10:30:45'

----------------------------------------------------------------------
CREATE TABLE WITH TIMESTAMP
----------------------------------------------------------------------
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    created_at TIMESTAMP
);

----------------------------------------------------------------------
AUTO TIMESTAMP (DEFAULT CURRENT TIME)
----------------------------------------------------------------------
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

→ Automatically stores current date & time

----------------------------------------------------------------------
AUTO UPDATE TIMESTAMP
----------------------------------------------------------------------
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP 
        DEFAULT CURRENT_TIMESTAMP 
        ON UPDATE CURRENT_TIMESTAMP
);

→ updated_at auto-updates on row change


CREATE TABLE user_logins(
login_id INT primary key auto_increment,
user_id INT,
login_time timestamp default current_timestamp,
logout_time timestamp
);

----------------------------------------------------------------------
INSERT VALUES
----------------------------------------------------------------------
INSERT INTO orders (created_at)
VALUES ('2024-03-20 10:30:45');

----------------------------------------------------------------------
TIMESTAMP vs DATETIME
----------------------------------------------------------------------

TIMESTAMP:
• Stored in UTC, converted to session timezone
• Auto-update possible, will convert to the user zone like india automatically when trying to read.
• Smaller range

DATETIME:
• Stored as given (no timezone conversion)
• No auto-update by default
• Larger range

----------------------------------------------------------------------
RANGE
----------------------------------------------------------------------
TIMESTAMP:
'1970-01-01 00:00:01' to '2038-01-19 03:14:07'

----------------------------------------------------------------------
COMMON FUNCTIONS
----------------------------------------------------------------------

SELECT NOW();
→ Current date & time

SELECT CURRENT_TIMESTAMP;
→ Same as NOW()

----------------------------------------------------------------------
COMPARISON
----------------------------------------------------------------------
SELECT * FROM orders
WHERE created_at > '2024-03-01 00:00:00';

----------------------------------------------------------------------
EXTRACT PARTS
----------------------------------------------------------------------
SELECT YEAR(created_at) FROM orders;
SELECT HOUR(created_at) FROM orders;

----------------------------------------------------------------------
NULL BEHAVIOR
----------------------------------------------------------------------
• Can allow NULL if defined
• Otherwise defaults may apply

----------------------------------------------------------------------
EDGE CASES
----------------------------------------------------------------------

1) TIMEZONE EFFECT
----------------------------------------------------------------------
• Value changes based on system timezone

----------------------------------------------------------------------
2) 2038 PROBLEM
----------------------------------------------------------------------
• Limited range (up to year 2038)

----------------------------------------------------------------------
3) AUTO UPDATE LIMITATION
----------------------------------------------------------------------
• Only columns defined with ON UPDATE auto-change

----------------------------------------------------------------------
4) INVALID FORMAT
----------------------------------------------------------------------
Wrong format → error or conversion

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Used for tracking (created_at, updated_at)
2) Auto timestamp feature is very useful
3) Timezone-aware
4) Prefer DATETIME for long-term data
