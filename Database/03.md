MYSQL DEFAULT CONSTRAINT
======================================================================

DEFINITION:
DEFAULT constraint assigns a default value to a column 
when no value is provided during INSERT.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
CREATE TABLE table_name (
    column_name datatype DEFAULT default_value
);

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    city VARCHAR(50) DEFAULT 'Delhi'
);

INSERT INTO student (id, name)
VALUES (1, 'Ajay');

OUTPUT:
id | name | city
---------------------
1  | Ajay | Delhi   ← default applied

----------------------------------------------------------------------
DEFAULT WITH INSERT
----------------------------------------------------------------------
INSERT INTO student (id, name, city)
VALUES (2, 'Rahul', 'Mumbai');

→ Given value overrides default

----------------------------------------------------------------------
USING DEFAULT KEYWORD
----------------------------------------------------------------------
INSERT INTO student (id, name, city)
VALUES (3, 'Amit', DEFAULT);

→ Uses default value explicitly

----------------------------------------------------------------------
ALTER TABLE (ADD DEFAULT)
----------------------------------------------------------------------
ALTER TABLE student
ALTER city SET DEFAULT 'Bangalore';

----------------------------------------------------------------------
REMOVE DEFAULT
----------------------------------------------------------------------
ALTER TABLE student
ALTER city DROP DEFAULT;
----------------------------------------------
cases:
it will prefer any value over default, even null value
----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Works only when value is not provided
2) DEFAULT ≠ NOT NULL (can still be NULL unless restricted)
3) Can be used with numbers, strings, dates
4) Improves data consistency
5) Useful for common values (city, status, date)

MYSQL AUTO_INCREMENT
======================================================================

DEFINITION:
AUTO_INCREMENT automatically generates a unique number 
for a column (usually used for PRIMARY KEY).

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
CREATE TABLE table_name (
    id INT AUTO_INCREMENT,
    column2 datatype,
    PRIMARY KEY (id)
);

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

INSERT INTO student (name, age)
VALUES ('Ajay', 22),
       ('Rahul', 23),
       ('Amit', 21);

OUTPUT:
id | name  | age
---------------------
1  | Ajay  | 22
2  | Rahul | 23
3  | Amit  | 21

----------------------------------------------------------------------
INSERT WITH AUTO_INCREMENT
----------------------------------------------------------------------
INSERT INTO student (name, age)
VALUES ('Neha', 20);

→ id auto-generated (next value = 4)

Case:
INSERT INTO student (id,name, age)
VALUES (10,'ajay', 20);
-now id will start from 10 and increment from 10 for next values.
----------------------------------------------------------------------
SET START VALUE
----------------------------------------------------------------------
ALTER TABLE student AUTO_INCREMENT = 100;

→ Next id will start from 100

----------------------------------------------------------------------
IMPORTANT RULES
----------------------------------------------------------------------
1) Only one AUTO_INCREMENT column per table
2) Must be PRIMARY KEY or UNIQUE
3) Value increases automatically (1,2,3...)
4) Cannot insert duplicate values
5) Skipped numbers are NOT reused (if row deleted)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Mostly used for ID columns
2) Saves manual work
3) Ensures uniqueness automatically
4) Works with INT, BIGINT

MYSQL COMPARISON OPERATORS
======================================================================

DEFINITION:
Comparison operators are used to compare values in conditions
(mainly used with WHERE clause).

----------------------------------------------------------------------
LIST OF OPERATORS
----------------------------------------------------------------------

=        Equal to
!=       Not equal to
<>       Not equal to (same as !=)
>        Greater than
<        Less than
>=       Greater than or equal to
<=       Less than or equal to

----------------------------------------------------------------------
EXAMPLES
----------------------------------------------------------------------

SELECT * FROM student WHERE age = 22;
→ age equal to 22

SELECT * FROM student WHERE age != 22;
→ age not equal to 22

SELECT * FROM student WHERE age > 20;
→ age greater than 20

SELECT * FROM student WHERE age < 25;
→ age less than 25

SELECT * FROM student WHERE age >= 21;
→ age ≥ 21

SELECT * FROM student WHERE age <= 23;
→ age ≤ 23

----------------------------------------------------------------------
OTHER RELATED COMPARISONS
----------------------------------------------------------------------

BETWEEN
SELECT * FROM student WHERE age BETWEEN 18 AND 22;
→ range (inclusive)

IN
SELECT * FROM student WHERE city IN ('Delhi', 'Mumbai');
→ matches any value in list

LIKE
SELECT * FROM student WHERE name LIKE 'A%';
→ pattern matching

IS NULL
SELECT * FROM student WHERE age IS NULL;
→ checks NULL values
---------------------------------------

CASES:
----------------------------------------------------------------------

1) RESULT CAN BE TRUE / FALSE / NULL
----------------------------------------------------------------------
• SQL uses 3-valued logic → TRUE, FALSE, NULL (UNKNOWN)
• If any operand is NULL → result is usually NULL

Example:
SELECT 5 = NULL;      → NULL
SELECT NULL > 10;     → NULL

----------------------------------------------------------------------

2) "=" DOES NOT WORK WITH NULL
----------------------------------------------------------------------
• NULL cannot be compared using = or !=

WRONG:
SELECT * FROM student WHERE age = NULL;

RIGHT:
SELECT * FROM student WHERE age IS NULL;
SELECT * FROM student WHERE age IS NOT NULL;

----------------------------------------------------------------------

3) SPACES MATTER IN STRING COMPARISON
----------------------------------------------------------------------
• Leading/trailing spaces affect result

SELECT 'Ajay' = 'Ajay ';   → FALSE

• Exception:
CHAR type may ignore trailing spaces (depends on config)

----------------------------------------------------------------------

4) STRING COMPARISON IS LEXICOGRAPHICAL
----------------------------------------------------------------------
• Compared like dictionary order (based on collation)

SELECT 'apple' < 'banana';   → TRUE
SELECT 'Zebra' < 'apple';    → depends on collation (case-sensitive or not)

----------------------------------------------------------------------

5) NO MATCH ≠ NULL RESULT
----------------------------------------------------------------------
• Comparison does NOT return NULL if no rows match
• It simply returns NO ROWS

Example:
SELECT * FROM student WHERE age < 10;

→ If no student satisfies → EMPTY RESULT (not NULL)

----------------------------------------------------------------------

6) NULL VALUES ARE SKIPPED IN COMPARISON
----------------------------------------------------------------------
• Rows with NULL are NOT included in conditions like:

SELECT * FROM student WHERE age < 30;

→ Rows where age = NULL are ignored

----------------------------------------------------------------------

7) USE <=> FOR NULL-SAFE COMPARISON (IMPORTANT)
----------------------------------------------------------------------
• Special operator to compare NULL safely

SELECT NULL <=> NULL;   → TRUE
SELECT 5 <=> NULL;      → FALSE

----------------------------------------------------------------------

NOTES
----------------------------------------------------------------------
1) NULL means “unknown”, not 0 or empty
2) Always use IS NULL / IS NOT NULL
3) Comparisons with NULL → result becomes NULL
4) No rows returned ≠ NULL value
----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Used inside WHERE clause
2) Returns TRUE / FALSE
3) Can be combined using AND / OR
4) <> and != both mean NOT EQUAL

----------------------------------------------------------------------
CREATE TABLE orders (

SELECT * FROM orders;

COMPARISON OPERATORS ON DATE
----------------------------------------------------------------------

1) EQUAL (=)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date = '2024-03-20';
→ Returns Order3 only

----------------------------------------------------------------------
2) GREATER THAN (>)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date > '2024-02-01';
→ Returns Order2, Order3, Order4

----------------------------------------------------------------------
3) LESS THAN (<)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date < '2024-03-01';
→ Returns Order1, Order2

----------------------------------------------------------------------
4) GREATER THAN OR EQUAL (>=)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date >= '2024-03-20';
→ Returns Order3, Order4

----------------------------------------------------------------------
5) LESS THAN OR EQUAL (<=)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date <= '2024-02-15';
→ Returns Order1, Order2

----------------------------------------------------------------------
6) BETWEEN (RANGE)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date BETWEEN '2024-02-01' AND '2024-03-20';

7) NOT EQUAL (!=)
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date != '2024-03-20';
→ Returns Order1, Order2, Order4 (NULL ignored)

----------------------------------------------------------------------
8) NULL CHECK
----------------------------------------------------------------------
SELECT * FROM orders
WHERE order_date IS NULL;
→ Returns Order5 only

MYSQL DATE / DATETIME — CASES & CORRECTIONS
======================================================================

CASES:
----------------------------------------------------------------------

1) WRONG DATE FORMAT
----------------------------------------------------------------------
• MySQL expects: 'YYYY-MM-DD'

SELECT * FROM orders WHERE order_date = '10-01-2024';
→ May return EMPTY or incorrect result (format mismatch)

✔ Correct:
SELECT * FROM orders WHERE order_date = '2024-01-10';

----------------------------------------------------------------------

2) QUOTES ARE IMPORTANT
----------------------------------------------------------------------
• Dates must be in 'single quotes'

WRONG:
SELECT * FROM orders WHERE order_date = 2024-01-10;
→ Interpreted as arithmetic → (2024 - 01 - 10 = 2013)
→ No match / wrong result

✔ Correct:
SELECT * FROM orders WHERE order_date = '2024-01-10';

----------------------------------------------------------------------

3) INVALID DATE VALUE
----------------------------------------------------------------------
SELECT * FROM orders WHERE order_date = '2024-02-30';
→ Invalid date → may return EMPTY or error (depends on SQL mode)

----------------------------------------------------------------------

ASSIGNMENT: DATE vs DATETIME COMPARISON
----------------------------------------------------------------------

CREATE TABLE events (
    id INT AUTO_INCREMENT PRIMARY KEY,
    event_date DATE,
    event_datetime DATETIME
);

----------------------------------------------------------------------
INSERT VALUES
----------------------------------------------------------------------
INSERT INTO events (event_date, event_datetime)
VALUES
('2024-03-20', '2024-03-20 10:30:00'),
('2024-03-21', '2024-03-21 15:45:00'),
('2024-03-22', '2024-03-22 00:00:00');

----------------------------------------------------------------------
COMPARISON CASES
----------------------------------------------------------------------

1) DATE = DATE
----------------------------------------------------------------------
SELECT * FROM events
WHERE event_date = '2024-03-20';
→ Returns row 1

----------------------------------------------------------------------
2) DATETIME = DATETIME
----------------------------------------------------------------------
SELECT * FROM events
WHERE event_datetime = '2024-03-20 10:30:00';
→ Returns row 1

----------------------------------------------------------------------
3) DATE vs DATETIME (IMPORTANT)
----------------------------------------------------------------------
SELECT * FROM events
WHERE event_datetime = '2024-03-20';
→ Returns EMPTY (time part missing)

Reason:
'2024-03-20' → treated as '2024-03-20 00:00:00'
Does not match '10:30:00'

----------------------------------------------------------------------
4) DATE COMPARISON WITH DATETIME USING RANGE
----------------------------------------------------------------------
SELECT * FROM events
WHERE event_datetime >= '2024-03-20'
AND event_datetime < '2024-03-21';
→ Returns all rows on 20th March

----------------------------------------------------------------------
5) USING DATE() FUNCTION
----------------------------------------------------------------------
SELECT * FROM events
WHERE DATE(event_datetime) = '2024-03-20';
→ Returns row 1 (ignores time part)

----------------------------------------------------------------------
6) GREATER THAN COMPARISON
----------------------------------------------------------------------
SELECT * FROM events
WHERE event_datetime > '2024-03-20';
→ Returns all rows after 20th midnight

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) DATE → only date (YYYY-MM-DD)
2) DATETIME → date + time (YYYY-MM-DD HH:MM:SS)
3) Comparing DATE with DATETIME → time matters
4) Use DATE() to ignore time part
5) Always use proper format + quotes

======================================================================
----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) DATE format → 'YYYY-MM-DD'
2) Dates compared chronologically
3) NULL rows are ignored in comparisons
4) BETWEEN includes both start & end
5) Use IS NULL for NULL values

MYSQL LOGICAL OPERATORS
======================================================================

DEFINITION:
Logical operators are used to combine multiple conditions
in a query (mainly with WHERE clause).

----------------------------------------------------------------------
TYPES OF LOGICAL OPERATORS
----------------------------------------------------------------------

1) AND
2) OR
3) NOT
4) XOR

----------------------------------------------------------------------
1) AND OPERATOR
----------------------------------------------------------------------
• All conditions must be TRUE

SYNTAX:
SELECT * FROM table_name
WHERE condition1 AND condition2;

EXAMPLE:
SELECT * FROM student
WHERE age > 20 AND city = 'Delhi';

→ Returns rows where BOTH conditions are true

CASE:
SELECT * FROM people
WHERE age<30
AND location = "USA"
AND name = "SAM";

here all three conditions should be true and flow will be from first condition to second and then third.

----------------------------------------------------------------------
2) OR OPERATOR
----------------------------------------------------------------------
• At least one condition must be TRUE

SYNTAX:
SELECT * FROM table_name
WHERE condition1 OR condition2;

EXAMPLE:
SELECT * FROM student
WHERE age < 20 OR city = 'Mumbai';

→ Returns rows where ANY one condition is true

----------------------------------------------------------------------
3) NOT OPERATOR
----------------------------------------------------------------------
• Reverses the condition

SYNTAX:
SELECT * FROM table_name
WHERE NOT condition;

EXAMPLE:
SELECT * FROM student
WHERE NOT city = 'Delhi';

→ Returns rows where city is NOT Delhi

----------------------------------------------------------------------
4) XOR OPERATOR
----------------------------------------------------------------------
• TRUE if ONLY ONE condition is TRUE
• FALSE if both are TRUE or both are FALSE

SYNTAX:
SELECT * FROM table_name
WHERE condition1 XOR condition2;

EXAMPLE:
SELECT * FROM student
WHERE age > 20 XOR city = 'Delhi';

→ Returns rows where ONLY one condition is true

----------------------------------------------------------------------
XOR TRUTH TABLE
----------------------------------------------------------------------
Condition1   Condition2   Result
--------------------------------
TRUE         TRUE         FALSE
TRUE         FALSE        TRUE
FALSE        TRUE         TRUE
FALSE        FALSE        FALSE

----------------------------------------------------------------------
OPERATOR PRECEDENCE
----------------------------------------------------------------------
1) NOT
2) AND
3) XOR
4) OR

----------------------------------------------------------------------
NULL BEHAVIOR (IMPORTANT)
----------------------------------------------------------------------
• TRUE AND NULL   → NULL
• FALSE AND NULL  → FALSE
• TRUE OR NULL    → TRUE
• FALSE OR NULL   → NULL
• TRUE XOR NULL   → NULL
• FALSE XOR NULL  → NULL

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) XOR is less commonly used but useful for exclusive conditions
2) Use brackets () for clarity
3) Works with comparison operators
4) NULL affects logical results

AND + OR TOGETHER (WITHOUT PARENTHESES)
======================================================================

IMPORTANT RULE:
----------------------------------------------------------------------
AND has higher priority than OR

Execution order:
→ AND runs first
→ then OR

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------

SELECT * FROM student
WHERE age > 20 OR city = 'Delhi' AND marks > 70;

----------------------------------------------------------------------
ACTUAL EXECUTION (INTERNALLY)
----------------------------------------------------------------------

WHERE age > 20 
   OR (city = 'Delhi' AND marks > 70);

----------------------------------------------------------------------
UNDERSTANDING RESULT
----------------------------------------------------------------------

• First: (city = 'Delhi' AND marks > 70) evaluated
• Then: age > 20 OR result

→ Rows satisfying ANY:
   1) age > 20
   OR
   2) BOTH (city = Delhi AND marks > 70)

----------------------------------------------------------------------
ANOTHER EXAMPLE
----------------------------------------------------------------------

SELECT * FROM student
WHERE age < 18 OR age > 60 AND city = 'Mumbai';

ACTUAL:
WHERE age < 18 
   OR (age > 60 AND city = 'Mumbai');

----------------------------------------------------------------------
COMMON MISTAKE
----------------------------------------------------------------------

Developer expects:
(age < 18 OR age > 60) AND city = 'Mumbai'

BUT MySQL does:
age < 18 OR (age > 60 AND city = 'Mumbai')

→ Different results ⚠️

----------------------------------------------------------------------
BEST PRACTICE
----------------------------------------------------------------------

Always use parentheses for clarity:

SELECT * FROM student
WHERE (age < 18 OR age > 60)
AND city = 'Mumbai';

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) AND > OR in priority
2) Without () → MySQL decides execution
3) Can lead to unexpected results
4) Always use () in real projects
