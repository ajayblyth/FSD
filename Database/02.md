MYSQL DATA TYPES
======================================================================

CATEGORY        DATA TYPE        SIZE (BYTES)        MEANING
----------------------------------------------------------------------

----------------------- NUMERIC TYPES -------------------------------
INTEGER         TINYINT          1                   Very small integer (-128 to 127)
INTEGER         SMALLINT         2                   Small integer
INTEGER         MEDIUMINT        3                   Medium integer
INTEGER         INT              4                   Standard integer
INTEGER         BIGINT           8                   Large integer

DECIMAL         DECIMAL(M,D)     Variable            Exact fixed-point number
FLOATING        FLOAT            4                   Approx decimal (less precision)
FLOATING        DOUBLE           8                   High precision decimal

----------------------- STRING TYPES --------------------------------
STRING          CHAR(n)          Fixed               Fixed-length string, length should equal to as mentioned.e.g 3 means exact 3.
STRING          VARCHAR(n)       Variable            Variable-length string, length can be anywhere till the mentioned length, e.g 0-10,

TEXT            TINYTEXT         255 chars           Small text
TEXT            TEXT             65,535 chars        Normal text
TEXT            MEDIUMTEXT       16 MB               Large text
TEXT            LONGTEXT         4 GB                Very large text

----------------------- DATE & TIME TYPES ---------------------------
DATE            DATE             3                   YYYY-MM-DD
TIME            TIME             3                   HH:MM:SS
DATETIME        DATETIME         8                   Date + time
TIMESTAMP       TIMESTAMP        4                   Auto-stored current time
YEAR            YEAR             1                   Year (YYYY)

----------------------- BINARY TYPES --------------------------------
BINARY          BINARY(n)        Fixed               Fixed binary data
BINARY          VARBINARY(n)     Variable            Variable binary data

BLOB            TINYBLOB         255 bytes           Small binary object
BLOB            BLOB             65 KB               Binary data
BLOB            MEDIUMBLOB       16 MB               Large binary data
BLOB            LONGBLOB         4 GB                Very large binary

----------------------- BOOLEAN -------------------------------------
BOOLEAN         BOOLEAN          1                   TRUE (1) / FALSE (0)

======================================================================
NOTES
======================================================================
1) INT is most commonly used for numbers
2) VARCHAR is most commonly used for strings
3) DECIMAL is used for money (exact values)
4) TIMESTAMP auto-updates (used in tracking records)
5) TEXT/BLOB used for large data (avoid unless needed)

MYSQL CREATE TABLE
======================================================================

DEFINITION:
CREATE TABLE is used to create a new table in the database.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype
);

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    age INT
);

----------------------------------------------------------------------
CREATE TABLE WITH CONSTRAINTS
----------------------------------------------------------------------
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    age INT CHECK (age >= 18),
    email VARCHAR(100) UNIQUE
);

----------------------------------------------------------------------
CREATE TABLE WITH AUTO_INCREMENT
----------------------------------------------------------------------
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

----------------------------------------------------------------------
CREATE TABLE WITH DEFAULT VALUE
----------------------------------------------------------------------
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    city VARCHAR(50) DEFAULT 'Delhi'
);

----------------------------------------------------------------------
CREATE TABLE WITH FOREIGN KEY
----------------------------------------------------------------------
CREATE TABLE course (
    course_id INT PRIMARY KEY
);

CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    course_id INT,
    FOREIGN KEY (course_id) REFERENCES course(course_id)
);

----------------------------------------------------------------------
CREATE TABLE FROM EXISTING TABLE
----------------------------------------------------------------------
CREATE TABLE new_table AS
SELECT * FROM old_table;

----------------------------------------------------------------------
SHOW TABLE STRUCTURE
----------------------------------------------------------------------
DESCRIBE student;

OR

DESC student;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Table name should be unique in database
2) Column names should be meaningful
3) Always define PRIMARY KEY
4) Use constraints for better data integrity
5) VARCHAR is most commonly used for text

MYSQL SELECT COMMAND (COMPLETE NOTES)
======================================================================

DEFINITION:
SELECT is used to retrieve (fetch) data from a table.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column1, column2
FROM table_name;

----------------------------------------------------------------------
SELECT ALL DATA
----------------------------------------------------------------------
SELECT * FROM people;

* → selects all columns

----------------------------------------------------------------------
SELECT SPECIFIC COLUMNS
----------------------------------------------------------------------
SELECT age FROM people;
SELECT name, age FROM people;

----------------------------------------------------------------------
UNDERSTANDING FROM
----------------------------------------------------------------------
FROM → specifies the table from which data is retrieved

----------------------------------------------------------------------
EXPRESSIONS IN SELECT
----------------------------------------------------------------------
SELECT age + 10 FROM people;
→ adds 10 to each age

SELECT 2 + 2;
→ Output: 4

----------------------------------------------------------------------
COLUMN ALIAS (RENAME OUTPUT COLUMN)
----------------------------------------------------------------------
SELECT name AS personName FROM people;

AS → gives temporary name to column

----------------------------------------------------------------------
USING DISTINCT (UNIQUE VALUES)
----------------------------------------------------------------------
SELECT DISTINCT age FROM people;

----------------------------------------------------------------------
USING WHERE (FILTER DATA)
----------------------------------------------------------------------
SELECT * FROM people
WHERE age > 20;

----------------------------------------------------------------------
ORDER BY (SORTING)
----------------------------------------------------------------------
SELECT * FROM people
ORDER BY age;           → Ascending (default)

SELECT * FROM people
ORDER BY age DESC;      → Descending

----------------------------------------------------------------------
LIMIT (RESTRICT NUMBER OF ROWS)
----------------------------------------------------------------------
SELECT * FROM people
LIMIT 5;

----------------------------------------------------------------------
COMBINING CLAUSES
----------------------------------------------------------------------
SELECT name, age
FROM people
WHERE age > 20
ORDER BY age DESC
LIMIT 3;

----------------------------------------------------------------------
USING ALIAS FOR TABLE
----------------------------------------------------------------------
SELECT p.name, p.age
FROM people AS p;

----------------------------------------------------------------------
IMPORTANT NOTES
----------------------------------------------------------------------
1) SELECT does not modify data (read-only)
2) Execution order:
   FROM → WHERE → SELECT → ORDER BY → LIMIT
3) Use ; to end query
4) Use up/down arrow to reuse previous commands
5) Aliases are temporary (only for output)

MYSQL INSERT COMMAND
======================================================================
INSERT is used to add new records (rows) into a table.

SYNTAX
INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
Table: student (id, name, age)

INSERT INTO student (id, name, age)
VALUES (1, 'Ajay', 22);

----------------------------------------------------------------------
INSERT WITHOUT COLUMN NAMES
(Only if you know exact column order)
----------------------------------------------------------------------
INSERT INTO student
VALUES (2, 'Rahul', 23);

----------------------------------------------------------------------
INSERT MULTIPLE ROWS
----------------------------------------------------------------------
INSERT INTO student (id, name, age)
VALUES 
(3, 'Amit', 21),
(4, 'Neha', 22),
(5, 'Riya', 20);

----------------------------------------------------------------------
INSERT WITH SELECT
(Copy data from another table)
----------------------------------------------------------------------
INSERT INTO new_table (col1, col2)
SELECT col1, col2 FROM old_table;

----------------------------------------------------------------------
INSERT IGNORE
(Skip errors like duplicate keys)
----------------------------------------------------------------------
INSERT IGNORE INTO student (id, name)
VALUES (1, 'Duplicate');

----------------------------------------------------------------------
INSERT WITH DEFAULT VALUES
----------------------------------------------------------------------
INSERT INTO student (id, name)
VALUES (6, DEFAULT);

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) Strings must be in 'single quotes'
2) Number values → no quotes
3) Column order must match values
4) Primary key should be unique
5) NULL can be inserted if allowed

-good practice; always specify column name.

MYSQL DISTINCT KEYWORD
======================================================================

DEFINITION:
DISTINCT is used to remove duplicate values and return only unique values.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT DISTINCT column_name
FROM table_name;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
Table: student (name, city)

SELECT DISTINCT city
FROM student;

OUTPUT:
Delhi
Mumbai
Bangalore

(duplicate cities removed)

----------------------------------------------------------------------
DISTINCT ON MULTIPLE COLUMNS
----------------------------------------------------------------------
SELECT DISTINCT name, city
FROM student;

(returns unique combinations of name + city)

----------------------------------------------------------------------
DISTINCT vs WITHOUT DISTINCT
----------------------------------------------------------------------
WITHOUT DISTINCT:
SELECT city FROM student;
→ Delhi, Delhi, Mumbai, Mumbai

WITH DISTINCT:
SELECT DISTINCT city FROM student;
→ Delhi, Mumbai

----------------------------------------------------------------------
COUNT WITH DISTINCT
----------------------------------------------------------------------
SELECT COUNT(DISTINCT city)
FROM student;

(Returns number of unique cities)

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) DISTINCT applies to selected columns only
2) Removes duplicate rows from result
3) Slightly slower on large data (extra processing)
4) Often used with SELECT

CASES:
if there are two duplicate values, it will give one unique value only.

MYSQL WHERE CLAUSE
======================================================================

DEFINITION:
WHERE is used to filter records based on a condition.

----------------------------------------------------------------------
BASIC SYNTAX
----------------------------------------------------------------------
SELECT column1, column2
FROM table_name
WHERE condition;

----------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------
Table: student (id, name, age)

SELECT * 
FROM student
WHERE age > 21;

(Returns students with age greater than 21)

----------------------------------------------------------------------
USING DIFFERENT OPERATORS
----------------------------------------------------------------------

=        Equal
!=       Not equal
>        Greater than
<        Less than
>=       Greater than or equal
<=       Less than or equal

EXAMPLE:
SELECT * FROM student WHERE age = 22;

----------------------------------------------------------------------
USING AND / OR
----------------------------------------------------------------------

AND → both conditions must be true
OR  → any one condition true

SELECT * FROM student
WHERE age > 20 AND name = 'Ajay';

SELECT * FROM student
WHERE age < 20 OR city = 'Delhi';

----------------------------------------------------------------------
USING BETWEEN
----------------------------------------------------------------------

SELECT * FROM student
WHERE age BETWEEN 18 AND 22;

----------------------------------------------------------------------
USING IN
----------------------------------------------------------------------

SELECT * FROM student
WHERE city IN ('Delhi', 'Mumbai');

----------------------------------------------------------------------
USING LIKE (PATTERN MATCH)
----------------------------------------------------------------------

% → any characters
_ → single character

SELECT * FROM student
WHERE name LIKE 'A%';     (starts with A)

SELECT * FROM student
WHERE name LIKE '_a%';    (second letter is 'a')

----------------------------------------------------------------------
USING IS NULL
----------------------------------------------------------------------

SELECT * FROM student
WHERE age IS NULL;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) WHERE filters rows before result is returned
2) Used with SELECT, UPDATE, DELETE
3) Strings must be in 'single quotes'
4) Conditions can be combined

MYSQL CONSTRAINTS
======================================================================

DEFINITION:
Constraints are rules applied on table columns to restrict and control data.

Note: use Describe column_name; to see constraints on values.
----------------------------------------------------------------------
WHY USE CONSTRAINTS
----------------------------------------------------------------------
• Ensure data accuracy
• Prevent invalid data
• Maintain data integrity

----------------------------------------------------------------------
TYPES OF CONSTRAINTS
----------------------------------------------------------------------

1) NOT NULL
----------------------------------------------------------------------
Ensures column cannot have NULL value

CREATE TABLE student (
    id INT NOT NULL,
    name VARCHAR(50) NOT NULL
);

----------------------------------------------------------------------
2) UNIQUE
----------------------------------------------------------------------
Ensures all values in column are different

CREATE TABLE student (
    email VARCHAR(100) UNIQUE
);

----------------------------------------------------------------------
3) PRIMARY KEY
----------------------------------------------------------------------
• Uniquely identifies each row
• Cannot be NULL + must be UNIQUE
-only one primary key column.
-should not change  its value
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

----------------------------------------------------------------------
4) FOREIGN KEY
----------------------------------------------------------------------
• Links one table to another
• Maintains relationship

CREATE TABLE orders (
    order_id INT,
    student_id INT,
    FOREIGN KEY (student_id) REFERENCES student(id)
);

----------------------------------------------------------------------
5) CHECK
----------------------------------------------------------------------
Ensures condition is satisfied

CREATE TABLE student (
    age INT CHECK (age >= 18)
);

----------------------------------------------------------------------
6) DEFAULT
----------------------------------------------------------------------
Sets default value if none provided

CREATE TABLE student (
    city VARCHAR(50) DEFAULT 'Delhi'
);

----------------------------------------------------------------------
7) AUTO_INCREMENT
----------------------------------------------------------------------
Automatically increments value (usually for id)

CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY
);

----------------------------------------------------------------------
ADDING CONSTRAINTS AFTER TABLE CREATION
----------------------------------------------------------------------
ALTER TABLE student
ADD CONSTRAINT unique_email UNIQUE (email);

----------------------------------------------------------------------
REMOVING CONSTRAINT
----------------------------------------------------------------------
ALTER TABLE student
DROP INDEX unique_email;

----------------------------------------------------------------------
NOTES
----------------------------------------------------------------------
1) PRIMARY KEY = UNIQUE + NOT NULL
2) A table can have only ONE primary key
3) FOREIGN KEY creates relationship between tables
4) Constraints improve data reliability
5) CHECK may be ignored in older MySQL versions

CASES:
can INSERT null  mutiple times even for unique values, as null is just absence of data.
