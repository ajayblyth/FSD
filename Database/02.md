====================================================================================================
MYSQL DATA TYPES
====================================================================================================

CATEGORY     | DATA TYPE        | SIZE        | DESCRIPTION
----------------------------------------------------------------------------------------------------

--------------------------- NUMERIC TYPES ----------------------------------------------------------
INTEGER      | TINYINT          | 1 byte      | Very small integer (-128 to 127)
INTEGER      | SMALLINT         | 2 bytes     | Small integer
INTEGER      | MEDIUMINT        | 3 bytes     | Medium integer
INTEGER      | INT              | 4 bytes     | Standard integer (most used)
INTEGER      | BIGINT           | 8 bytes     | Large integer

DECIMAL      | DECIMAL(M,D)     | Variable    | Exact fixed-point (money)
FLOATING     | FLOAT            | 4 bytes     | Approx (low precision)
FLOATING     | DOUBLE           | 8 bytes     | High precision

Note:DECIMAL(M,D)

• M → Maximum number of digits (precision)
• D → Number of digits after decimal point 

Example: DECIMAL(8,3)
• Total digits = 8, Digits after decimal = 3
• Can store: 12345.678, 12.345, 0.001
• Max value = 99999.999
--------------------------- STRING TYPES -----------------------------------------------------------
Definition:
• A string is a sequence of characters (letters, numbers, symbols)
• Used to store textual data in tables
STRING       | CHAR(n)          | Fixed       | Fixed length (exact size)
STRING       | VARCHAR(n)       | Variable    | Variable length (0 → n)

TEXT         | TINYTEXT         | 255 chars   | Small text
TEXT         | TEXT             | 65 KB       | Normal text
TEXT         | MEDIUMTEXT       | 16 MB       | Large text
TEXT         | LONGTEXT         | 4 GB        | Very large text

--------------------------- DATE & TIME TYPES ------------------------------------------------------
DATE         | DATE             | 3 bytes     | YYYY-MM-DD
TIME         | TIME             | 3 bytes     | HH:MM:SS
DATETIME     | DATETIME         | 8 bytes     | Date + Time
TIMESTAMP    | TIMESTAMP        | 4 bytes     | Auto timestamp (tracking)
YEAR         | YEAR             | 1 byte      | YYYY

--------------------------- BINARY TYPES -----------------------------------------------------------
BINARY       | BINARY(n)        | Fixed       | Fixed binary
BINARY       | VARBINARY(n)     | Variable    | Variable binary

BLOB         | TINYBLOB         | 255 bytes   | Small binary
BLOB         | BLOB             | 65 KB       | Binary data
BLOB         | MEDIUMBLOB       | 16 MB       | Large binary
BLOB         | LONGBLOB         | 4 GB        | Very large binary

--------------------------- BOOLEAN ----------------------------------------------------------------
BOOLEAN      | BOOLEAN          | 1 byte      | TRUE(1) / FALSE(0)

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• INT, VARCHAR → Most used
• DECIMAL → Money (exact)
• TIMESTAMP → Auto tracking
• TEXT/BLOB → Use only when needed (heavy)

====================================================================================================
MYSQL CREATE TABLE
====================================================================================================

Definition :
CREATE TABLE → Creates a new table

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
CREATE TABLE table_name (
    column datatype
);

----------------------------------------------------------------------------------------------------
EXAMPLE
----------------------------------------------------------------------------------------------------
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    age INT
);

----------------------------------------------------------------------------------------------------
COMMON VARIATIONS
----------------------------------------------------------------------------------------------------
Definition:
• Constraints are rules applied on table columns
• Ensure data integrity and accuracy in database

-- CONSTRAINTS
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    age INT CHECK (age >= 18),
    email VARCHAR(100) UNIQUE
);

-- AUTO_INCREMENT
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

-- DEFAULT
CREATE TABLE student (
    id INT,
    name VARCHAR(50),
    city VARCHAR(50) DEFAULT 'Delhi'
);

-- FOREIGN KEY
CREATE TABLE course (
    course_id INT PRIMARY KEY
);

CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    course_id INT,
    FOREIGN KEY (course_id) REFERENCES course(course_id)
);

-- COPY TABLE
CREATE TABLE new_table AS
SELECT * FROM old_table;

----------------------------------------------------------------------------------------------------
TABLE STRUCTURE
----------------------------------------------------------------------------------------------------
DESCRIBE student;
DESC student;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Table name → unique
• Always define PRIMARY KEY
• Use constraints for integrity
• Meaningful column names

====================================================================================================
MYSQL SELECT
====================================================================================================

Definition :
SELECT → Fetch data (read-only)

----------------------------------------------------------------------------------------------------
BASIC
----------------------------------------------------------------------------------------------------
SELECT column1, column2 FROM table_name;
SELECT * FROM people;

----------------------------------------------------------------------------------------------------
USAGE
----------------------------------------------------------------------------------------------------
SELECT name, age FROM people;        → specific columns
SELECT DISTINCT age FROM people;     → unique values
SELECT age + 10 FROM people;         → expression
SELECT 2 + 2;                        → calculation

----------------------------------------------------------------------------------------------------
ALIAS
----------------------------------------------------------------------------------------------------
SELECT name AS personName FROM people;
SELECT p.name, p.age FROM people AS p;

----------------------------------------------------------------------------------------------------
FILTER / SORT / LIMIT
----------------------------------------------------------------------------------------------------
SELECT * FROM people WHERE age > 20;
SELECT * FROM people ORDER BY age DESC;
SELECT * FROM people LIMIT 5;

----------------------------------------------------------------------------------------------------
COMBINED
----------------------------------------------------------------------------------------------------
SELECT name, age
FROM people
WHERE age > 20
ORDER BY age DESC
LIMIT 3;

----------------------------------------------------------------------------------------------------
EXECUTION ORDER
----------------------------------------------------------------------------------------------------
FROM → WHERE → SELECT → ORDER BY → LIMIT

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Read-only
• Aliases → temporary
• ";" mandatory

====================================================================================================
MYSQL INSERT
====================================================================================================

Definition :
INSERT → Add rows

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
INSERT INTO table_name (col1, col2)
VALUES (val1, val2);

----------------------------------------------------------------------------------------------------
USAGE
----------------------------------------------------------------------------------------------------
-- Single
INSERT INTO student (id, name, age)
VALUES (1, 'Ajay', 22);

-- Without columns
INSERT INTO student
VALUES (2, 'Rahul', 23);

-- Multiple
INSERT INTO student (id, name, age)
VALUES 
(3, 'Amit', 21),
(4, 'Neha', 22);

-- From table
INSERT INTO new_table (col1, col2)
SELECT col1, col2 FROM old_table;

-- Ignore errors
INSERT IGNORE INTO student (id, name) //
VALUES (1, 'Duplicate');

//
• INSERT IGNORE → attempts to insert data, but ignores errors
• Common use → skip duplicate key errors or constraint violations
• If row already exists → query does nothing instead of failing
• Useful when bulk inserting without stopping on errors

-- Default
INSERT INTO student (id, name)
VALUES (6, DEFAULT);

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Strings → 'quotes'     • Numbers → no quotes
• Column order must match
• PK → unique
• NULL allowed if defined

• "5" → auto converted, "abc" fails for INT
• Best practice → specify columns

====================================================================================================
MYSQL DISTINCT
====================================================================================================

Definition :
DISTINCT → Removes duplicates

----------------------------------------------------------------------------------------------------
USAGE
----------------------------------------------------------------------------------------------------
SELECT DISTINCT city FROM student;
SELECT DISTINCT name, city FROM student;  • Here, combination of `name` and `city` must be unique


SELECT COUNT(DISTINCT city) FROM student;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Works on selected columns
• Removes duplicate rows
• Slightly slower on large data

====================================================================================================
MYSQL WHERE
====================================================================================================

Definition :
WHERE → Filters rows

----------------------------------------------------------------------------------------------------
SYNTAX
----------------------------------------------------------------------------------------------------
SELECT * FROM table WHERE condition;

----------------------------------------------------------------------------------------------------
OPERATORS
----------------------------------------------------------------------------------------------------
=   !=   >   <   >=   <=

----------------------------------------------------------------------------------------------------
CONDITIONS
----------------------------------------------------------------------------------------------------
-- AND / OR
SELECT * FROM student WHERE age > 20 AND name = 'Ajay';
SELECT * FROM student WHERE age < 20 OR city = 'Delhi';

-- BETWEEN
SELECT * FROM student WHERE age BETWEEN 18 AND 22;

-- IN
SELECT * FROM student WHERE city IN ('Delhi', 'Mumbai');

-- LIKE
%  → any chars     _ → single char
SELECT * FROM student WHERE name LIKE 'A%';
SELECT * FROM student WHERE name LIKE '_a%';

-- NULL
SELECT * FROM student WHERE age IS NULL;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• Filters before result
• Used in SELECT / UPDATE / DELETE
• Conditions can be combined

====================================================================================================
MYSQL CONSTRAINTS
====================================================================================================

Definition :
Constraints → Rules to control data validity

----------------------------------------------------------------------------------------------------
WHY USE
----------------------------------------------------------------------------------------------------
• Data accuracy     • Prevent invalid data     • Maintain integrity

----------------------------------------------------------------------------------------------------
TYPES
----------------------------------------------------------------------------------------------------

1) NOT NULL
→ No NULL allowed
id INT NOT NULL

2) UNIQUE
→ No duplicate values
email VARCHAR(100) UNIQUE

3) PRIMARY KEY
→ UNIQUE + NOT NULL (1 per table)
→ Should not change
id INT PRIMARY KEY

4) FOREIGN KEY
→ Links tables
FOREIGN KEY (student_id) REFERENCES student(id)

5) CHECK
→ Condition validation
age INT CHECK (age >= 18)

6) DEFAULT
→ Default value
city VARCHAR(50) DEFAULT 'Delhi'

7) AUTO_INCREMENT
→ Auto-increase values
id INT AUTO_INCREMENT PRIMARY KEY

----------------------------------------------------------------------------------------------------
ALTER OPERATIONS
----------------------------------------------------------------------------------------------------
-- ADD
ALTER TABLE student
ADD CONSTRAINT unique_email UNIQUE (email);

-- REMOVE
ALTER TABLE student
DROP INDEX unique_email;

----------------------------------------------------------------------------------------------------
NOTES
----------------------------------------------------------------------------------------------------
• PRIMARY KEY = UNIQUE + NOT NULL
• Only ONE PK per table
• FK → creates relationships
• Improves reliability
• CHECK may not work in older MySQL

• UNIQUE allows multiple NULLs (treated as no value)
