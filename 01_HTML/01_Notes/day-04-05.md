HTML ENTITIES
--------------------------------------------------------------------------------
HTML entities are **special codes** used to display characters
that are reserved in HTML or not easily typed.

Purpose:
• Display reserved HTML symbols (<, >, &, ") safely
• Show special characters (©, ®, €, ±, emojis)
• Prevent browser from interpreting characters as HTML tags

Syntax:
• Named entity: &name;      → Example: &lt;  → <
• Numeric entity: &#number; → Example: &#60; → <
• Hexadecimal entity: &#xHex; → Example: &#x3C; → <

Common HTML Entities:
• &lt;  → <      • &gt;  → >      • &amp; → &
• &quot; → "     • &apos; → '      • &copy; → ©
• &reg; → ®     • &euro; → €      • &nbsp; → non-breaking space

Example in HTML:
<p>10 &lt; 20 &amp; 5 &gt; 2</p>
<!-- Displays: 10 < 20 & 5 > 2 -->

Interview One-Liner:
HTML entities encode special characters so browsers render them correctly
without breaking HTML syntax.

HTML TABLES

Main table tags:
<table>   → table container
<caption> → table title
<thead>   → header section
<tbody>   → data section
<tfoot>   → footer/summary
<tr>      → table row
<th>      → header cell (bold, centered)
<td>      → data cell

Complete example:

<table border="1" cellpadding="10" cellspacing="0">

  <caption>Student Details</caption>
  <thead>
    <tr><th>Name</th><th>Age</th><th>City</th></tr>
  </thead>
  <tbody>
    <tr><td>Ajay</td><td>24</td><td>Delhi</td></tr>
    <tr><td>Rahul</td><td>26</td><td>Mumbai</td></tr>
  </tbody>
  <tfoot>
    <tr><td colspan="3">Total Students: 2</td></tr>
  </tfoot>
</table>

------------------------------------
TABLE ATTRIBUTES

rowspan → merge rows
colspan → merge columns

Examples:
<td colspan="3">Total</td>
<td rowspan="2">IT</td>

Example:
<table border="1" cellpadding="8" cellspacing="0">

    <tr>
        <th>Name</th>
        <th>Math</th>
        <th>Science</th>
        <th>Department</th>
    </tr>

    <tr>
        <td>Ajay</td>
        <td>80</td>
        <td>85</td>
        <td rowspan="2">IT</td>   <!-- IT spans 2 rows -->
    </tr>

    <tr>
        <td>Rahul</td>
        <td>75</td>
        <td>78</td>
        <!-- Department cell is skipped because of rowspan -->
    </tr>

    <tr>
        <td colspan="3">Total</td> <!-- Total spans 3 columns -->
        <td>IT</td>
    </tr>

</table>

Other attributes:
border → table border
cellpadding → space inside cells
cellspacing → space between cells
align → deprecated (use CSS)

Rule:
rowspan = merge rows | colspan = merge columns

HTML FORMS

Forms collect user input and send data to backend.

<form> attributes:
action → where data is sent
method → GET or POST

Form elements:
<input>    → user input (text, password, email, radio, checkbox, submit)
<label>    → input description (accessibility)
<textarea> → multi-line input
<select>   → dropdown
<option>   → dropdown items
<button>   → clickable button

Example:

<form action="/server" method="post">

    <!-- Username -->
    <label for="username">Username:</label>
    <input type="text" id="username" name="name">
    <br><br>

    Note:id → for label, CSS, JavaScript
    name → for sending data to server

    <!-- Password -->
    <label for="password">Password:</label>
    <input type="password" id="password" name="password">
    <br><br>

    <!-- Submit using input , --> 
    <input type="submit" value="Submit">
    <br><br>

    <!-- Button without type (acts as submit by default) -->
    <button>Click Me</button>
    <br><br>

    <!-- Button that does NOT submit the form -->
    <button type="button">Click Only</button>

</form>


------------------------------------
BUTTON TYPES & USES
--------------------------------------------------------------------------------
1) type="submit"   → Submits the form data to server
2) type="reset"    → Resets all form fields to default
3) type="button"   → Regular button, works with JS (no default action)

EXAMPLE (ALL TYPES)
--------------------------------------------------------------------------------
<form action="/submit">
  <input type="text" name="username" placeholder="Enter Name">
  
  <button type="submit">Submit</button>   <!-- Sends form -->
  <button type="reset">Reset</button>     <!-- Clears form -->
  <button type="button" onclick="alert('Hello!')">Click Me</button> <!-- JS action -->
</form>
--------------------------------------------------------------------------------

Important:
Default button type inside <form> is submit
Use type="button" to avoid submission

INPUT ATTRIBUTES (IMPORTANT)

name        → key sent to server (key=value)
value       → default value
placeholder → hint text
required    → mandatory field
readonly    → cannot edit
disabled    → inactive field
checked     → pre-selected (radio/checkbox)

RADIO BUTTONS

Used to select ONLY ONE option.

<form action="server">

<label>
  <input type="radio" id="apple" name="fruit" value="apple"> Apple
</label>


  <input type="radio" id="mango" name="fruit" value="mango">
  <label for="mango">Mango</label>

  <input type="radio" id="banana" name="fruit" value="banana">
  <label for="banana">Banana</label>

  <button type="submit">Submit</button>

</form>

Note:
📝 Why you didn’t need value in text input
<input type="text" name="username">

👉 Because:

User types the value manually
That typed text becomes the value

Example:

username=John
NOTES:
name → groups radio buttons together
same name → allows only one selection in the group
browser enforces single selection automatically
value → actual data sent to server
submitted as: name=value (example: fruit=mango)
- checkbox allows multiple, radio does not
