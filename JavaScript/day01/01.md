JAVASCRIPT (JS) 
============================================================================================================================

Definition:
JavaScript is a programming language that adds dynamic behavior and functionality
to web pages, making web pages interactive and responsive to user actions.

Dynamic means webpage content can change automatically without reloading.
Example: Clicking “Show Details” shows extra information instantly on the same page.

Functionality means the webpage can perform tasks based on user input.
Example: A login form checks email and password and responds accordingly.

Key Features:
✔ Change content, colors, and layout dynamically(without page reload)
✔ Make buttons clickable, forms work, and pages respond to events
✔ Perform filtering, validation, and real-time updates
✔ Runs in browser (client-side) and also on server (Node.js)

Use Cases:
- Form validation
- Interactive buttons, menus, modals
- Real-time updates (notifications, chat messages)
- Dynamic content changes without page reload

Interview One-Liner:
JavaScript makes web pages interactive by adding dynamic behavior, real-time updates,
and user-driven functionality.

JAVASCRIPT VARIABLES – QUICK NOTES
============================================================================================================================

Definition:
A variable is the name of a storage location in JS.
It acts like a labeled container that holds information and allows you to reference
and manipulate data dynamically.

Keywords:
var   → Function-scoped, can be redeclared (legacy)
let   → Block-scoped, can be updated (modern JS)
const → Block-scoped, cannot be updated or redeclared

// Why "var" is not preferred anymore:

// 1) Not block-scoped → escapes {} blocks
if (true) {
    var x = 10;
}
console.log(x); // 10 (unexpected, works outside block)

// 2) Allows redeclaration → can cause bugs
var a = 5;
var a = 20; // No error

// ✔ Modern practice: use let or const (block-scoped and safer)

Naming Rules:
- Cannot start with a number
- No spaces or special characters (except _ or $)
- Case-sensitive

Examples:
var name = "Ajay";     
let age = 25;          
const pi = 3.1416;     

Notes:
- Prefer let and const in modern JS
- Variables can hold any data type (dynamic typing)

JAVASCRIPT DATA TYPES – QUICK NOTES
============================================================================================================================
Definition:
JS has dynamic typing (type can change at runtime, means no need to mention the type in declaration).
// Example:

let value = 10;        // number
console.log(typeof value);  // "number"

value = "Hello";       // now string
console.log(typeof value);  // "string"

List of Primitive Types ( 7), immutable.

1) Number     → 10, 3.14  [ 2^53 - 1 (safe integer limit)]

2) String     → "Hello"
3) Boolean    → true / false
4) Undefined  → Variable declared but not assigned
5) Null       → Intentional absence of value
6) Symbol     → Unique identifier (ES6), Creates a unique identifier.Used to create a unique property key in an object to avoid name conflicts.
Example:
let id = Symbol("id");

let id2 = Symbol("id");

console.log(id === id2);  // false (every Symbol is unique)


7) BigInt     → Large integers (123n) //// Without 'n', it becomes a Number and not BigInt.


Note:
- typeof can be used to check type
- Primitive values are stored by value
- They are immutable

// Rule for adding string and number:
// + operator → If ONE value is string → it does CONCATENATION
// -, *, /    → will Always try numeric conversion

NON-PRIMITIVE (Reference Types, Mutable):They store collections of values or complex data.
They are mutable and stored by reference.

Object     → {name: "Ajay", age: 25}
Array      → [1, 2, 3]
Function   → Reusable code block (functions are objects)

Notes:
- typeof object → "object"
- typeof array  → "object"
- typeof function → "function"
- They are mutable (can be modified)
- Compared by reference, not by value

Examples:
let num = 100;
let name = "Ajay";
let isActive = true;
let data;
let emptyValue = null;
let id = Symbol("id");
let big = 123456789012345678901n;
let person = {name:"Ajay"};
let arr = [1,2,3];
function greet(){ return "Hi"; }

Notes:
- typeof operator checks type
- Primitives are immutable
- Objects and arrays are mutable

============================================================================================================================
== vs === IN JAVASCRIPT  (Comparison Operators)
============================================================================================================================

In Java:
==  → Used to compare values (primitive types)
For objects → compares references (memory address)

In JavaScript:
==   → Loose Equality (compares value only, allows type coercion)
===  → Strict Equality (compares value AND type, no type coercion)

------------------------------------------------------------------------------------------------
JAVASCRIPT: ==
------------------------------------------------------------------------------------------------
Definition:
Compares values after converting (coercing) types automatically.

Examples:
5 == "5"        → true     (string converted to number)
true == 1       → true
null == undefined → true

Problem:
Can give unexpected results because of automatic type conversion.

------------------------------------------------------------------------------------------------
JAVASCRIPT: ===
------------------------------------------------------------------------------------------------
Definition:
Compares both value AND type.
No type conversion happens.

Examples:
5 === "5"       → false    (number !== string)
5 === 5         → true
true === 1      → false
null === undefined → false

------------------------------------------------------------------------------------------------
KEY DIFFERENCE
------------------------------------------------------------------------------------------------
==   → Value comparison only (type conversion happens)
===  → Value + Type comparison (no conversion)

------------------------------------------------------------------------------------------------
INTERVIEW BEST PRACTICE
------------------------------------------------------------------------------------------------
✔ Always use === in JavaScript
✔ Avoid == unless you clearly understand type coercion

// =========================================
// TYPE COERCION (JavaScript)
// =========================================

// Coercion → Automatic type conversion
// done by JavaScript during operations.


// -----------------------------------------
// 1) Implicit Coercion (Automatic)
// -----------------------------------------

console.log("5" + 2);   // "52"  (number → string)
console.log("5" - 2);   // 3     (string → number)
console.log(true + 1);  // 2     (true → 1)


// -----------------------------------------
// 2) Explicit Coercion (Manual)
// -----------------------------------------

let str = "100";

console.log(Number(str));   // 100
console.log(String(50));    // "50"
console.log(Boolean(0));    // false

============================================================================================================================

Definition:
NaN = "Not-a-Number"

OPERATOR PRECEDENCE & ASSOCIATIVITY
============================================================================================================================

Definition:
Precedence decides which operator runs first.
Associativity decides direction when precedence is same.

Associativity:
Left-to-Right  → Most operators
Right-to-Left  → Assignment (=), Exponentiation (**), Ternary (?:)

------------------------------------------------------------------------------------------------
PRECEDENCE (High → Low)
------------------------------------------------------------------------------------------------

 1) ()                      // paranthesis
 2) ++  --  !  +  -  typeof // Unary
 3) **                      // Exponentiation
 4) *  /  %                 // Multiplication / Division / Modulus
 5) +  -                    // Addition / Subtraction
 6) <  <=  >  >=            // Relational
 7) ==  !=  ===  !==        // Equality
 8) &&                      // Logical AND
 9) ||                      // Logical OR
 10) ?:                     // Ternary
 11) =  +=  -=  *=  /= %=   // Assignment

Examples:
10 + 5 * 2        → 20
(10 + 5) * 2      → 30
10 - 5 + 2        → 7
m = n = 10        → Right-to-Left (both 10)

JAVASCRIPT "let" KEYWORD – QUICK NOTES
============================================================================================================================

Definition:
let is used to declare variables in JavaScript.
It is block-scoped and introduced in ES6 (modern JS).

- Declare means to create a variable and give it a name in memory.

------------------------------------------------------------------------------------------------
KEY FEATURES
------------------------------------------------------------------------------------------------
✔ Block-scoped (accessible only inside { })
✔ Can be updated (value can change)
✔ Cannot be redeclared in the same scope
✔ Safer than var

------------------------------------------------------------------------------------------------
WHY let IS SAFER THAN var
------------------------------------------------------------------------------------------------
- let has block scope, var has function scope
- Prevents accidental redeclaration
- Does NOT attach to window object (browser)
- Avoids hoisting issues caused by var

------------------------------------------------------------------------------------------------
EXAMPLES
------------------------------------------------------------------------------------------------
1) Reassignment
let age = 23;
age = age + 1;          // 24

2) Declaration without initial value
let cgpa;
cgpa = 8.9;

3) Initializing with expression
let num1 = 1;
let num2 = 2;
let sum = num1 + num2;

------------------------------------------------------------------------------------------------
BLOCK SCOPE EXAMPLE
------------------------------------------------------------------------------------------------
{
    let x = 10;
    console.log(x);     // 10
}
console.log(x);         // Error (not defined outside block)

------------------------------------------------------------------------------------------------
UPDATE vs REDECLARE
------------------------------------------------------------------------------------------------
Update Allowed:
let age = 25;
age = 26;               // ✔ Allowed

Redeclare Not Allowed:
let num = 5;
let num = 10;           // ❌ Error

------------------------------------------------------------------------------------------------
DIFFERENCE FROM var
------------------------------------------------------------------------------------------------
let → Block scope
var → Function scope
let does NOT attach to window object

------------------------------------------------------------------------------------------------
BEST PRACTICE
------------------------------------------------------------------------------------------------
✔ Use let when value needs to change
✔ Prefer const by default


============================================================================================================================
JAVASCRIPT "const" KEYWORD – QUICK NOTES
============================================================================================================================

Definition:
const is used to declare variables whose value cannot be reassigned.
It is block-scoped and introduced in ES6 (modern JS).

------------------------------------------------------------------------------------------------
KEY FEATURES
------------------------------------------------------------------------------------------------
✔ Block-scoped
✔ Cannot be reassigned
✔ Cannot be redeclared in same scope
✔ Must be initialized at declaration time
✔ Preferred by default

------------------------------------------------------------------------------------------------
WHY const IS SAFER
------------------------------------------------------------------------------------------------
- Prevents accidental reassignment
- Makes code predictable
- Reduces bugs from value changes
- Encourages intentional coding

------------------------------------------------------------------------------------------------
INITIALIZATION REQUIRED
------------------------------------------------------------------------------------------------
const pi;          // ❌ Error
const pi = 3.14;   // ✔ Correct

------------------------------------------------------------------------------------------------
REASSIGNMENT NOT ALLOWED
------------------------------------------------------------------------------------------------
const age = 25;
age = 26;          // ❌ Error

------------------------------------------------------------------------------------------------
IMPORTANT: OBJECTS & ARRAYS
------------------------------------------------------------------------------------------------
const prevents reassignment, NOT mutation.

Object Example:
const person = { name: "Ajay" };
person.name = "Rahul";     // ✔ Allowed (modified)
person = {};               // ❌ Error (reassign not allowed)

Array Example:
const arr = [1,2,3];
arr.push(4);               // ✔ Allowed (modified)
arr = [5,6];               // ❌ Error

------------------------------------------------------------------------------------------------
BEST PRACTICE
------------------------------------------------------------------------------------------------
✔ Use const by default
✔ Use let only when value needs to change
✔ Avoid var in modern JS
