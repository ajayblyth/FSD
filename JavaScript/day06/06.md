ARRAY REFERENCES & NESTED ARRAYS (JS)
============================================================================================================================
Array:It is an ordered collection of values (of any type) stored in a single variable,
 accessible by numeric indices starting from 0.

ARRAY REFERENCES
------------------------------------------------
- Arrays are reference types (stored in heap)
- Variable stores memory reference, not actual data

Note:Primitives (number, string, boolean, etc.) store actual values; copying makes a separate value.

Objects/arrays/functions store references; copying points to the same memory.

Example:
let arr1 = [1,2,3];
let arr2 = arr1;
arr2.push(4);

arr1 → [1,2,3,4]
arr2 → [1,2,3,4]
// arr1 & arr2 point to same memory

------------------------------------------------
COMPARISON USING ===
------------------------------------------------
- === compares references (addresses), not values

let a = [1,2];
let b = [1,2];
console.log(a === b); // false → different memory

let c = a;
console.log(a === c); // true → same reference

------------------------------------------------
PRIMITIVES VS OBJECTS/ARRAYS
------------------------------------------------
PRIMITIVES (Number, String, Boolean, null, undefined, BigInt, Symbol)
- Stored by value
- === compares actual values

Objects/Arrays/Functions
- Stored by reference (heap)
- === compares memory address

Assignment:
- Primitives → copy value
- Objects/Arrays → copy reference

------------------------------------------------
CONSTANT ARRAYS
------------------------------------------------
const arr = [1,2,3]; // reference is constant, values mutable
arr.push(4);          // ✔ adds 4
arr.pop();            // ✔ removes 4

============================================================================================================================
NESTED ARRAYS
------------------------------------------------
Definition:
- Array inside another array (2D structure)

Syntax:
let arr = [[1,2],[3,4],[5,6]];

Access:
arr[row][column]

Example:
let matrix = [[1,2,3],
              [4,5,6],
              [7,8,9]];

console.log(matrix[0]);     // [1,2,3]
console.log(matrix[1][2]);  // 6

Use Cases:
- 2D grids, tables, matrix problems
- Outer array → rows
- Inner array → columns
- Indexing starts from 0

Example Tic-Tac-Toe Board:
let arr = [["x", null, "o"],
           [null,"x",null],
           ["o",null,"x"]];

LOOPS IN JAVASCRIPT
------------------------------------------------
Purpose: Execute a block repeatedly

1) for loop (known iterations) – use when number of iterations is known
   for(initialization; condition; update) {
       // code
   }
   Example:
   for(let i = 0; i < 3; i++) {
       console.log(i);
   }
   // e.g., looping 10 times to print first 10 numbers

2) while loop (condition-based) – use when number of iterations is unknown
   let i = 0;
   while(i < 3) {
       console.log(i);
       i++;
   }
   // e.g., reading user input until they type "exit"

3) do...while (executes at least once) – use when code must run at least once
   let i = 0;
   do {
       console.log(i);
       i++;
   } while(i < 3);
   // e.g., showing a menu at least once before user chooses to quit

4) for...of (iterate values) – use when you want to directly access iterable values
for...of → used to iterate over values in arrays, strings, or other iterables.

   - ex.1) for(let val of [1, 2, 3]) 
   {
       console.log(val);
   }
   - ex.2) let str = "Hello";
for (let ch of str) {
    console.log(ch);

   // e.g., printing all items in an array of fruits

5) for...in (iterate keys / indices) – use when you want to iterate over object keys/indices
for...in → used to iterate over keys/properties in objects (or array indices).
   for(let key in {a:1, b:2}) {
       console.log(key);
   }

  ex.2) let arr = [10, 20, 30];
for (let index in arr) {
    console.log(index);
}
// Output:
// 0
// 1
// 2

ex.3) To get values using the index:
for (let index in arr) {
    console.log(arr[index]);
}
// Output:
// 10
// 20
// 30
   // e.g., checking all properties in a user object

Loop Controls:
- break → stops loop
- continue → skips current iteration
