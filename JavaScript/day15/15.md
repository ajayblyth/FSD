====================================================
DOM EVENTS + addEventListener – MASTER NOTES
====================================================

WHAT IS DOM?
----------------------------------------------------
DOM (Document Object Model) is a tree-like
structure representation of a webpage.

It contains all elements:
✔ Headings
✔ Paragraphs
✔ Buttons
✔ Images
✔ Forms

JavaScript uses the DOM to:
✔ Access elements
✔ Modify content and styles
✔ Attach event listeners


====================================================
WHAT IS A DOM EVENT?
====================================================

An event is a signal that something has happened
on a webpage.

Types:
✔ User actions (click, type, hover)
✔ Browser actions (load, resize)


COMMON EVENTS
----------------------------------------------------
"click"       → Element clicked
"mouseenter"  → Mouse enters element
"keydown"     → Key pressed
"submit"      → Form submitted
"load"        → Page loaded
"resize"      → Window resized


====================================================
HOW EVENTS MAKE A PAGE INTERACTIVE
====================================================

User Action
     ↓
Event Occurs
     ↓
JavaScript Listener Detects
     ↓
Callback Function Executes
     ↓
Page Reacts / Updates


====================================================
addEventListener() METHOD
====================================================

Recommended and modern way to attach event listeners.

SYNTAX
----------------------------------------------------
element.addEventListener("event", callback);

PARAMETERS
----------------------------------------------------
1) element
   → The specific HTML element to watch
   Example:
   let btn = document.querySelector("button");

2) event
   → Event type (string)
   Examples:
   "click", "keydown", "mouseover", "submit"

3) callback
   → Function that runs when the event happens
   → Executes only after the event occurs


====================================================
EXAMPLE
====================================================

let btn = document.querySelector("button");

btn.addEventListener("click", function () {
    console.log("Button clicked");
});

Step-by-step flow:
1) User clicks button
2) "click" event triggers
3) Callback function runs
4) Message prints in console


====================================================
NAMED FUNCTION EXAMPLE
====================================================

function sayHello() {
    console.log("Hello");
}

btn.addEventListener("click", sayHello);

✔ Pass function reference
✘ Do NOT write sayHello() (that runs immediately)


- To run multiple functions at one event , use addEventListerner():
for (let b of btns) {
    b.addEventListener("click", sayHello);
    b.addEventListener("click", sayName);
}
====================================================
WHAT IS A CALLBACK?
====================================================

A callback is a function passed as an argument
that runs later when the event occurs.

It does NOT execute immediately.


====================================================
USING "this" IN EVENT LISTENERS
====================================================

Regular Function:
----------------------------------------------------
btn.addEventListener("click", function () {
    console.log(this);
});

✔ "this" refers to the element
✔ Here → button element


Arrow Function:
----------------------------------------------------
btn.addEventListener("click", () => {
    console.log(this);
});

✘ "this" does NOT refer to button
✔ Refers to outer (lexical) scope


Interview Point:
Regular function → this = element
Arrow function   → this = outer scope


====================================================
WHY addEventListener IS PREFERRED
====================================================

✔ Allows multiple event handlers on same element
✔ Cleaner & modern approach
✔ Can remove later using removeEventListener()
✔ Supports event capturing & bubbling


====================================================
CORE CONCEPT
====================================================

Element + Event + Callback
          ↓
When event happens
          ↓
Callback function executes

====================================================
EVENT OBJECT – COMPLETE NOTES
====================================================

WHAT IS EVENT OBJECT?
----------------------------------------------------
The event object is an automatically created
object that contains information about
the event that just happened.

It is passed to the callback function
whenever an event occurs.


====================================================
HOW TO ACCESS EVENT OBJECT
====================================================

When writing an event listener, add a parameter
inside the callback function.

Example:
----------------------------------------------------

btn.addEventListener("click", function (event) {
    console.log(event);
});

You can name it anything:
✔ event
✔ e
✔ evt

Common practice:
function(e) { ... }


====================================================
WHAT DOES EVENT OBJECT CONTAIN?
====================================================

It stores details like:

✔ Type of event
✔ Target element
✔ Mouse position
✔ Key pressed
✔ Time of event
✔ Whether default was prevented


====================================================
IMPORTANT PROPERTIES
====================================================

1) event.type
----------------------------------------------------
Returns the event name.

Example:
console.log(e.type);
Output → "click"


2) event.target
----------------------------------------------------
Returns the element that triggered the event.

Example:
console.log(e.target);


3) event.currentTarget
----------------------------------------------------
Element on which listener is attached.

Difference:
target → actual clicked element
currentTarget → element with listener


4) event.clientX / event.clientY
----------------------------------------------------
Mouse position (X & Y coordinates)


5) event.key
----------------------------------------------------
Returns key pressed (used in keydown).

Example:
document.addEventListener("keydown", function(e) {
    console.log(e.key);
});


6) event.preventDefault()
----------------------------------------------------
Stops default browser behavior.

Example (Form submit):

form.addEventListener("submit", function(e) {
    e.preventDefault();
});


7) event.stopPropagation()
----------------------------------------------------
Stops event bubbling.


====================================================
MOUSE EVENT EXAMPLE
====================================================

btn.addEventListener("click", function(e) {
    console.log("Event type:", e.type);
    console.log("Clicked element:", e.target);
    console.log("Mouse X:", e.clientX);
});


====================================================
KEYBOARD EVENT EXAMPLE
====================================================

document.addEventListener("keydown", function(e) {
    console.log("Key pressed:", e.key);
});


====================================================
FLOW OF EVENT OBJECT
====================================================

User Action
     ↓
Event Occurs
     ↓
Browser Creates Event Object
     ↓
Event Object Passed to Callback
     ↓
You Access Its Properties


====================================================
IMPORTANT INTERVIEW POINTS
====================================================

✔ Event object is automatically passed
✔ You must receive it as a parameter
✔ It gives full details about the event
✔ Used heavily in forms, keyboard, and mouse handling


====================================================
CORE IDEA
====================================================

addEventListener("event", function(e) {
        // e = event object
});

====================================================
FORM EVENTS – ORGANIZED MASTER NOTES
====================================================

Forms are used to collect user input.

Examples:
✔ Login form
✔ Signup form
✔ Search bar
✔ Feedback form


====================================================
COMMON FORM EVENTS
====================================================

1) "submit"
2) "input"
3) "change"
4) "focus"
5) "blur"


====================================================
1) SUBMIT EVENT
====================================================

Triggered when a form is submitted.

Default Behavior:
✔ Form data is sent
✔ Page reloads

To stop default behavior:
→ event.preventDefault()

Example:
----------------------------------------------------

let form = document.querySelector("#f");

form.addEventListener("submit", function(e) {
    e.preventDefault();   // stops page refresh

    let username = document.querySelector("#username").value;
    console.log("Form submitted");
    console.log("Username:", username);
});


====================================================
WHAT IS event.preventDefault() ?
====================================================

Definition:
Stops the default browser action
for that event so that JavaScript can control what happens instead.

Commonly used with:
✔ submit (stop page reload)
✔ click on links (stop navigation)


====================================================
Example – Stop Link Navigation
====================================================

HTML:
<a href="https://google.com" id="link">Go</a>

JavaScript:
document.querySelector("#link").addEventListener("click", function(e) {
    e.preventDefault();   // prevents navigation
    console.log("Link clicked");
});


====================================================
2) INPUT EVENT
====================================================

Triggered every time user types.

Example:
----------------------------------------------------

let input = document.querySelector("#username");

input.addEventListener("input", function(e) {
    console.log(e.target.value);
});

✔ Runs on every keystroke


====================================================
3) CHANGE EVENT
====================================================

Triggered when input loses focus
AFTER value is changed.

Works well with:
✔ Dropdown
✔ Checkbox
✔ Radio

Example:
----------------------------------------------------

input.addEventListener("change", function(e) {
    console.log("Changed to:", e.target.value);
});

Difference:
input  → runs instantly
change → runs after leaving field


====================================================
4) FOCUS EVENT
====================================================

Triggered when input is selected.

input.addEventListener("focus", function() {
    console.log("Input focused");
});


====================================================
5) BLUR EVENT
====================================================

Triggered when input loses focus.

input.addEventListener("blur", function() {
    console.log("Input lost focus");
});


====================================================
FORM EVENT FLOW
====================================================

User fills form
     ↓
User clicks submit
     ↓
"submit" event fires
     ↓
Event object passed
     ↓
e.preventDefault() (optional)
     ↓
JavaScript handles data


====================================================
CORE IDEA
====================================================

form.addEventListener("submit", function(e) {
    e.preventDefault();
    // handle form data using .value
});

====================================================
MORE EVENTS – CHANGE & INPUT
====================================================

CHANGE EVENT
----------------------------------------------------
The "change" event occurs when the value of an
element is changed AND the element loses focus.

Works only on:
✔ <input>
✔ <textarea>
✔ <select>

Example:
----------------------------------------------------

let user = document.querySelector("#user");

user.addEventListener("change", function () {
    console.log("change event");
    console.log("final value =", this.value);
});

✔ Runs after editing is finished
✔ Gives final value


====================================================
INPUT EVENT
====================================================

The "input" event fires immediately whenever
the value of an element changes at every step.

Works on:
✔ <input>
✔ <textarea>
✔ <select>

Example 1:
----------------------------------------------------

let user = document.querySelector("#user");

user.addEventListener("input", function () {
    console.log("current value =", this.value);
});

✔ Runs on every keystroke


Example 2 (Live Text Update):
----------------------------------------------------

let inp = document.querySelector("#text");
let p = document.querySelector("p");

inp.addEventListener("input", function () {
    console.log(inp.value);
    p.innerText = inp.value;
});

✔ Updates paragraph instantly
✔ Used for live preview


====================================================
DIFFERENCE: input vs change
====================================================

input  → Runs instantly while typing
change → Runs after leaving the field
