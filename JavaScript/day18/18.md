================================================================================================
CALL STACK IN JAVASCRIPT
================================================================================================

DEFINITION
--------------------------------------------------------------------------------
Call Stack is a data structure used by the JavaScript engine to keep track of
function execution. It follows the LIFO principle (Last In, First Out).

Meaning:
The last function added to the stack is the first one to be removed.


PURPOSE
--------------------------------------------------------------------------------
1) Keeps track of which function is currently executing
2) Manages function calls
3) Handles nested function execution
4) Removes functions after execution finishes


WORKING PRINCIPLE
--------------------------------------------------------------------------------
JavaScript is single-threaded, so it executes one task at a time using the
Call Stack.

Process:

1) When a function is called → it is PUSHED into the stack
2) When the function finishes → it is POPPED out of the stack


STACK OPERATIONS
--------------------------------------------------------------------------------
1) Push  → Add function to stack
2) Pop   → Remove function from stack


EXAMPLE
--------------------------------------------------------------------------------

function one(){
    console.log("Function One");
}

function two(){
    one();
    console.log("Function Two");
}

two();


EXECUTION FLOW
--------------------------------------------------------------------------------

Step 1
Call Stack:
two()

Step 2
two() calls one()

Call Stack:
one()
two()

Step 3
one() finishes

Call Stack:
two()

Step 4
two() finishes

Call Stack:
(empty)


OUTPUT
--------------------------------------------------------------------------------
Function One
Function Two

--------------------------------------------------------------------------------

---------

IMPORTANT POINTS
--------------------------------------------------------------------------------
1) JavaScript executes code using a Call Stack
2) Only one function runs at a time
3) Functions are pushed and popped from the stack
4) Follows LIFO (Last In First Out)

RELATED CONCEPT
--------------------------------------------------------------------------------
STACK OVERFLOW

Definition
--------------------------------------------------------------------------------
Stack Overflow occurs when the Call Stack becomes full because too many
function calls are added without finishing.

This usually happens due to:
1) Infinite recursion
2) Deep nested function calls
3) Missing stopping condition in recursion


HOW IT HAPPENS
--------------------------------------------------------------------------------
Each function call is pushed onto the Call Stack.

If a function keeps calling itself repeatedly without stopping,
new stack frames keep getting added.

Eventually the stack limit is reached → JavaScript throws an error.


EXAMPLE (INFINITE RECURSION)
--------------------------------------------------------------------------------

function test(){
    test();
}

test();


FLOW
--------------------------------------------------------------------------------
Call Stack grows like this:

test()
test()
test()
test()
test()
test()
test()
...

Since there is no stopping condition, the stack keeps growing
until memory limit is reached.

Error produced:

RangeError: Maximum call stack size exceeded

SOLUTION: USE A BASE CASE
--------------------------------------------------------------------------------
When using recursion, always define a condition where the function stops.

This stopping condition is called the BASE CASE.

-- Recursion is a programming technique where a function calls itself to solve a problem until a base case (stopping condition) is reached.

EXAMPLE WITH BASE CASE
--------------------------------------------------------------------------------

function countDown(n){

    if(n === 0){
        return;        // base case (stop recursion)
    }

    console.log(n);
    countDown(n - 1);  // recursive call
}

countDown(5);


FLOW
--------------------------------------------------------------------------------

countDown(5)
countDown(4)
countDown(3)
countDown(2)
countDown(1)
countDown(0)  → stops


OUTPUT
--------------------------------------------------------------------------------
5
4
3
2
1


HOW TO AVOID STACK OVERFLOW
--------------------------------------------------------------------------------
1) Always define a BASE CASE in recursive functions

2) Ensure recursion moves toward the base condition
   (example: n → n-1 or n → n/2)

3) Avoid infinite loops inside recursive calls

4) For very large iterations, prefer loops instead of recursion


LOOP ALTERNATIVE
--------------------------------------------------------------------------------

for(let i = 5; i > 0; i--){
    console.log(i);
}


KEY IDEA
--------------------------------------------------------------------------------
Recursion must always have:

1) Base Case      → when to stop
2) Recursive Case → how the function continues
3) Progress       → each call moves closer to base case
