==================== MONGODB (QUICK NOTES) ====================

1. WHAT IS MONGODB
---------------------------------------------------------------
- MongoDB = NoSQL database
- Stores data in JSON-like format (BSON)
- No tables → uses collections & documents
- Schema is flexible (no fixed structure)


==================== INSTALL MONGODB (WINDOWS) ====================

1. DOWNLOAD
---------------------------------------------------------------
- Go to: https://www.mongodb.com/try/download/community
- Select:
    Version  → Latest
    OS       → Windows
    Package  → MSI
- Click Download

---------------------------------------------------------------

2. INSTALL
---------------------------------------------------------------
- Open downloaded .msi file
- Choose → Complete setup
- IMPORTANT:
    ✔ Check "Install MongoDB as a Service"
    ✔ Check "Install MongoDB Compass" (GUI tool)

- Click Next → Install → Finish

---------------------------------------------------------------

3. VERIFY INSTALLATION
---------------------------------------------------------------
- Open Command Prompt

Check version:
> mongod --version

If it shows version → installed correctly ✅

---------------------------------------------------------------

4. START MONGODB
---------------------------------------------------------------
Usually auto-starts as service

If not:
> net start MongoDB

---------------------------------------------------------------

5. CONNECT (SHELL)
---------------------------------------------------------------
Open new terminal:
> mongosh

If connected → success 🎉

---------------------------------------------------------------

6. TEST
---------------------------------------------------------------
> show dbs
> use testdb
> db.users.insertOne({ name: "Ajay" })
> db.users.find()

mongosh
The modern MongoDB Shell for interacting with your database

Start mongosh

Just type mongosh in your terminal.

show dbs
List all databases

use <db>
Switch to or create a database

showIcollections
List collections in the current database

db.createCollection("name")
Create a collection

db .< collection>.insertOne((})
Insert a document

db .< collection>.find()
Query all documents

db .< collection>.updateOne()
Update a document

db .< collection>.deleteOne()
Delete a document

Exit the shell

4. CRUD OPERATIONS
---------------------------------------------------------------

INSERT
------
db.users.insertOne({
  name: "Ajay",
  age: 22
});

READ
----
db.users.find();              // all data
db.users.find({ age: 22 });   // filter

UPDATE
------
db.users.updateOne(
  { name: "Ajay" },
  { $set: { age: 23 } }
);

DELETE
------
db.users.deleteOne({ name: "Ajay" });

==================== COLLECTION vs DOCUMENT ====================

1. COLLECTION
---------------------------------------------------------------
- A collection = group of documents
- Similar to "table" in SQL
- No fixed schema (documents can differ)

Example:
users (collection)
  → many user documents inside

---------------------------------------------------------------

2. DOCUMENT
---------------------------------------------------------------
- A document = single record (data)
- Stored in JSON/BSON format
- Similar to "row" in SQL

Example:
{
  name: "Ajay",
  age: 22,
  skills: ["Java", "Node"]
}

---------------------------------------------------------------

3. RELATION (VERY IMPORTANT)
---------------------------------------------------------------
Database
  → Collection
      → Document
          → Fields (key-value pairs)

---------------------------------------------------------------

4. EXAMPLE
---------------------------------------------------------------
Collection: users

Document 1:
{
  name: "Ajay",
  age: 22
}

Document 2:
{
  name: "Rahul",
  age: 25,
  city: "Bangalore"
}

👉 Notice:
- Different structure allowed (flexible schema)

==================== SQL vs MONGODB (MAPPING + NOTES) ====================

1. CORE MAPPING
---------------------------------------------------------------------------
SQL (RDBMS)        | MongoDB (NoSQL)        | DESCRIPTION
-------------------|------------------------|------------------------------
Database           | Database               | Top-level container
Table              | Collection             | Group of related data
Row / Record       | Document               | Single entry of data
Column             | Field                  | Property / attribute
Primary Key        | _id                    | Unique identifier
JOIN               | Embedded Doc / $lookup | Linking related data

---------------------------------------------------------------------------

2. DATA REPRESENTATION
---------------------------------------------------------------------------

SQL TABLE (FIXED STRUCTURE)
---------------------------------------------------------------------------
+----+--------+-----+-----------+
| id | name   | age | city      |
+----+--------+-----+-----------+
| 1  | Ajay   | 22  | Bangalore |
| 2  | Rahul  | 25  | Delhi     |
+----+--------+-----+-----------+

👉 Same columns for every row (strict schema)

---------------------------------------------------------------------------

MONGODB COLLECTION (FLEXIBLE DOCUMENTS)
---------------------------------------------------------------------------
{
  _id: 1,
  name: "Ajay",
  age: 22,
  city: "Bangalore"
}

{
  _id: 2,
  name: "Rahul",
  age: 25
}

👉 Fields can vary (flexible schema)

---------------------------------------------------------------------------

3. RELATION HANDLING
---------------------------------------------------------------------------

SQL (USING JOIN)
---------------------------------------------------------------------------
users table + orders table → JOIN needed

MongoDB (EMBEDDED DOCUMENT)
---------------------------------------------------------------------------
{
  name: "Ajay",
  orders: [
    { product: "Laptop", price: 70000 },
    { product: "Mouse", price: 500 }
  ]
}

👉 No JOIN needed (data stored together)

---------------------------------------------------------------------------

4. QUICK DIFFERENCE
---------------------------------------------------------------------------
SQL                        | MongoDB
---------------------------|-------------------------
Table (rows/columns)       | Collection (documents)
Strict schema              | Flexible schema
Joins for relations        | Embedded documents
Structured data            | Semi-structured data
