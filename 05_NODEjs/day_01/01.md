==================== API CALL FLOW (JS + AXIOS + EVENT) ====================

1) URL (API ENDPOINT)
--------------------------------------------------------------------------------
• A URL is the address of the server/API.
• Example:
  let url = "https://api.example.com/data";

• Alone, it does nothing → must send HTTP request to use it.

--------------------------------------------------------------------------------
2) SELECT ELEMENT (BUTTON)
--------------------------------------------------------------------------------
• We need a UI trigger (like a button).

HTML:
<button id="btn">Fetch Data</button>

JS:
let btn = document.querySelector("#btn");

• querySelector() is used to select element from DOM.

--------------------------------------------------------------------------------
3) ADD EVENT LISTENER
--------------------------------------------------------------------------------
• Used to run code when user performs an action (click).

btn.addEventListener("click", function () {
    // code runs on click
});

• Makes app interactive (not auto-running).

--------------------------------------------------------------------------------
4) ASYNC FUNCTION + AXIOS
--------------------------------------------------------------------------------
• API call is asynchronous → use async/await.

btn.addEventListener("click", async function () {

    try {
        let res = await axios.get(url); // send GET request

        console.log(res.data); // full response data

    } catch (err) {
        console.error(err); // handle error
    }

});

--------------------------------------------------------------------------------
5) DISPLAY DATA IN HTML
--------------------------------------------------------------------------------
• Add response to DOM using innerText / innerHTML

document.querySelector("#output").innerText = res.data;

• If object:
document.querySelector("#output").innerText = JSON.stringify(res.data);

• If specific field:
document.querySelector("#output").innerText = res.data.fact;

--------------------------------------------------------------------------------
6) COMPLETE FLOW
--------------------------------------------------------------------------------
User clicks button
→ Event listener triggers
→ axios sends request to URL
→ Server responds
→ Data received (res.data)
→ Data shown in HTML

--------------------------------------------------------------------------------
7) KEY CONCEPTS
--------------------------------------------------------------------------------
• URL → where request goes
• querySelector → selects HTML element
• addEventListener → handles user action
• axios.get() → sends HTTP request (returns Promise)
• async/await → handles async code cleanly
• res.data → actual API response
• DOM (innerText/innerHTML) → display data

--------------------------------------------------------------------------------
INTERVIEW LINE 🚀
--------------------------------------------------------------------------------
• On button click, we use addEventListener to trigger an async function,
  call an API using axios, and display the response data in the DOM.

NODE.JS
=====================================================================

DEFINITION
---------------------------------------------------------------------
• Node.js is a runtime environment that allows you to run JavaScript
  outside the browser (on server side).
• Built on Chrome's V8 JavaScript engine.

Note: • Runtime means the environment where code is EXECUTED


WHY USE NODE.JS
---------------------------------------------------------------------
• Fast execution (uses V8 engine)
• Non-blocking / asynchronous (handles multiple requests efficiently)
• Lightweight and scalable
• Used for building APIs, backend services, real-time apps

KEY FEATURES
---------------------------------------------------------------------
• Single-threaded event loop
• Asynchronous (non-blocking I/O)
• Event-driven architecture
• Cross-platform

HOW IT WORKS
---------------------------------------------------------------------
• Uses Event Loop:
  - Takes request → processes → sends response
  - Does NOT wait for one task to finish before starting another

EXAMPLE
---------------------------------------------------------------------
const http = require("http");

const server = http.createServer((req, res) => {
    res.end("Hello from Node.js");
});

server.listen(3000);

USES
---------------------------------------------------------------------
• Backend development (APIs)
• Real-time apps (chat apps)
• Streaming apps
• Microservices

NODE.JS VS BROWSER JS
---------------------------------------------------------------------
• Browser JS → runs in browser (UI, DOM)
• Node.js → runs on server (no DOM, backend work)

PROCESS & PROCESS.ARGV (NODE.JS)
=====================================================================

PROCESS OBJECT
---------------------------------------------------------------------
• process is a global object in Node.js
• Provides information and control over current Node.js program

COMMON USES
---------------------------------------------------------------------
• Access environment variables
• Get current working directory
• Exit the program
• Read command line arguments

EXAMPLES
---------------------------------------------------------------------
console.log(process.platform);   // OS (win32, linux, etc.)
console.log(process.cwd());      // current working directory
process.exit();                  // stop program

---------------------------------------------------------------------

PROCESS.ARGV
---------------------------------------------------------------------
• process.argv is an array
• Contains command line arguments passed to script

• Command line arguments are values passed to a program while
  running it from the terminal

STRUCTURE
---------------------------------------------------------------------
process.argv = [
    path_to_node_exe,
    path_to_script,
    arg1,
    arg2,
    ...
]

EXAMPLE
---------------------------------------------------------------------
node app.js hello 123

console.log(process.argv);

OUTPUT
---------------------------------------------------------------------
[
  'C:\\Program Files\\nodejs\\node.exe',
  'C:\\project\\app.js',
  'hello',
  '123'
]

ACCESSING VALUES
---------------------------------------------------------------------
console.log(process.argv[2]); // hello
console.log(process.argv[3]); // 123

USE CASE
---------------------------------------------------------------------
• Taking user input from terminal
• Building CLI (Command Line Interface) apps

GLOBAL OBJECT (NODE.JS)
=====================================================================

DEFINITION
---------------------------------------------------------------------
• global is the top-level object in Node.js
• Similar to "window" in browser

KEY POINTS
---------------------------------------------------------------------
• Available everywhere (no need to import)
• Used to define global variables/functions
• In modern JS, "globalThis" is preferred (works everywhere)

EXAMPLE
---------------------------------------------------------------------
global.message = "Hello";

console.log(global.message); // Hello

---------------------------------------------------------------------

COMMON GLOBALS PROVIDED BY NODE
---------------------------------------------------------------------
• console      → logging (console.log)
• setTimeout   → timer function
• setInterval  → repeated timer
• process      → system & runtime info
• __dirname    → current folder path
• __filename   → current file path

---------------------------------------------------------------------

IMPORTANT NOTE
---------------------------------------------------------------------
• Variables declared with var/let/const are NOT added to global

Example:
let x = 10;
console.log(global.x); // undefined

---------------------------------------------------------------------

SUMMARY
---------------------------------------------------------------------
• global = global object in Node.js
• Prefer using globalThis for standard usage

EXPORTING & IMPORTING FILES (NODE.JS)
=====================================================================

DEFINITION
---------------------------------------------------------------------
• Exporting → making variables/functions available from a file
• Importing → using those in another file

METHODS
---------------------------------------------------------------------
1. module.exports
2. exports (shortcut)

=====================================================================

1. EXPORT SINGLE VALUE (WHOLE FILE)
---------------------------------------------------------------------
• Export entire file as one value (function/object/string)

math.js

function add(a, b) {
    return a + b;
}

module.exports = add;

--------------------------------------------------

IMPORT (app.js)

const add = require("./math");

console.log(add(2, 3)); // 5

=====================================================================

2. EXPORT OBJECT (MULTIPLE VALUES)
---------------------------------------------------------------------
• Export multiple functions/values as object

math.js

function add(a, b) {
    return a + b;
}

function sub(a, b) {
    return a - b;
}

module.exports = { add, sub };

--------------------------------------------------

IMPORT (app.js)

const math = require("./math");

console.log(math.add(5, 2)); // 7
console.log(math.sub(5, 2)); // 3

=====================================================================

3. IMPORT USING DESTRUCTURING
---------------------------------------------------------------------
• Directly extract required functions

app.js

const { add, sub } = require("./math");

console.log(add(10, 5)); // 15
console.log(sub(10, 5)); // 5

=====================================================================

4. USING exports (SHORTCUT)
---------------------------------------------------------------------
• Shortcut to add properties to module.exports

math.js

exports.add = (a, b) => a + b;
exports.sub = (a, b) => a - b;

=====================================================================

5. EXPORT WHOLE OBJECT (CUSTOM OBJECT)
---------------------------------------------------------------------
• Export custom data/config as object

config.js

const config = {
    name: "Ajay",
    version: "1.0"
};

module.exports = config;

--------------------------------------------------

IMPORT (app.js)

const config = require("./config");

console.log(config.name); // Ajay

=====================================================================

6. IMPORT BUILT-IN MODULE
---------------------------------------------------------------------
const http = require("http");

=====================================================================

IMPORTANT NOTES
---------------------------------------------------------------------
• "./" → current folder
• require() → used to import
• module.exports → main export object
• exports → reference to module.exports
• If module.exports is reassigned → exports will break

Example (WRONG):
exports = { a: 1 }; // ❌

Example (CORRECT):
module.exports = { a: 1 }; // ✅

=====================================================================

SUMMARY
---------------------------------------------------------------------
• Single export → module.exports = value
• Multiple export → module.exports = { ... }
• Shortcut → exports.name = value
• Import → require()
