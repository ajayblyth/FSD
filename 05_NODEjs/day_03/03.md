EXPRESS.JS NOTES (STRUCTURED)
=====================================================================

1. DEFINITION
---------------------------------------------------------------------
• Express.js is a lightweight web framework for Node.js
• Used to build web servers and APIs
• Simplifies handling requests and responses

=====================================================================

2. FLOW OF EXPRESS (HOW IT WORKS)
---------------------------------------------------------------------
1. Listen for incoming requests
2. Parse incoming request data
3. Match request with routes
4. Process logic
5. Send response

=====================================================================

3. LISTEN FOR INCOMING REQUESTS
---------------------------------------------------------------------
• Server waits for client requests

Example:
const express = require('express');
const app = express();

let port = 8080;

app.listen(port, () => {
    console.log("Server running on port 3000");
});

---------------------------------------------------------------------
PORTS
---------------------------------------------------------------------
• Ports are logical endpoints of a network connection
• Used for communication between client and server
• Help identify which service/app should receive data

Example:
• 3000 → Express server
• 80   → HTTP
• 443  → HTTPS

---------------------------------------------------------------------
WHY NOT JUST IP ADDRESS?
---------------------------------------------------------------------
• IP address identifies a device (computer) ✔
• One device can run multiple applications:
  → Web server
  → Database
  → Email service

• IP alone cannot decide WHICH app should get data ❌

---------------------------------------------------------------------
ROLE OF PORT
---------------------------------------------------------------------
IP + PORT → Exact destination ✔

=====================================================================

4. PARSE INPUT REQUEST
---------------------------------------------------------------------
• Reads data from client (body, params, query)

Example:
app.use(express.json()); // for JSON data

=====================================================================

5. MATCH REQUEST WITH ROUTES
---------------------------------------------------------------------
• Based on URL + HTTP method

Example:
app.get('/', (req, res) => {
    res.send("Home Page");
});

=====================================================================

6. PROCESS LOGIC
---------------------------------------------------------------------
• Perform operations (DB, calculations, etc.)

Example:
app.get('/user', (req, res) => {
    const user = { name: "Ajay" };
    res.send(user);
});

=====================================================================

7. SEND RESPONSE
---------------------------------------------------------------------
• Send data back to client

Example:
res.send("Hello");
res.json({ msg: "Success" });

=====================================================================
EXPRESS ROUTES & ROUTING (CONNECTED CONCEPT)
=====================================================================

DEFINITION
---------------------------------------------------------------------
• Routing = process of deciding how server responds to requests ✔
• Routes = actual paths/functions that handle those requests ✔

→ Routing (process) uses Routes (implementation)

=====================================================================

WHY ROUTES?
---------------------------------------------------------------------
• Real apps have multiple functionalities:
  → Login, Register, Get users, Delete user

• One route ❌ → messy code
• Multiple routes ✔ → clean & organized

---------------------------------------------------------------------
ANALOGY
---------------------------------------------------------------------
• Like bank counters:
  Routing = deciding which counter to go
  Route   = actual counter handling work

=====================================================================

HOW ROUTING WORKS
---------------------------------------------------------------------
1. Client sends request (URL + method)
2. Express performs routing (matching)
3. Matching route is found
4. That route’s handler runs
5. Response sent back

=====================================================================

BASIC SYNTAX (ROUTES)
---------------------------------------------------------------------
app.METHOD(PATH, HANDLER)

Example:
app.get('/home', (req, res) => {
    res.send("Home Page");
});

=====================================================================

HTTP METHODS
---------------------------------------------------------------------
• GET    → Fetch data
• POST   → Send data
• PUT    → Update data
• DELETE → Remove data

=====================================================================

ROUTE MATCHING
---------------------------------------------------------------------
• Same path can behave differently based on method

Example:
app.get('/user', ...)   → fetch user
app.post('/user', ...)  → create user

=====================================================================

DYNAMIC ROUTES (PARAMETERS)
---------------------------------------------------------------------
app.get('/user/:id', (req, res) => {
    res.send(req.params.id);
});

=====================================================================

QUERY PARAMETERS
---------------------------------------------------------------------
/search?name=ajay

app.get('/search', (req, res) => {
    res.send(req.query.name);
});

=====================================================================

SUMMARY
---------------------------------------------------------------------
• Routing = decision process ✔
• Route   = actual handler ✔
• Route = METHOD + PATH + HANDLER ✔

10. LOCALHOST (127.0.0.1)
---------------------------------------------------------------------
• Special IP referring to same system

LOOPBACK
---------------------------------------------------------------------
• Request never goes to internet
• Routed back to same system

HOW IT WORKS
---------------------------------------------------------------------
1. Request → localhost
2. OS intercepts
3. Sends back to same machine
4. Local server handles it

WHY USED
---------------------------------------------------------------------
• Local testing
• No internet needed
• Fast & secure

EXAMPLE
---------------------------------------------------------------------
localhost:3000 → Node app
localhost:8080 → another service

ANALOGY
---------------------------------------------------------------------
• Like calling your own phone

11. app.use() (MIDDLEWARE)
---------------------------------------------------------------------
DEFINITION
---------------------------------------------------------------------
• Used to register middleware
• Middleware runs between request & response

---------------------------------------------------------------------
WHY USE?
---------------------------------------------------------------------
• Common logic for multiple routes
• Avoid repetition
• Used for parsing, logging, auth

---------------------------------------------------------------------
HOW IT WORKS
---------------------------------------------------------------------
1. Request comes
2. Middleware runs
3. Passes to next
4. Response sent

---------------------------------------------------------------------
SYNTAX
---------------------------------------------------------------------
app.use((req, res, next) => {
    console.log("Middleware running");
    next();
});

---------------------------------------------------------------------
COMMON USES
---------------------------------------------------------------------
app.use(express.json());

app.use((req, res, next) => {
    console.log(req.method, req.url);
    next();
});

---------------------------------------------------------------------
PATH BASED
---------------------------------------------------------------------
app.use('/user', (req, res, next) => {
    console.log("User middleware");
    next();
});

---------------------------------------------------------------------
IMPORTANT
---------------------------------------------------------------------
• next() is required to pass control ✔
• Without next() → request gets stuck ❌

---------------------------------------------------------------------
SUMMARY
---------------------------------------------------------------------
app.use() → middleware handler ✔
Used before routes for common processing ✔

PATH PARAMETERS (EXPRESS)
------------------------------------------------------------

DEFINITION
------------------------------------------------------------
• Path parameters are dynamic values in the URL
• Used to send specific data as part of the path

------------------------------------------------------------

SYNTAX
------------------------------------------------------------
/path/:parameterName

Example:
app.get('/user/:id', (req, res) => {
    res.send(req.params.id);
});

------------------------------------------------------------

HOW IT WORKS
------------------------------------------------------------
• URL:
  /user/101

• Express extracts:
  id = 101

• Access using:
  req.params.id

------------------------------------------------------------

MULTIPLE PARAMETERS
------------------------------------------------------------
app.get('/user/:id/order/:orderId', (req, res) => {
    res.send({
        userId: req.params.id,
        orderId: req.params.orderId
    });
});

URL:
 /user/10/order/500

------------------------------------------------------------

IMPORTANT POINTS
------------------------------------------------------------
• Defined using colon (:) ✔
• Always part of URL path (not query)
• Values are strings by default
• Useful for identifying specific resources

------------------------------------------------------------

USE CASES
------------------------------------------------------------
• Get user by ID → /user/101
• Get product → /product/55
• Delete item → /item/20

------------------------------------------------------------

SUMMARY
------------------------------------------------------------
Path parameter = dynamic value in URL ✔
Access using req.params ✔

QUERY STRINGS (EXPRESS)
------------------------------------------------------------

DEFINITION
------------------------------------------------------------
• Query strings are key-value pairs sent in URL
• Used to pass optional/filter data to server

------------------------------------------------------------

SYNTAX
------------------------------------------------------------
/path?key=value

Example:
 /search?name=ajay&age=22

------------------------------------------------------------

HOW IT WORKS
------------------------------------------------------------
• URL:
  /search?name=ajay&age=22

• Express extracts:
  name = ajay
  age  = 22

• Access using:
  req.query.name
  req.query.age

------------------------------------------------------------

EXAMPLE IN CODE
------------------------------------------------------------
app.get('/search', (req, res) => {
    const name = req.query.name;
    const age = req.query.age;

    res.send({
        name: name,
        age: age
    });
});

------------------------------------------------------------

IMPORTANT POINTS
------------------------------------------------------------
• Starts with ? in URL ✔
• Multiple params separated by & ✔
• Values are strings by default
• Optional (route works even without them)

------------------------------------------------------------

USE CASES
------------------------------------------------------------
• Filtering → /products?category=mobile
• Searching → /search?name=ajay
• Pagination → /users?page=2

------------------------------------------------------------

PATH PARAM vs QUERY STRING
------------------------------------------------------------
• Path param → required (part of URL)
  /user/101

• Query string → optional
  /user?id=101

------------------------------------------------------------

SUMMARY
------------------------------------------------------------
Query string = optional data in URL ✔
Access using req.query ✔
