=====================================================================
NPM (NODE PACKAGE MANAGER) - NODE.JS (CLEAN + STRUCTURED)
=====================================================================

DEFINITION
---------------------------------------------------------------------
• NPM → Default package manager for Node.js
• Used to install, manage, share packages (npmjs.com)

---------------------------------------------------------------------
KEY COMPONENTS
---------------------------------------------------------------------

1. PACKAGE
   • Reusable code (library/module)
   • Example → express, axios

---------------------------------------------------------------------

2. package.json (MAIN FILE)
   • Created by developer
   • Stores project info, dependencies, scripts

DEPENDENCIES
---------------------------------------------------------------------
• dependencies → Required to run app (production)
• Example → express, axios
• Install → npm install <pkg>

DEVDEPENDENCIES
---------------------------------------------------------------------
• devDependencies → Only for development
• Example → nodemon, jest
• Install → npm install <pkg> --save-dev

DIFFERENCE
---------------------------------------------------------------------
• dependencies → runtime
• devDependencies → development
• Production skips devDependencies

---------------------------------------------------------------------

3. SCRIPTS
---------------------------------------------------------------------
• Custom commands inside package.json
• Format → "key": "command"

Examples:
• "start": "node index.js"
• "dev": "nodemon app.js"

Run:
• npm run dev
• npm start (special shortcut)

---------------------------------------------------------------------

4. node_modules
---------------------------------------------------------------------
• Folder storing all installed packages
• Auto-created by npm

---------------------------------------------------------------------

5. package-lock.json
---------------------------------------------------------------------
• Auto-generated file
• Stores exact versions (no changes)
• Ensures same install everywhere

------------------------------------------------------------------
NPM COMMANDS (EXPLAINED)
------------------------------------------------------------

1. npm init
• First command to run in any Node project ✔
• Creates package.json by asking questions (name, version, etc.)
• Sets up project configuration

------------------------------------------------------------

2. npm init -y
• Same as npm init but skips all questions
• Automatically creates package.json with default values
• Faster way to start project

------------------------------------------------------------

3. npm install <pkg>
• Installs package locally (inside node_modules)
• Adds it to dependencies in package.json
• Used when package is needed only for this project

------------------------------------------------------------

4. npm install -g <pkg>
• Installs package globally (usable anywhere in system)
• Not added to project’s package.json
• Used for tools like nodemon, typescript

------------------------------------------------------------

5. npm uninstall <pkg>
• Removes package from node_modules
• Also removes from package.json dependencies
• Keeps project clean from unused packages

------------------------------------------------------------

6. npm install pkg@version
• Installs a specific version of a package
• Useful to avoid breaking changes from latest version
• Example: npm install express@4.18.2

------------------------------------------------------------

SUMMARY
------------------------------------------------------------
Start → npm init ✔
Then install/remove/manage packages using npm commands

---------------------------------------------------------------------
LOCAL vs GLOBAL (IMPORTANT)
---------------------------------------------------------------------
• Default → local install (node_modules)
• Works only inside that project
• Running app.js outside → module not found
• Global → npm install -g <pkg>
• Can run anywhere (e.g., nodemon)

---------------------------------------------------------------------
WORKFLOW
---------------------------------------------------------------------
1. npm init
2. npm install <pkg>
3. Use in code
4. Run using scripts

---------------------------------------------------------------------
PACKAGE.JSON vs PACKAGE-LOCK.JSON
---------------------------------------------------------------------
• package.json → what to install
• package-lock.json → exact installed version

---------------------------------------------------------------------
NPM LINK (CONNECT LOCAL PACKAGE TO APP)
---------------------------------------------------------------------

Concept:
• my-package → package you are building
• my-app → project where you use it

STEP 1 (inside package folder)
---------------------------------------------------------------------
cd my-package
npm link
→ Makes package globally available (local symlink)

STEP 2 (inside app folder)
---------------------------------------------------------------------
cd my-app
npm link my-package
→ Connects your app to local package

RESULT
---------------------------------------------------------------------
• You can use your own package in app
• Changes in package reflect directly (no reinstall needed)

=====================================================================
WHY USE NPM
=====================================================================
• Huge ecosystem
• Easy dependency management
• Saves development time

REQUIRE vs IMPORT (NODE.js)
------------------------------------------------------------

BASICS
------------------------------------------------------------
1. require → CommonJS (older)
2. import  → ES Modules (modern)

SYNTAX
------------------------------------------------------------
3. require → const x = require('module')
4. import  → import x from 'module'

USAGE
------------------------------------------------------------
5. require → Default in older Node.js
6. import  → Needs "type":"module" in package.json

BEHAVIOUR
------------------------------------------------------------
7. require → Synchronous (loads immediately, blocks)
8. import  → Static import (resolved before execution)

IMPORT vs IMPORT()
------------------------------------------------------------
9. import   → Static, top-level only
10.import() → Dynamic, can be used anywhere

11.import   → Loads module at start
12.import() → Loads on demand (lazy loading)

13.import   → Cannot use await
14.import() → Async, returns Promise (can use await)

SELECTIVE IMPORT
------------------------------------------------------------
15.require → Imports full module (module.exports)
16.import  → Selective → import {a, b} from 'module'
17.Example → import {add, sub} from "./math.js"

IMPORT ALL (NAMESPACE IMPORT)
------------------------------------------------------------
18.import * as math from "./math.js"
19.Access → math.add(), math.sub()

NOTE
------------------------------------------------------------
20.require → No config needed
21.import  → Modern, cleaner & optimized

NOTE
------------------------------------------------------------
• Use either CommonJS OR ES Modules in a project ✔
• Mixing both can cause errors ❌

------------------------------------------------------------

1. COMMONJS (OLD STYLE)
------------------------------------------------------------
• Uses require and module.exports

Example:
const greet = require('./file');
module.exports = greet;

------------------------------------------------------------

2. ES MODULES (NEW STYLE)
------------------------------------------------------------
• Uses import and export

Example:
import greet from './file.js';
export default greet;

------------------------------------------------------------

IMPORTANT
------------------------------------------------------------
• To use ES Modules → add in package.json:
  "type": "module"

• File extension (.js) required in import
