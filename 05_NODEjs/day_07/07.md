METHOD OVERRIDE (PATCH) - NOTES (EXPRESS + EJS)
=============================================================================

WHAT IS METHOD OVERRIDE?
-----------------------------------------------------------------------------
HTML forms support only:
- GET
- POST

But REST APIs need:
- PATCH (update)
- DELETE

So we use "method-override" to convert POST → PATCH/DELETE


INSTALLATION
-----------------------------------------------------------------------------
npm install method-override


SETUP IN EXPRESS
-----------------------------------------------------------------------------
const methodOverride = require('method-override');

app.use(methodOverride('_method'));

NOTE:
- '_method' is the query parameter name


HOW IT WORKS
-----------------------------------------------------------------------------
Form sends request as:
POST /posts/1?_method=PATCH

method-override converts it to:
PATCH /posts/1


FRONTEND (EJS / HTML FORM)
-----------------------------------------------------------------------------
<form action="/posts/<%= post.id %>?_method=PATCH" method="POST">
    <input type="text" name="content" />
    <button type="submit">Update</button>
</form>


BACKEND ROUTE (PATCH)
-----------------------------------------------------------------------------
app.patch('/posts/:id', (req, res) => {
    let { id } = req.params;
    let { content } = req.body;

    let post = posts.find(p => p.id === id);
    post.content = content;

    res.redirect('/posts');
});


FLOW (INTERVIEW IMPORTANT)
-----------------------------------------------------------------------------
1. User submits form (POST)
2. URL contains ?_method=PATCH
3. method-override middleware runs
4. Converts POST → PATCH
5. Express handles app.patch()


PATCH vs PUT
-----------------------------------------------------------------------------
PATCH  -> Partial update (only specific field)
PUT    -> Full replace (entire object)


COMMON MISTAKES
-----------------------------------------------------------------------------
1. Forgot middleware setup
   app.use(methodOverride('_method'))

2. Using method="PATCH" in form (WRONG)

3. Missing ?_method=PATCH in form action

4. Wrong file path or route mismatch


SUMMARY
-----------------------------------------------------------------------------
POST + _method=PATCH  =>  acts like PATCH request

Used to update data from HTML forms in Express apps
