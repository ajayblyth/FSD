HTTP REQUESTS (HYPERTEXT TRANSFER PROTOCOL) – NOTES
=============================================================================

DEFINITION
-----------------------------------------------------------------------------
• HTTP Request is a message sent by client (browser/app) to server.
• Used to request data, send data, or perform operations on server.
• Works on Request → Response model.

BASIC STRUCTURE
-----------------------------------------------------------------------------
1. Request Line
2. Headers
3. Blank Line
4. Body (optional)

Example:
-----------------------------------------------------------------------------
GET /api/users HTTP/1.1
Host: example.com
User-Agent: Chrome

-----------------------------------------------------------------------------

HTTP METHODS (VERY IMPORTANT)
-----------------------------------------------------------------------------
METHOD     PURPOSE
-----------------------------------------------------------------------------
GET        Retrieve data from server
POST       Send new data to server (create)
PUT        Update/replace existing data (full update)
PATCH      Update partial data
DELETE     Remove data
HEAD       Same as GET but returns only headers
OPTIONS    Shows allowed methods for resource

-----------------------------------------------------------------------------

GET vs POST (INTERVIEW POINT)
-----------------------------------------------------------------------------
GET
• Used to retrieve data from server.
• Data is sent in URL (query parameters).
• Visible in browser URL.
• Can be cached and bookmarked.
• No request body is used.

POST
• Used to send/create data on server.
• Data is sent in request body.
• Not visible in URL.
• Not cached or bookmarked.
• Supports large amount of data.

Handling Post requests
Set up POST request route to get some response

Parse POST request data

JavaScript
app.use(express.urlencoded({ extended: true }));
app.use(express.json());

-----------------------------------------------------------------------------

HTTP STATUS CODES
-----------------------------------------------------------------------------
CODE   MEANING
-----------------------------------------------------------------------------
1xx    Informational
2xx    Success
       200 OK
       201 Created
       204 No Content

3xx    Redirection
       301 Moved Permanently
       302 Found

4xx    Client Error
       400 Bad Request
       401 Unauthorized
       403 Forbidden
       404 Not Found

5xx    Server Error
       500 Internal Server Error
       503 Service Unavailable

-----------------------------------------------------------------------------

HEADERS (IMPORTANT)
-----------------------------------------------------------------------------
• Key-value pairs providing extra info

COMMON HEADERS:
-----------------------------------------------------------------------------
Host           → Server domain
User-Agent     → Client info (browser/app)
Content-Type   → Type of data (JSON, HTML)
Authorization  → Token/credentials
Accept         → Expected response type

Example:
Content-Type: application/json

-----------------------------------------------------------------------------

REQUEST BODY
-----------------------------------------------------------------------------
• Contains data sent to server (mainly in POST/PUT/PATCH)

Example (JSON):
-----------------------------------------------------------------------------
{
  "name": "Ajay",
  "age": 25
}

-----------------------------------------------------------------------------

QUERY PARAMETERS
-----------------------------------------------------------------------------
• Extra data in URL after '?'

Example:
-----------------------------------------------------------------------------
/api/users?age=25&city=Delhi

• Used for filtering/searching

-----------------------------------------------------------------------------

PATH PARAMETERS
-----------------------------------------------------------------------------
• Dynamic values in URL path

Example:
-----------------------------------------------------------------------------
/api/users/101

• Here 101 is user ID

-----------------------------------------------------------------------------

HTTP vs HTTPS
-----------------------------------------------------------------------------
HTTP
• Not secure
• Data sent in plain text

HTTPS
• Secure (SSL/TLS encryption)
• Data encrypted
• Used in real-world apps

-----------------------------------------------------------------------------

IDEMPOTENT METHODS (IMPORTANT CONCEPT)
-----------------------------------------------------------------------------
• Same request multiple times → same result

Idempotent:
GET, PUT, DELETE

Not Idempotent:
POST

-----------------------------------------------------------------------------

COMMON FLOW (REAL WORLD)
-----------------------------------------------------------------------------
1. Client sends request (GET /users)
2. Server processes request
3. Server sends response (data + status code)

-----------------------------------------------------------------------------

URL ENCODED IN SERVER (EXPRESS JS)
=============================================================================

WHY NEEDED
-----------------------------------------------------------------------------
• Browser sends form data in urlencoded format.
• Server cannot read it directly.
• Middleware is required to parse it into usable object.

-----------------------------------------------------------------------------

EXPRESS MIDDLEWARE
-----------------------------------------------------------------------------
app.use(express.urlencoded({ extended: true }));

-----------------------------------------------------------------------------

WHAT IT DOES
-----------------------------------------------------------------------------
• Converts incoming data into JavaScript object.
• Makes data available inside: req.body

Example Input (from form):
-----------------------------------------------------------------------------
name=Ajay&age=25

After parsing:
-----------------------------------------------------------------------------
req.body = {
  name: "Ajay",
  age: "25"
}

-----------------------------------------------------------------------------

EXTENDED: TRUE vs FALSE
-----------------------------------------------------------------------------
extended: true
• Uses "qs" library
• Supports nested objects

Example:
user[name]=Ajay&user[age]=25

→ req.body:
{
  user: {
    name: "Ajay",
    age: "25"
  }
}

-----------------------------------------------------------------------------

extended: false
• Uses "querystring" (simple parsing)
• Does NOT support nested objects

Same input:
user[name]=Ajay

→ req.body:
{
  "user[name]": "Ajay"
}

-----------------------------------------------------------------------------

FULL EXPRESS EXAMPLE
-----------------------------------------------------------------------------
const express = require('express');
const app = express();

app.use(express.urlencoded({ extended: true }));

app.post('/submit', (req, res) => {
    console.log(req.body);
    res.send("Data received");
});

app.listen(3000);

-----------------------------------------------------------------------------

HTML FORM EXAMPLE
-----------------------------------------------------------------------------
<form action="/submit" method="POST">
    <input type="text" name="name">
    <input type="number" name="age">
    <button type="submit">Send</button>
</form>

-----------------------------------------------------------------------------

IMPORTANT POINTS
-----------------------------------------------------------------------------
• Works only for urlencoded data (forms)
• For JSON → use express.json()
• Must be used BEFORE routes
• req.body is undefined without middleware

-----------------------------------------------------------------------------

FLOW (END TO END)
-----------------------------------------------------------------------------
1. User submits form
2. Browser sends urlencoded data
3. express.urlencoded() parses it
4. Data available in req.body
5. Server processes it

OBJECT ORIENTED PROGRAMMING (OOP) – STRUCTURE NOTES
=============================================================================

PURPOSE
-----------------------------------------------------------------------------
• Used to structure and organize code.
• Makes code reusable, scalable, and maintainable.

-----------------------------------------------------------------------------

CORE FLOW (JS OOP LEARNING PATH)
-----------------------------------------------------------------------------
1. Object & Prototypes
2. Constructor Functions
3. new Operator
4. Classes (ES6)
5. Keywords (extends, super)

-----------------------------------------------------------------------------

1. OBJECT PROTOTYPES
-----------------------------------------------------------------------------
• Prototype is an object in JavaScript from which other objects inherit properties and methods.

• Every JavaScript object has an internal link to a prototype object.

• If a property/method is not found in an object, JavaScript looks into its prototype (prototype chain).

Example:
-----------------------------------------------------------------------------
obj → obj.__proto__ → Object.prototype → null

Example:
-----------------------------------------------------------------------------
let obj = {
  name: "Ajay"
};

console.log(obj.__proto__); // points to Object prototype

-----------------------------------------------------------------------------

PROTOTYPE CHAIN
-----------------------------------------------------------------------------
• If property not found in object → JS checks prototype
• This continues → called prototype chaining

-----------------------------------------------------------------------------
WAYS TO ACCESS PROTOTYPE (ORGANIZED)
=============================================================================

METHODS
-----------------------------------------------------------------------------
1. __proto__                 → Direct reference (old, not recommended)
2. Object.getPrototypeOf()  → Get prototype (recommended)
3. Object.setPrototypeOf()  → Set prototype

-----------------------------------------------------------------------------

KEY CONCEPTS (IMPORTANT)
-----------------------------------------------------------------------------
• arr.__proto__         → Reference to prototype
• Array.prototype      → Actual prototype object

• str.__proto__         → Reference
• String.prototype     → Actual object

ADVANTAGES / USE OF PROTOTYPE
-----------------------------------------------------------------------------

• Saves memory (shared methods, no duplication)
• Enables code reusability
• Supports inheritance (prototype chain)
• Improves performance (less memory usage)
• Allows dynamic updates (add methods later)
• Forms base of JavaScript OOP (used in classes/constructors)
-----------------------------------------------------------------------------

2. CONSTRUCTOR FUNCTIONS
-----------------------------------------------------------------------------
• Used to create multiple objects with same structure

Example:
-----------------------------------------------------------------------------
function Person(name, age) {
  this.name = name;
  this.age = age;
}

-----------------------------------------------------------------------------

3. NEW OPERATOR
-----------------------------------------------------------------------------
• Used with constructor to create objects

WHAT IT DOES:
-----------------------------------------------------------------------------
1. Creates empty object {}
2. Sets prototype (links to constructor.prototype)
3. Binds "this" to new object
4. Returns object

Example:
-----------------------------------------------------------------------------
let p1 = new Person("Ajay", 25);

-----------------------------------------------------------------------------

4. CLASSES (ES6)
-----------------------------------------------------------------------------
• Modern syntax for constructor + prototype

Example:
-----------------------------------------------------------------------------
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log("Hello " + this.name);
  }
}

-----------------------------------------------------------------------------

5. KEYWORDS (IMPORTANT)
-----------------------------------------------------------------------------
extends
• Used for inheritance

super
• Calls parent constructor

Example:
-----------------------------------------------------------------------------
class Student extends Person {
  constructor(name, age, marks) {
    super(name, age);
    this.marks = marks;
  }
}

-----------------------------------------------------------------------------

IMPORTANT SUMMARY
-----------------------------------------------------------------------------
• Prototypes → base concept of inheritance
• Constructor → blueprint for objects
• new → creates object + links prototype
• Classes → cleaner syntax of same concept
• extends/super → used for inheritance

=============================================================================
Object.getPrototypeOf(value)
-----------------------------------------------------------------------------

DEFINITION
-----------------------------------------------------------------------------
• Returns the prototype (parent object) of a given value.
• Used to see from where the object inherits properties/methods.

-----------------------------------------------------------------------------

SYNTAX
-----------------------------------------------------------------------------
Object.getPrototypeOf(value)

-----------------------------------------------------------------------------

EXAMPLE 1 (STRING)
-----------------------------------------------------------------------------
let str = "hello";

console.log(Object.getPrototypeOf(str));

→ Output:
String.prototype (contains methods like toUpperCase, slice, etc.)

-----------------------------------------------------------------------------

EXAMPLE 2 (ARRAY)
-----------------------------------------------------------------------------
let arr = [1, 2, 3];

console.log(Object.getPrototypeOf(arr));

→ Output:
Array.prototype (contains methods like push, pop, map, etc.)

-----------------------------------------------------------------------------

SUMMARY
-----------------------------------------------------------------------------
• Finds parent (prototype) of any value
• Helps understand inheritance in JavaScript

-----------------------------------------------------------------------------
PROTOTYPE EXAMPLE (WHY NEEDED FOR SHARING METHODS)
-----------------------------------------------------------------------------

PROBLEM (WITHOUT PROTOTYPE)
-----------------------------------------------------------------------------
let arr1 = [1, 2, 3];
let arr2 = [4, 5, 6];

arr1.sayHello = function() {
  console.log("Hello from arr1");
};

arr1.sayHello(); // ✅ works
arr2.sayHello(); // ❌ ERROR (not defined in arr2)

• Method is only inside arr1 → not reusable
• If we add same function to many arrays → memory waste

-----------------------------------------------------------------------------

SOLUTION (USING PROTOTYPE)
-----------------------------------------------------------------------------
Array.prototype.sayHello = function() {
  console.log("Hello from prototype");
};

let arr1 = [1, 2, 3];
let arr2 = [4, 5, 6];

arr1.sayHello(); // ✅ works
arr2.sayHello(); // ✅ works

-----------------------------------------------------------------------------

HOW IT WORKS
-----------------------------------------------------------------------------
arr1 → Array.prototype → Object.prototype
arr2 → Array.prototype → Object.prototype

• Both arrays are linked to same Array.prototype
• Method stored once → shared by all arrays

-----------------------------------------------------------------------------

ADVANTAGE
-----------------------------------------------------------------------------
• No duplication of function
• Saves memory
• Enables inheritance (all arrays can use it)
• Same concept used by built-in methods (push, pop, map)

-----------------------------------------------------------------------------

REAL UNDERSTANDING
-----------------------------------------------------------------------------
• arr1 doesn’t have sayHello()
• JS looks into its prototype → finds it
• This is called PROTOTYPE CHAIN

-----------------------------------------------------------------------------

=============================================================================
FACTORY FUNCTION vs CONSTRUCTOR FUNCTION (WITH PROTOTYPE)
=============================================================================

1. FACTORY FUNCTION (NORMAL FUNCTION), a function that creates object
-----------------------------------------------------------------------------
function Student(name, age, id) {
  return {
    name: name,
    age: age,
    id: id,
    getDetails: function() {
      console.log(this.name, this.age, this.id);
    }
  };
}

let s1 = Student("Nithin", 22, 12);

-----------------------------------------------------------------------------

CHARACTERISTICS
-----------------------------------------------------------------------------
• No "new" keyword used
• Returns object manually
• Each object gets its own copy of methods ❌
• More memory usage (function recreated every time)

-----------------------------------------------------------------------------

=============================================================================
2. CONSTRUCTOR FUNCTION + PROTOTYPE (ORGANIZED NOTES)
=============================================================================

CODE
-----------------------------------------------------------------------------
function Student(name, age, id) {
  this.name = name;
  this.age = age;
  this.id = id;
}

Student.prototype.getDetails = function() {
  console.log(this.name, this.age, this.id);
};
// ⚠️ Do NOT use arrow function here

let s1 = new Student("Nithin", 22, 12);

console.log(s1);

-----------------------------------------------------------------------------

WHAT "new" DOES
-----------------------------------------------------------------------------
• Creates empty object {}
• Sets prototype → Student.prototype
• Binds this → new object
• Returns the object

-----------------------------------------------------------------------------

WHY NOT ARROW FUNCTION
-----------------------------------------------------------------------------
❌ Wrong:
Student.prototype.getDetails = () => {
  console.log(this.name);
};

• Arrow function has no own "this"
• It takes "this" from outer scope (not s1)
• So this.name will be undefined

✔️ Correct:
Use normal function → gets "this" from caller (s1)

-----------------------------------------------------------------------------

PROTOTYPE LINK
-----------------------------------------------------------------------------
s1 → Student.prototype → Object.prototype → null

-----------------------------------------------------------------------------

WHAT console.log(s1) SHOWS
-----------------------------------------------------------------------------
{
  name: "Nithin",
  age: 22,
  id: 12
}

• getDetails() is NOT inside object directly
• It exists in prototype (shared)

-----------------------------------------------------------------------------

HOW METHOD WORKS
-----------------------------------------------------------------------------
s1.getDetails()

• JS checks inside s1 → not found
• Goes to prototype → finds getDetails()
• Executes with this = s1

-----------------------------------------------------------------------------

FINAL SUMMARY
-----------------------------------------------------------------------------
• Use "new" → to create object + link prototype
• Store methods in prototype → memory efficient
• Use normal function → correct "this"
• Prototype enables inheritance

-----------------------------------------------------------------------------

-----------------------------------------------------------------------------

CHARACTERISTICS
-----------------------------------------------------------------------------
• Used with "new" keyword
• Object created automatically
• Methods stored in prototype (shared) ✅
• Memory efficient

-----------------------------------------------------------------------------

KEY DIFFERENCE
-----------------------------------------------------------------------------
FACTORY FUNCTION
• Method created for every object ❌
• Higher memory usage

CONSTRUCTOR + PROTOTYPE
• Method shared across objects ✅
• Lower memory usage

-----------------------------------------------------------------------------

THIS KEYWORD BEHAVIOR
-----------------------------------------------------------------------------
WITH new
• this → points to newly created object

WITHOUT new
• this → global object (window) OR undefined (strict mode)

-----------------------------------------------------------------------------

FINAL SUMMARY
-----------------------------------------------------------------------------
• Factory → simple but not memory efficient
• Constructor + Prototype → scalable and efficient
• Always use prototype for shared methods
• Always use "new" with constructor

=============================================================================
CONSTRUCTOR + NEW KEYWORD (IMPORTANT NOTES)
=============================================================================

CONSTRUCTOR FUNCTION
-----------------------------------------------------------------------------
• Special function used to create objects
• By convention → starts with Capital letter

Example:
-----------------------------------------------------------------------------
function Student(name, age) {
  this.name = name;
  this.age = age;
  // no return
}

-----------------------------------------------------------------------------

DO WE RETURN IN CONSTRUCTOR?
-----------------------------------------------------------------------------
• Normally → NO return statement

WHY?
-----------------------------------------------------------------------------
• "new" automatically returns the created object

-----------------------------------------------------------------------------

WHAT "new" DOES
-----------------------------------------------------------------------------
1. Creates empty object {}
2. Sets prototype → Student.prototype
3. Binds this → new object
4. Runs constructor code
5. Returns the object automatically

-----------------------------------------------------------------------------

EXAMPLE
-----------------------------------------------------------------------------
let s1 = new Student("Ajay", 25);

-----------------------------------------------------------------------------

SPECIAL CASE (IMPORTANT)
-----------------------------------------------------------------------------
function Test() {
  this.name = "Ajay";
  return { city: "Delhi" };
}

let t1 = new Test();

console.log(t1); 
// { city: "Delhi" }

-----------------------------------------------------------------------------
• If you return an OBJECT → it replaces default object

-----------------------------------------------------------------------------

ANOTHER CASE
-----------------------------------------------------------------------------
function Test() {
  this.name = "Ajay";
  return 10;
}

let t1 = new Test();

console.log(t1);
// { name: "Ajay" }

-----------------------------------------------------------------------------
• If you return primitive → ignored

-----------------------------------------------------------------------------

SUMMARY
-----------------------------------------------------------------------------
• No need to return in constructor
• "new" returns object automatically
• Return object → overrides default
• Return primitive → ignored
